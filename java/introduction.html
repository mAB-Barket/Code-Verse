Introduction to Java

What is Java?

Java is a high-level, class-based, object-oriented programming language that was designed to have as few implementation dependencies as possible. It follows the principle of "Write Once, Run Anywhere" (WORA), meaning that compiled Java code can run on any platform that supports Java without the need for recompilation. Java is used in a vast range of applications — from Android mobile apps and enterprise web servers to scientific computing and cloud-based platforms.

Java was created by James Gosling at Sun Microsystems and was first released in 1995. Originally called "Oak" and later "Green," the language was eventually renamed Java, inspired by Java coffee from Indonesia. Sun Microsystems was later acquired by Oracle Corporation in 2010, which now maintains and develops the language. Java has since become one of the most widely adopted programming languages in history, powering billions of devices worldwide.

💡

Fun Fact: Java was originally designed for interactive television, but the technology was too advanced for the digital cable television industry at the time. It later found its perfect home in web and enterprise development.

Key Features of Java

Java offers several powerful features that have contributed to its enduring popularity:

  • Write Once, Run Anywhere (WORA): Java source code is compiled into platform-independent bytecode that runs on any system with a Java Virtual Machine (JVM).
  • Object-Oriented Programming (OOP): Java is built on OOP principles — encapsulation, inheritance, polymorphism, and abstraction — enabling modular, reusable, and maintainable code.
  • Platform Independence: Unlike C or C++, Java programs are not compiled to a specific processor architecture. The JVM abstracts away hardware differences.
  • Automatic Garbage Collection: Java automatically manages memory by reclaiming objects that are no longer in use, reducing the risk of memory leaks and dangling pointers.
  • Strong Typing: Java enforces strict type checking at compile time, catching many errors before your code ever runs. Every variable must be declared with a specific data type.
  • Rich Standard Library: Java comes with an extensive standard library (Java API) covering data structures, networking, file I/O, concurrency, and much more.
  • Multithreading: Java has built-in support for concurrent programming, allowing multiple threads of execution within a single program.

Java vs Other Languages

Java occupies a unique position among programming languages. Unlike Python, which is interpreted and dynamically typed, Java is compiled to bytecode and statically typed, offering better performance and earlier error detection. Compared to C++, Java trades some raw performance for automatic memory management and platform independence. Unlike JavaScript, which runs primarily in web browsers, Java is a general-purpose language used on servers, desktops, and mobile devices. Java's combination of performance, portability, and robustness makes it the backbone of enterprise software development.

JDK, JRE, and JVM

Understanding the Java ecosystem requires knowing three key components:

  • JVM (Java Virtual Machine): The engine that executes Java bytecode. Each operating system has its own JVM implementation, which is what makes Java platform-independent. The JVM reads the compiled .class files and translates the bytecode into machine-specific instructions at runtime.
  • JRE (Java Runtime Environment): The JRE includes the JVM plus the core class libraries needed to run Java applications. If you only need to run Java programs, the JRE is sufficient.
  • JDK (Java Development Kit): The JDK is the complete development toolkit. It includes the JRE, the Java compiler (javac), the debugger, and other development tools. You need the JDK to write and compile Java programs.

To start developing in Java, download the latest JDK from oracle.com or use an open-source distribution like OpenJDK. After installation, verify it by opening your terminal and typing:

java 
// Check your Java version in the terminal
// Type these commands in your terminal (not in a Java file):
// java --version
// javac --version

// You should see output like:
// java 21.0.1 2023-10-17 LTS
// javac 21.0.1

Your First Java Program

Every programmer's journey begins with "Hello, World!" — a simple program that displays text on the screen. In Java, every application must have at least one class, and execution begins in the main method. Create a new file called HelloWorld.java (the filename must match the class name) and type the following:

java 
// My first Java program
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

To compile and run this program, open your terminal, navigate to the directory containing your file, and type:

java 
// Step 1: Compile the source code into bytecode
// javac HelloWorld.java

// Step 2: Run the compiled bytecode with the JVM
// java HelloWorld

// Output:
// Hello, World!

The javac command compiles your .java source file into a .class file containing bytecode. The java command then launches the JVM, which loads and executes that bytecode. Notice how Java requires a class wrapper and a main method — this is the entry point for every Java application. The public keyword makes the class and method accessible from anywhere, static means the method belongs to the class itself (not an instance), and void indicates the method doesn't return a value.

📝

Note: In Java, the filename must match the public class name exactly (including case). If your class is called HelloWorld, the file must be named HelloWorld.java. Java is case-sensitive, so helloworld.java would cause a compilation error.

⚠

Important: Every statement in Java must end with a semicolon (;). Unlike Python, which uses indentation to define code blocks, Java uses curly braces {}. While indentation is not required by the compiler, it is strongly recommended for readability.
