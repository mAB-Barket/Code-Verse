Strings in Java

Creating Strings

In Java, a String is not a primitive type — it's a reference type (an object of the java.lang.String class). Strings are one of the most commonly used types in Java and represent sequences of characters. There are two main ways to create a String: using a string literal or the new keyword.

When you create a string using a literal, Java checks the String Pool (a special area in memory) to see if an identical string already exists. If it does, Java reuses it instead of creating a new object — this saves memory. When you use the new keyword, a new String object is always created in heap memory, even if an identical string exists in the pool.

Like Python, strings in Java are immutable — once a String object is created, its content cannot be changed. Any operation that appears to modify a string actually creates a new String object. This immutability makes strings thread-safe and allows for safe sharing across your program.

java 
// Creating strings — two approaches
String s1 = "Hello";                  // string literal (uses String Pool)
String s2 = new String("Hello");     // new object (uses heap memory)

// Both hold the same text, but are stored differently
String greeting = "Hello, World!";
String language = "Java";
String empty = "";                    // empty string

// Strings are immutable — cannot change characters in place
// greeting[0] = 'h';  // ERROR! Strings are immutable

// String length
System.out.println(greeting.length());  // 13

// Accessing characters
System.out.println(greeting.charAt(0));  // H
System.out.println(greeting.charAt(7));  // W
💡

Tip: Always prefer string literals ("Hello") over new String("Hello"). Literals are more memory-efficient because they use the String Pool, and your code will be cleaner and faster.

String Methods

Java's String class comes with a rich set of built-in methods for manipulating and inspecting text. Since strings are immutable, these methods always return a new String — the original string remains unchanged.

java 
String text = "  Hello, Java World!  ";

// Length
System.out.println(text.length());           // 22

// Character access
System.out.println(text.charAt(2));            // H

// Substring extraction
System.out.println(text.substring(2, 7));      // Hello
System.out.println(text.substring(9));         // Java World!  

// Case conversion
System.out.println(text.toUpperCase());       // "  HELLO, JAVA WORLD!  "
System.out.println(text.toLowerCase());       // "  hello, java world!  "

// Search
System.out.println(text.contains("Java"));    // true
System.out.println(text.indexOf("Java"));     // 9
System.out.println(text.indexOf("Python"));   // -1 (not found)

// Whitespace removal
System.out.println(text.trim());              // "Hello, Java World!"

// Replace
System.out.println(text.replace("Java", "Python"));  // "  Hello, Python World!  "

// Check start and end
String clean = text.trim();
System.out.println(clean.startsWith("Hello"));  // true
System.out.println(clean.endsWith("!"));        // true

// Equality check
System.out.println(clean.equals("Hello, Java World!"));       // true
System.out.println(clean.equalsIgnoreCase("hello, java world!")); // true

String Concatenation

There are several ways to combine strings in Java. The simplest approach is using the + operator, which is intuitive and works well for small concatenations. You can also use the concat() method, which joins two strings together.

java 
// String concatenation with + operator
String firstName = "John";
String lastName = "Doe";
String fullName = firstName + " " + lastName;
System.out.println(fullName);  // John Doe

// Concatenation with numbers — auto-converts to string
int age = 25;
System.out.println("Age: " + age);  // Age: 25

// concat() method
String hello = "Hello";
String world = hello.concat(", ").concat("World!");
System.out.println(world);  // Hello, World!

// String.format() — similar to printf
String formatted = String.format("Name: %s, Age: %d", firstName, age);
System.out.println(formatted);  // Name: John, Age: 25
⚠

Caution: Using + to concatenate strings inside a loop is inefficient because each concatenation creates a new String object. For repeated concatenation, use StringBuilder instead — it can be orders of magnitude faster.

StringBuilder

Unlike String, StringBuilder is mutable — you can modify its content without creating new objects. This makes it much more efficient for building strings dynamically, especially in loops. StringBuilder provides methods like append(), insert(), delete(), and reverse().

java 
// Creating a StringBuilder
StringBuilder sb = new StringBuilder("Hello");

// append() — add to the end
sb.append(", ");
sb.append("World!");
System.out.println(sb);  // Hello, World!

// insert() — add at specific position
sb.insert(5, " Java");
System.out.println(sb);  // Hello Java, World!

// reverse() — reverse the entire string
StringBuilder rev = new StringBuilder("Java");
rev.reverse();
System.out.println(rev);  // avaJ

// Efficient string building in a loop
StringBuilder csv = new StringBuilder();
String[] fruits = {"Apple", "Banana", "Cherry"};
for (int i = 0; i < fruits.length; i++) {
    csv.append(fruits[i]);
    if (i < fruits.length - 1) {
        csv.append(", ");
    }
}
System.out.println(csv.toString());  // Apple, Banana, Cherry

String Comparison

Comparing strings in Java requires special attention. The == operator compares references (whether two variables point to the same object in memory), not the actual content. To compare the content of strings, you must use the .equals() method. This is one of the most common mistakes Java beginners make.

java 
String a = "Hello";
String b = "Hello";
String c = new String("Hello");

// == compares references (memory addresses)
System.out.println(a == b);        // true  (same pool object)
System.out.println(a == c);        // false (different objects!)

// .equals() compares content — ALWAYS use this!
System.out.println(a.equals(b));   // true
System.out.println(a.equals(c));   // true

// .equalsIgnoreCase() — case-insensitive comparison
String upper = "HELLO";
System.out.println(a.equalsIgnoreCase(upper));  // true

// .compareTo() — lexicographic comparison
System.out.println("Apple".compareTo("Banana"));  // negative (A < B)
System.out.println("Banana".compareTo("Apple"));  // positive (B > A)
System.out.println("Hello".compareTo("Hello"));  // 0 (equal)
📝

Remember: Always use .equals() to compare string content in Java. Using == will work sometimes (when strings come from the pool) but fail in other cases — this creates hard-to-find bugs. Make .equals() your default habit!
