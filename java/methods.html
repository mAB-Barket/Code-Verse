Methods in Java

Methods are reusable blocks of code that perform a specific task. They help you organize your program, avoid repetition, and make your code easier to read, test, and maintain. In Java, every method belongs to a class and is defined with an access modifier, a return type, a name, and an optional list of parameters.

Defining Methods

A method definition in Java follows this general syntax: accessModifier returnType methodName(parameters) { body }. The access modifier controls visibility (public, private, protected), the return type specifies what the method gives back, and the parameters define the inputs the method accepts.

java 
// A simple method that adds two integers
public static int add(int a, int b) {
    return a + b;
}

// Calling the method
int sum = add(5, 3);
System.out.println("Sum: " + sum);  // Sum: 8

// A method that returns nothing
public static void greet(String name) {
    System.out.println("Hello, " + name + "!");
}

greet("Alice");  // Hello, Alice!

Method Parameters

In Java, all arguments are passed by value. For primitive types (int, double, etc.), a copy of the value is passed, so changes inside the method do not affect the original variable. For objects, a copy of the reference is passed—the method can modify the object's state, but it cannot reassign the original reference.

java 
// Primitive — passed by value (copy)
public static void tryChange(int x) {
    x = 100;  // does NOT affect the caller
}

int value = 5;
tryChange(value);
System.out.println(value);  // 5 (unchanged)

// Object reference — copy of reference is passed
public static void addItem(ArrayList<String> list) {
    list.add("new item");  // modifies the original object
}

ArrayList<String> items = new ArrayList<>();
addItem(items);
System.out.println(items);  // [new item]

Return Types

Every method must declare a return type. Use void when the method performs an action but doesn't return a value. Otherwise, specify the type (int, String, boolean, double, arrays, objects, etc.) and include a return statement.

java 
// Returns an int
public static int square(int n) {
    return n * n;
}

// Returns a boolean
public static boolean isEven(int n) {
    return n % 2 == 0;
}

// Returns a String
public static String getGreeting(String name) {
    return "Welcome, " + name + "!";
}

// void — no return value
public static void logMessage(String msg) {
    System.out.println("[LOG] " + msg);
}

Method Overloading

Method overloading allows you to define multiple methods with the same name but different parameter lists (different number, type, or order of parameters). The compiler determines which version to call based on the arguments provided. Overloading makes your API more flexible and intuitive.

java 
// Overloaded add methods
public static int add(int a, int b) {
    return a + b;
}

public static double add(double a, double b) {
    return a + b;
}

public static int add(int a, int b, int c) {
    return a + b + c;
}

// The compiler picks the right version
System.out.println(add(2, 3));           // 5       (int version)
System.out.println(add(2.5, 3.5));       // 6.0     (double version)
System.out.println(add(1, 2, 3));       // 6       (three-arg version)
💡

Note: Overloading is resolved at compile time based on the method signature. Changing only the return type is not valid overloading—the parameter list must differ.

Static vs Instance Methods

A static method belongs to the class itself and is called using the class name. It does not require an object instance. An instance method belongs to a specific object and can access instance fields and other instance methods through this.

java 
public class Calculator {

    // Instance field
    private int memory = 0;

    // Static method — called via class name
    public static int multiply(int a, int b) {
        return a * b;
    }

    // Instance method — called on an object
    public void addToMemory(int value) {
        this.memory += value;
    }

    public int getMemory() {
        return this.memory;
    }
}

// Using static method — no object needed
int result = Calculator.multiply(4, 5);
System.out.println(result);  // 20

// Using instance method — requires an object
Calculator calc = new Calculator();
calc.addToMemory(10);
calc.addToMemory(25);
System.out.println(calc.getMemory());  // 35
