Variables & Data Types

Declaring Variables

Java is a statically typed language, which means you must declare a variable's data type before you can use it. The compiler checks types at compile time, catching many errors before your program even runs. This makes Java code more predictable and easier to debug compared to dynamically typed languages.

To declare a variable in Java, you specify the type followed by the variable name, and optionally assign a value. Variable names must start with a letter, underscore, or dollar sign, and can contain letters, digits, underscores, and dollar signs. Java convention uses camelCase for variable names — starting with a lowercase letter and capitalizing each subsequent word.

Every variable in Java must be declared with a specific type, and once declared, that type cannot change. This strict typing system helps prevent subtle bugs and makes your code self-documenting — you always know what kind of data a variable holds.

java 
// Declaring and initializing variables
int age = 25;                  // integer
double price = 19.99;           // decimal number
char grade = 'A';               // single character
boolean isActive = true;        // true or false
String name = "Alice";           // text (reference type)

// Declare first, assign later
int score;
score = 100;

// Multiple variables of the same type
int x = 1, y = 2, z = 3;

// Print variables
System.out.println("Name: " + name);       // Name: Alice
System.out.println("Age: " + age);         // Age: 25
System.out.println("Price: " + price);     // Price: 19.99
System.out.println("Active: " + isActive); // Active: true

Primitive Data Types

Java has 8 primitive data types that serve as the building blocks for all data manipulation. These types are built into the language and are not objects — they hold their values directly in memory, making them fast and efficient.

// Primitive data type examples
byte smallNum = 127;              // max value for byte
short mediumNum = 32000;          // fits in short
int count = 1000000;              // most common integer type
long bigNum = 9999999999L;        // note the 'L' suffix

float pi = 3.14f;                // note the 'f' suffix
double precise = 3.141592653589;  // default decimal type

char letter = 'A';                // single character in single quotes
char unicode = '\u0041';          // Unicode for 'A'

boolean isJavaFun = true;         // true or false only
boolean isFishTasty = false;
💡

Tip: Use int for most integer values and double for decimals. Only use byte, short, long, or float when you have a specific reason (memory optimization, API requirements, etc.).

Type Casting

Type casting is the process of converting a value from one data type to another. Java supports two types of casting:

  • Widening casting (automatic) — Converting a smaller type to a larger type. This happens automatically because there is no risk of data loss: byte → short → int → long → float → double
  • Narrowing casting (manual) — Converting a larger type to a smaller type. This must be done explicitly using parentheses because data may be lost in the conversion.
java 
// Widening casting (automatic) — small to large
int x = 10;
double y = x;              // int automatically converts to double
System.out.println(y);     // 10.0

byte b = 42;
int i = b;                 // byte automatically converts to int
long l = i;                // int automatically converts to long

// Narrowing casting (manual) — large to small
double a = 9.78;
int c = (int) a;           // manual cast — truncates decimal
System.out.println(c);     // 9 (decimal part is lost!)

int bigValue = 130;
byte smallValue = (byte) bigValue;  // may overflow!
System.out.println(smallValue);     // -126 (overflow)

// String to number conversion
String numStr = "42";
int num = Integer.parseInt(numStr);         // 42
double dbl = Double.parseDouble("3.14");   // 3.14
⚠

Caution: Narrowing casting can lead to data loss or overflow. When you cast a double to an int, the decimal part is truncated (not rounded). When you cast a value too large for the target type, it wraps around due to overflow.

Constants

In Java, you declare constants using the final keyword. Once a final variable is assigned a value, it cannot be reassigned. By convention, constant names are written in UPPER_SNAKE_CASE to clearly distinguish them from regular variables.

java 
// Declaring constants with the final keyword
final double PI = 3.14159;
final int MAX_SPEED = 300000;          // km/s (speed of light)
final double GRAVITY = 9.81;           // m/s²
final String APP_NAME = "CodeVerse";

// Using constants
double radius = 5.0;
double area = PI * radius * radius;
System.out.println("Area: " + area);   // Area: 78.53975

// PI = 3.0;  // ERROR! Cannot reassign a final variable
📝

Note: The final keyword in Java is similar to const in C++ and JavaScript. For class-level constants that are shared across all instances, you typically combine static and final: public static final double PI = 3.14159;
