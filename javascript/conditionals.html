Conditionals

If, Else If, and Else

Conditional statements are fundamental to every programming language — they let your program make decisions and execute different code paths based on conditions. In JavaScript, the if statement evaluates a condition inside parentheses, and if it's truthy, the code block in curly braces runs. You can chain additional conditions with else if and provide a fallback with else.

JavaScript evaluates conditions using truthy and falsy values. There are exactly six falsy values: false, 0, "" (empty string), null, undefined, and NaN. Every other value is truthy — including empty arrays [] and empty objects {}. Understanding this is crucial for writing clean conditional logic.

You can combine conditions using logical operators: && (AND), || (OR), and ! (NOT). JavaScript uses short-circuit evaluation — with &&, if the first operand is falsy, the second is never evaluated. With ||, if the first operand is truthy, the second is skipped. This behavior is often used for default values and guard clauses.

javascript 
// Basic if / else if / else
const score = 85;

if (score >= 90) {
    console.log("Grade: A");
} else if (score >= 80) {
    console.log("Grade: B");        // ← This runs
} else if (score >= 70) {
    console.log("Grade: C");
} else {
    console.log("Grade: F");
}

// Logical operators
const age = 25;
const hasLicense = true;

if (age >= 18 && hasLicense) {
    console.log("You can drive!");
}

if (age < 13 || age > 65) {
    console.log("Discounted ticket");
}

// Truthy and falsy values
const username = "";
if (!username) {
    console.log("Please enter a username");  // Runs — "" is falsy
}

// Short-circuit for defaults
const name = username || "Guest";  // "Guest" (fallback for falsy)
const name2 = username ?? "Guest"; // "" (nullish coalescing — only null/undefined)

Switch Statements

When you need to compare a single value against many possible cases, a switch statement is often cleaner than a chain of else if blocks. The switch evaluates an expression and compares it with each case using strict equality (===). Don't forget the break statement — without it, execution "falls through" to the next case.

javascript 
const day = "Monday";

switch (day) {
    case "Monday":
    case "Tuesday":
    case "Wednesday":
    case "Thursday":
    case "Friday":
        console.log("Weekday — time to work!");
        break;
    case "Saturday":
    case "Sunday":
        console.log("Weekend — time to relax!");
        break;
    default:
        console.log("Invalid day");
}

// Switch with different actions per case
const statusCode = 404;

switch (statusCode) {
    case 200:
        console.log("OK — Request succeeded");
        break;
    case 301:
        console.log("Redirected");
        break;
    case 404:
        console.log("Not Found");       // ← This runs
        break;
    case 500:
        console.log("Server Error");
        break;
    default:
        console.log("Unknown status");
}

Ternary Operator & Modern Patterns

The ternary operator (condition ? valueIfTrue : valueIfFalse) is a concise alternative to simple if/else statements. It's an expression, meaning it returns a value — perfect for inline assignments and template literals. For complex logic, stick with if/else; for simple choices, the ternary is elegant and idiomatic.

javascript 
// Ternary operator — condition ? ifTrue : ifFalse
const age = 20;
const status = age >= 18 ? "Adult" : "Minor";
console.log(status);  // "Adult"

// Ternary in template literals
console.log(`You are ${age >= 18 ? "an adult" : "a minor"}`);

// Nullish coalescing (??) — only null/undefined
const userInput = 0;
const value1 = userInput || 10;    // 10 (0 is falsy!)
const value2 = userInput ?? 10;    // 0 (only null/undefined trigger ??)

// Optional chaining (?.) — safe property access
const user = { profile: { name: "Alice" } };
console.log(user.profile?.name);       // "Alice"
console.log(user.settings?.theme);    // undefined (no error!)
// Without ?.: user.settings.theme would throw TypeError
💡

Modern JS Tip: Use the nullish coalescing operator ?? instead of || for default values when 0, "", or false are valid inputs. And use optional chaining ?. to safely access nested properties without checking each level manually.
