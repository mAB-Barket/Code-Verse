Loops

The for Loop

Loops are essential programming constructs that let you execute a block of code repeatedly. The classic for loop is the most common loop in JavaScript, consisting of three parts: an initializer (runs once before the loop starts), a condition (checked before each iteration), and an update (runs after each iteration). When the condition becomes false, the loop stops.

JavaScript offers several types of loops, and choosing the right one matters. The traditional for loop is ideal when you know how many iterations you need or need access to the index. The while loop is best when you don't know how many iterations are needed and the loop depends on a condition being met. Modern loops like for...of and for...in provide cleaner syntax for iterating over arrays and objects respectively.

Two keywords control loop flow: break exits the loop entirely, and continue skips the current iteration and moves to the next one. Use them judiciously — excessive use of break and continue can make loops harder to follow.

javascript 
// Classic for loop
for (let i = 0; i < 5; i++) {
    console.log(`Iteration ${i}`);  // 0, 1, 2, 3, 4
}

// Looping through an array by index
const colors = ["red", "green", "blue"];
for (let i = 0; i < colors.length; i++) {
    console.log(`${i}: ${colors[i]}`);
}

// Counting backwards
for (let i = 10; i >= 0; i--) {
    console.log(i);  // 10, 9, 8, ... 1, 0
}

// break — exit the loop early
for (let i = 0; i < 100; i++) {
    if (i === 5) break;       // Stops at 5
    console.log(i);
}

// continue — skip current iteration
for (let i = 0; i < 10; i++) {
    if (i % 2 !== 0) continue;  // Skip odd numbers
    console.log(i);              // 0, 2, 4, 6, 8
}

while and do...while Loops

The while loop keeps running as long as its condition is truthy. It checks the condition before each iteration, so if the condition is false from the start, the body never executes. The do...while loop is similar but checks the condition after each iteration, guaranteeing the body runs at least once.

javascript 
// while loop — checks condition BEFORE each iteration
let count = 0;
while (count < 5) {
    console.log(`Count: ${count}`);
    count++;
}

// do...while — runs at least once, checks AFTER
let input;
do {
    input = prompt("Enter 'quit' to exit:");
    console.log(`You typed: ${input}`);
} while (input !== "quit");

// Practical example: find the first power of 2 over 1000
let power = 1;
while (power <= 1000) {
    power *= 2;
}
console.log(power);  // 1024
⚠

Warning: Always ensure your loop has an exit condition! A while (true) loop without a break will run forever and freeze your browser tab. If your page becomes unresponsive, it's often caused by an infinite loop.

for...of and for...in

ES6 introduced the for...of loop, which iterates over the values of any iterable object (arrays, strings, Maps, Sets). It's the cleanest way to loop over arrays when you don't need the index. The older for...in loop iterates over the keys (property names) of an object and is best used with plain objects, not arrays.

javascript 
// for...of — iterate over VALUES (arrays, strings, etc.)
const fruits = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
    console.log(fruit);  // "apple", "banana", "cherry"
}

// for...of with strings (iterates characters)
for (const char of "Hello") {
    console.log(char);   // "H", "e", "l", "l", "o"
}

// for...of with entries() — get index AND value
for (const [index, fruit] of fruits.entries()) {
    console.log(`${index}: ${fruit}`);
}

// for...in — iterate over KEYS (best for objects)
const person = {
    name: "Alice",
    age: 28,
    city: "New York"
};

for (const key in person) {
    console.log(`${key}: ${person[key]}`);
}
// "name: Alice", "age: 28", "city: New York"

// Alternative: Object.entries() with for...of
for (const [key, value] of Object.entries(person)) {
    console.log(`${key} = ${value}`);
}
💡

Quick Guide: Use for...of for arrays and iterables (values). Use for...in for objects (keys). Use classic for when you need the index or complex iteration logic. Use while when you don't know how many iterations needed. And consider array methods like forEach(), map(), and filter() as functional alternatives.
