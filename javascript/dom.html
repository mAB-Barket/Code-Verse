DOM Manipulation

What is the DOM?

The Document Object Model (DOM) is a programming interface that represents an HTML document as a tree of objects. Every HTML element becomes a node in this tree, and JavaScript can access, modify, add, or remove any of these nodes. The DOM is what makes web pages interactive — without it, JavaScript couldn't change what you see on the screen.

When a browser loads an HTML page, it parses the markup and constructs the DOM tree. The root of this tree is the document object, which represents the entire page. From there, you can traverse to any element using methods like querySelector(), getElementById(), or getElementsByClassName(). The DOM is not part of JavaScript itself — it's a Web API provided by the browser that JavaScript can interact with.

Modern DOM manipulation revolves around a few core operations: selecting elements, reading and modifying their content and attributes, creating and removing elements, and listening for events. Understanding these four operations gives you the power to build any interactive web feature from scratch.

javascript 
// === SELECTING ELEMENTS ===

// querySelector — returns the FIRST matching element
const heading = document.querySelector("h1");
const btn = document.querySelector(".submit-btn");
const nav = document.querySelector("#mainNav");

// querySelectorAll — returns ALL matching elements (NodeList)
const allLinks = document.querySelectorAll("a");
const cards = document.querySelectorAll(".card");

// Older methods (still valid but less flexible)
const byId = document.getElementById("mainNav");
const byClass = document.getElementsByClassName("card");
const byTag = document.getElementsByTagName("p");

// Loop over NodeList results
allLinks.forEach(link => {
    console.log(link.href);
});

Modifying Content & Attributes

Once you've selected an element, you can read or change its content using properties like textContent (plain text), innerHTML (HTML markup), and innerText (rendered text). You can also modify any HTML attribute, toggle CSS classes, and dynamically change inline styles — all without reloading the page.

javascript 
// === MODIFYING CONTENT ===
const title = document.querySelector("h1");

// Read content
console.log(title.textContent);    // "Original Title"

// Change content
title.textContent = "New Title";
title.innerHTML = "<em>Italic</em> Title";  // Parses HTML

// === MODIFYING STYLES ===
title.style.color = "blue";
title.style.fontSize = "2rem";       // camelCase, not kebab-case
title.style.backgroundColor = "#f0f0f0";

// === MODIFYING CLASSES (recommended over inline styles) ===
title.classList.add("highlight");
title.classList.remove("old-class");
title.classList.toggle("active");     // Add if missing, remove if present
title.classList.contains("highlight"); // true

// === MODIFYING ATTRIBUTES ===
const link = document.querySelector("a");
link.setAttribute("href", "https://example.com");
link.setAttribute("target", "_blank");
console.log(link.getAttribute("href"));
link.removeAttribute("target");
💡

Best Practice: Prefer classList.add() / classList.toggle() over directly setting style properties. This keeps styling in CSS where it belongs and makes your JavaScript cleaner. Only use inline styles for truly dynamic values (like calculated positions or sizes).

Creating Elements & Handling Events

JavaScript can dynamically create new HTML elements, insert them into the page, and remove existing ones. Combined with event listeners, you can build fully interactive interfaces. Events are actions or occurrences — clicks, key presses, form submissions, scrolls, hovers — that your code can respond to. The addEventListener() method is the modern, recommended way to handle events.

javascript 
// === CREATING & INSERTING ELEMENTS ===
const newDiv = document.createElement("div");
newDiv.textContent = "I was created by JavaScript!";
newDiv.className = "dynamic-card";

// Append to the end of body
document.body.appendChild(newDiv);

// Insert before a specific element
const container = document.querySelector(".container");
const firstChild = container.firstElementChild;
container.insertBefore(newDiv, firstChild);

// Remove an element
const oldElement = document.querySelector(".remove-me");
oldElement?.remove();  // Safe removal with optional chaining

// === EVENT HANDLING ===
const button = document.querySelector("#myButton");

// addEventListener — the modern, recommended approach
button.addEventListener("click", (event) => {
    console.log("Button clicked!");
    console.log("Target:", event.target);
    event.target.textContent = "Clicked!";
});

// Common events: click, input, submit, keydown, mouseover, scroll
const input = document.querySelector("#searchInput");
input?.addEventListener("input", (e) => {
    console.log(`User typed: ${e.target.value}`);
});

// Event delegation — handle events on parent
document.querySelector(".card-list")?.addEventListener("click", (e) => {
    if (e.target.matches(".card")) {
        console.log("Card clicked:", e.target.textContent);
    }
});
⚠

Security Warning: Be cautious with innerHTML — it parses HTML, which means it can execute scripts if you insert user-provided content. This is a Cross-Site Scripting (XSS) vulnerability. For user-generated text, always use textContent instead, which treats everything as plain text.

📝

Note: Place your <script> tag at the bottom of the <body>, or use the defer attribute: <script src="app.js" defer></script>. This ensures the DOM is fully loaded before your JavaScript runs. Otherwise, querySelector() will return null because the elements don't exist yet.
