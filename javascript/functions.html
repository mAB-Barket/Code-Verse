Functions & Closures

Function Declarations and Expressions

Functions are the building blocks of JavaScript programs. They are reusable blocks of code that perform a specific task. JavaScript offers several ways to define functions: function declarations (using the function keyword), function expressions (assigning a function to a variable), and arrow functions (a concise ES6 syntax). Each has slightly different behavior regarding hoisting, this binding, and syntax.

Function declarations are hoisted — meaning they are moved to the top of their scope during compilation, so you can call them before they appear in your code. Function expressions and arrow functions are not hoisted, so they must be defined before they are called. Functions can accept parameters (inputs), perform operations, and return a value. If no return statement is provided, the function returns undefined.

JavaScript functions are first-class citizens — they can be assigned to variables, passed as arguments to other functions, returned from functions, and stored in data structures. This is what enables powerful patterns like callbacks, higher-order functions, and functional programming in JavaScript.

javascript 
// Function declaration (hoisted — can call before definition)
function greet(name) {
    return `Hello, ${name}!`;
}
console.log(greet("Alice"));  // "Hello, Alice!"

// Function expression (NOT hoisted)
const add = function(a, b) {
    return a + b;
};
console.log(add(3, 5));  // 8

// Default parameters (ES6)
function createUser(name, role = "viewer") {
    return { name, role };
}
console.log(createUser("Bob"));           // { name: "Bob", role: "viewer" }
console.log(createUser("Eve", "admin"));  // { name: "Eve", role: "admin" }

// Rest parameters — collect remaining arguments
function sum(...numbers) {
    return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4));  // 10

Arrow Functions

Arrow functions, introduced in ES6, provide a shorter syntax for writing functions. They are especially useful for short, one-line functions and as callbacks passed to array methods. Arrow functions have an important difference from regular functions: they do not have their own this binding — they inherit this from the surrounding scope. This makes them ideal for callbacks but unsuitable for object methods.

javascript 
// Arrow function — full syntax
const multiply = (a, b) => {
    return a * b;
};

// Arrow function — concise body (implicit return)
const double = n => n * 2;
const isEven = n => n % 2 === 0;

console.log(double(5));    // 10
console.log(isEven(4));    // true

// Perfect for array method callbacks
const numbers = [1, 2, 3, 4, 5];
const squared = numbers.map(n => n ** 2);
const evens = numbers.filter(n => n % 2 === 0);
console.log(squared);  // [1, 4, 9, 16, 25]
console.log(evens);    // [2, 4]

// Return an object (wrap in parentheses)
const makeUser = (name, age) => ({ name, age });
console.log(makeUser("Alice", 25));  // { name: "Alice", age: 25 }

// No arguments? Use empty parentheses
const getTimestamp = () => Date.now();

Closures

A closure is one of the most powerful concepts in JavaScript. It occurs when a function "remembers" the variables from the scope in which it was created, even after that outer function has finished executing. Every time you create a function inside another function, you create a closure. Closures enable data privacy, factory functions, and stateful behavior without global variables.

javascript 
// Closure — inner function remembers outer variables
function createCounter() {
    let count = 0;  // Private variable — not accessible from outside

    return {
        increment: () => ++count,
        decrement: () => --count,
        getCount: () => count
    };
}

const counter = createCounter();
console.log(counter.increment());  // 1
console.log(counter.increment());  // 2
console.log(counter.decrement());  // 1
console.log(counter.getCount());   // 1
// counter.count is undefined — count is truly private!

// Practical closure: function factory
function createMultiplier(factor) {
    return (number) => number * factor;
}

const triple = createMultiplier(3);
const tenTimes = createMultiplier(10);
console.log(triple(5));     // 15
console.log(tenTimes(5));   // 50

// Callback functions — passing functions as arguments
function doOperation(a, b, operation) {
    return operation(a, b);
}
console.log(doOperation(10, 3, (a, b) => a + b));  // 13
console.log(doOperation(10, 3, (a, b) => a * b));  // 30
📝

Key Insight: Closures are not just a theoretical concept — they're everywhere in JavaScript. Event handlers, callbacks, setTimeout, React hooks, module patterns, and even array methods all rely on closures. If you've used a callback function, you've already used closures!
