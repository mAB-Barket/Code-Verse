Variables & Data Types

Declaring Variables: var, let, and const

JavaScript provides three keywords for declaring variables: var, let, and const. Understanding the differences between them is one of the most important fundamentals of modern JavaScript. In early JavaScript, var was the only option, but ES6 (2015) introduced let and const to address its shortcomings.

let declares a block-scoped variable that can be reassigned. const declares a block-scoped variable that cannot be reassigned after initialization — it must be given a value when declared. var is function-scoped (not block-scoped), gets hoisted to the top of its scope, and can lead to subtle bugs. Modern best practice is to always use const by default, and only use let when you need to reassign the variable. Avoid var entirely in new code.

Block scope means the variable only exists within the nearest pair of curly braces {}. This prevents variables from leaking into outer scopes and makes your code more predictable. The old var keyword ignores block scope entirely, which is why it's considered problematic.

javascript 
// const — cannot be reassigned (use by default)
const PI = 3.14159;
const APP_NAME = "CodeVerse";
// PI = 3.14;  // ❌ TypeError: Assignment to constant variable

// let — can be reassigned (use when value will change)
let score = 0;
score = 10;           // ✅ Reassignment is allowed
score += 5;           // ✅ score is now 15

// var — function-scoped, avoid in modern code
var oldStyle = "legacy";  // ❌ Avoid — use let or const instead

// Block scope difference
if (true) {
    let blockScoped = "inside";
    var notBlockScoped = "leaks out";
}
// console.log(blockScoped);     // ❌ ReferenceError
console.log(notBlockScoped);  // ✅ "leaks out" — var ignores block scope!
💡

Best Practice: Use const for everything by default. Only switch to let when you know the value needs to change (counters, accumulators, state). Never use var in modern JavaScript. This makes your code more predictable and easier to debug.

JavaScript Data Types

JavaScript is a dynamically typed language, meaning variables don't have fixed types — the type is determined by the value assigned. JavaScript has 8 data types, divided into primitive types (immutable) and reference types (mutable objects).

javascript 
// === Primitive Data Types ===

// Number — integers and decimals (64-bit floating point)
const age = 25;
const price = 19.99;
const negative = -42;
const infinity = Infinity;
const notANumber = NaN;           // "Not a Number"

// String — text data
const greeting = "Hello, World!";
const single = 'Single quotes work too';
const template = `Backticks allow ${age} interpolation`;

// Boolean — true or false
const isActive = true;
const isDeleted = false;

// Undefined — declared but not assigned
let uninitialised;
console.log(uninitialised);       // undefined

// Null — intentional absence of value
const emptyValue = null;

// Symbol — unique identifier (ES6)
const id = Symbol("id");

// BigInt — arbitrary-precision integers (ES2020)
const huge = 9007199254740991n;

// === typeof operator — check a value's type ===
console.log(typeof age);          // "number"
console.log(typeof greeting);     // "string"
console.log(typeof isActive);     // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" (known JS quirk!)

Type Conversion & Coercion

JavaScript performs type coercion automatically in many situations — silently converting one type to another. This can lead to surprising results if you're not careful. You can also perform explicit conversion (type casting) using built-in functions like Number(), String(), and Boolean().

javascript 
// Explicit type conversion
const str = "42";
const num = Number(str);        // 42 (number)
const back = String(num);       // "42" (string)
const bool = Boolean(1);        // true
const parsed = parseInt("100px"); // 100

// Implicit coercion (automatic — be careful!)
console.log("5" + 3);            // "53" (string concatenation!)
console.log("5" - 3);            // 2 (numeric subtraction)
console.log("5" * 2);            // 10 (numeric multiplication)
console.log(true + 1);           // 2 (true becomes 1)

// Strict vs. loose equality
console.log(5 == "5");          // true (loose — coerces types)
console.log(5 === "5");         // false (strict — no coercion) ✅
console.log(null == undefined);  // true (loose)
console.log(null === undefined); // false (strict)
⚠

Important: Always use strict equality === and strict inequality !== instead of == and !=. Loose equality performs type coercion which can cause hard-to-find bugs. For example, 0 == "" is true, and false == "" is true — almost certainly not what you intend.
