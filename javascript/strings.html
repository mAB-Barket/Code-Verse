Strings & Template Literals

Creating Strings

Strings in JavaScript represent textual data and are one of the most frequently used data types. You can create strings using single quotes ('...'), double quotes ("..."), or backticks (`...`). Single and double quotes are functionally identical, but backticks unlock template literals — a powerful ES6 feature that allows embedded expressions and multi-line strings.

Strings in JavaScript are immutable — once created, a string's characters cannot be changed in place. Methods like toUpperCase() or replace() always return a new string rather than modifying the original. Strings are also zero-indexed, meaning the first character has index 0, the second has index 1, and so on.

JavaScript strings are sequences of UTF-16 code units, which means they fully support Unicode characters, including emoji. You can access individual characters using bracket notation (str[0]) or the charAt() method, and find the length of any string using the .length property.

javascript 
// Three ways to create strings
const single = 'Hello, World!';
const double = "Hello, World!";
const backtick = `Hello, World!`;

// String properties and access
const language = "JavaScript";
console.log(language.length);     // 10
console.log(language[0]);          // "J"
console.log(language[4]);          // "S"
console.log(language.charAt(0));   // "J"

// Escape characters
const withQuotes = "She said \"hello\"";
const withNewline = "Line 1\nLine 2";
const withTab = "Col1\tCol2";

Template Literals (Backtick Strings)

Template literals, introduced in ES6, are one of the most useful additions to JavaScript. They use backticks (`) instead of quotes and provide two killer features: string interpolation with ${expression} syntax, and multi-line strings without needing escape characters. Template literals make building dynamic strings clean and readable.

javascript 
// String interpolation with template literals
const name = "Alice";
const age = 28;

// Old way (concatenation) — messy
const old = "My name is " + name + " and I am " + age + " years old.";

// New way (template literals) — clean! ✅
const modern = `My name is ${name} and I am ${age} years old.`;

// Expressions inside ${} are fully evaluated
console.log(`Next year I'll be ${age + 1}`);       // "Next year I'll be 29"
console.log(`Is adult: ${age >= 18}`);              // "Is adult: true"
console.log(`Uppercase: ${name.toUpperCase()}`);   // "Uppercase: ALICE"

// Multi-line strings (no \n needed)
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>Age: ${age}</p>
  </div>
`;
console.log(html);
💡

Pro Tip: Template literals are especially useful when building HTML strings, constructing URLs, or creating log messages. Anything inside ${} is evaluated as a JavaScript expression — you can call functions, use ternary operators, and even nest template literals inside each other.

Essential String Methods

JavaScript strings come with a rich set of built-in methods for searching, extracting, transforming, and manipulating text. Since strings are immutable, all these methods return new strings without modifying the original. Here are the most commonly used methods every JavaScript developer should know:

javascript 
const str = "  Hello, JavaScript World!  ";

// Searching
str.includes("JavaScript");     // true
str.startsWith("  Hello");     // true
str.endsWith("!  ");            // true
str.indexOf("JavaScript");      // 9

// Transforming
str.toUpperCase();               // "  HELLO, JAVASCRIPT WORLD!  "
str.toLowerCase();               // "  hello, javascript world!  "
str.trim();                      // "Hello, JavaScript World!"
str.trimStart();                 // "Hello, JavaScript World!  "
str.trimEnd();                   // "  Hello, JavaScript World!"

// Extracting
str.slice(2, 7);                // "Hello"
str.slice(-8);                  // "World!  "
str.substring(2, 7);            // "Hello"

// Replacing
str.replace("JavaScript", "JS");  // "  Hello, JS World!  "
str.replaceAll(" ", "-");          // "--Hello,-JavaScript-World!--"

// Splitting and joining
const words = "apple,banana,cherry".split(",");
console.log(words);              // ["apple", "banana", "cherry"]
console.log(words.join(" | "));  // "apple | banana | cherry"

// Repeating and padding
"ha".repeat(3);                  // "hahaha"
"5".padStart(3, "0");            // "005"
"5".padEnd(3, "0");              // "500"
📝

Note: Remember that strings are immutable in JavaScript. Every string method returns a new string — the original is never modified. If you want to keep the result, you must assign it to a variable: const trimmed = str.trim();
