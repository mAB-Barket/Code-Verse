11. Switch Statements

📚 Lesson 11 of 55 ⏱ 15 min read 🌱 Beginner

What Is a Switch Statement?

In the previous lesson you learned how to use if-else if-else chains to test multiple conditions. While that approach works, it can become verbose and hard to read when you are comparing a single variable against many possible values. The switch statement provides a cleaner alternative for exactly this scenario.

A switch evaluates an expression once and then jumps directly to the matching case label, making the code easier to read and sometimes faster than a long chain of if-else if comparisons.

In this lesson you will learn:

  • The syntax of switch, case, break, and default
  • How fall-through behavior works (and how to avoid it)
  • The importance of the break statement
  • How to use default as a catch-all
  • Multiple cases sharing the same code
  • Switch with char, int, and enum types
  • Nested switch statements
  • When to use switch vs if-else
  • Common mistakes and pitfalls
  • C++17 switch with initializer
  • Practical examples: calculator, menu system, day of the week

Key Point

Use a switch when you need to compare one variable against a series of constant values. For range checks (e.g., x > 10) or complex conditions, stick with if-else.

Switch Syntax

The general form of a switch statement is:

switch (expression) {
    case value1:
        // Code for value1
        break;
    case value2:
        // Code for value2
        break;
    case value3:
        // Code for value3
        break;
    default:
        // Code if no case matches
        break;
}

Here is how it works step by step:

  1. The expression inside the parentheses is evaluated once.
  2. The result is compared against each case value from top to bottom.
  3. When a match is found, execution jumps to that case label.
  4. The break statement exits the switch block.
  5. If no case matches, the default block runs (if present).

Tip

The expression must evaluate to an integral or enumeration type — int, char, short, long, enum, or bool. You cannot use float, double, or std::string in a switch.

Your First Switch Example

Let’s start with a classic: printing the day of the week based on a number.

#include <iostream>
using namespace std;

int main() {
    int day = 3;

    switch (day) {
        case 1:
            cout << "Monday" << endl;
            break;
        case 2:
            cout << "Tuesday" << endl;
            break;
        case 3:
            cout << "Wednesday" << endl;
            break;
        case 4:
            cout << "Thursday" << endl;
            break;
        case 5:
            cout << "Friday" << endl;
            break;
        case 6:
            cout << "Saturday" << endl;
            break;
        case 7:
            cout << "Sunday" << endl;
            break;
        default:
            cout << "Invalid day number" << endl;
            break;
    }

    return 0;
}

Output:

Wednesday

The value of day (3) matches case 3, so “Wednesday” is printed. The break then exits the switch. If day were 9, none of the cases would match and the default block would run.

The break Statement

The break keyword tells the program to exit the switch block immediately. Without it, execution “falls through” to the next case — even if that case’s value doesn’t match.

What Happens Without break?

#include <iostream>
using namespace std;

int main() {
    int x = 2;

    switch (x) {
        case 1:
            cout << "One" << endl;
        case 2:
            cout << "Two" << endl;
        case 3:
            cout << "Three" << endl;
        case 4:
            cout << "Four" << endl;
        default:
            cout << "Default" << endl;
    }

    return 0;
}

Output:

Two
Three
Four
Default

Execution starts at case 2 and then falls through every subsequent case (and default) because there are no break statements. This is almost always a bug unless you intentionally want fall-through.

Key Point

Always include break at the end of each case unless you have a deliberate reason for fall-through. Forgetting break is one of the most common switch-related bugs.

Fall-Through Behavior

While usually a mistake, fall-through can be used intentionally. A common use is grouping multiple case labels that should execute the same code.

Multiple Cases, Same Code

#include <iostream>
using namespace std;

int main() {
    int day = 6;

    switch (day) {
        case 1:
        case 2:
        case 3:
        case 4:
        case 5:
            cout << "Weekday" << endl;
            break;
        case 6:
        case 7:
            cout << "Weekend" << endl;
            break;
        default:
            cout << "Invalid day" << endl;
            break;
    }

    return 0;
}

Output:

Weekend

Cases 1 through 5 all fall through to the cout << "Weekday" statement. Cases 6 and 7 share the “Weekend” output. This is cleaner than writing five separate if conditions with ||.

Tip

In C++17, you can use the [[fallthrough]] attribute to tell the compiler (and other developers) that the fall-through is intentional, suppressing compiler warnings.

switch (value) {
    case 1:
        doSomething();
        [[fallthrough]];  // Intentional fall-through
    case 2:
        doMore();
        break;
}

The default Case

The default label is optional but highly recommended. It acts as a catch-all for values that don’t match any case.

#include <iostream>
using namespace std;

int main() {
    char grade = 'X';

    switch (grade) {
        case 'A':
            cout << "Excellent!" << endl;
            break;
        case 'B':
            cout << "Good job!" << endl;
            break;
        case 'C':
            cout << "Well done." << endl;
            break;
        case 'D':
            cout << "You passed." << endl;
            break;
        case 'F':
            cout << "Better try again." << endl;
            break;
        default:
            cout << "Invalid grade." << endl;
            break;
    }

    return 0;
}

Output:

Invalid grade.

The default block can appear anywhere inside the switch, but by convention it is placed last. It does not require a break if it is the last block, but including one is good practice for consistency.

Key Point

Always include a default case. It catches unexpected values and makes debugging easier. Even if you think you’ve covered all possibilities, a default provides a safety net.

Switch with char

Characters are stored as integers (ASCII values), so they work perfectly with switch.

#include <iostream>
using namespace std;

int main() {
    char op;
    cout << "Enter an operator (+, -, *, /): ";
    cin >> op;

    double a, b;
    cout << "Enter two numbers: ";
    cin >> a >> b;

    switch (op) {
        case '+':
            cout << a << " + " << b << " = " << (a + b) << endl;
            break;
        case '-':
            cout << a << " - " << b << " = " << (a - b) << endl;
            break;
        case '*':
            cout << a << " * " << b << " = " << (a * b) << endl;
            break;
        case '/':
            if (b != 0)
                cout << a << " / " << b << " = " << (a / b) << endl;
            else
                cout << "Error: Division by zero!" << endl;
            break;
        default:
            cout << "Invalid operator!" << endl;
            break;
    }

    return 0;
}

This simple calculator uses a char in the switch. Each arithmetic operator maps to a case label. Notice how we added a division-by-zero check inside the '/' case.

Switch with enum

Enumerations and switch go hand-in-hand. An enum defines a set of named constants — exactly the kind of values switch is designed for.

#include <iostream>
using namespace std;

enum Color { RED, GREEN, BLUE, YELLOW };

int main() {
    Color c = GREEN;

    switch (c) {
        case RED:
            cout << "Red: #FF0000" << endl;
            break;
        case GREEN:
            cout << "Green: #00FF00" << endl;
            break;
        case BLUE:
            cout << "Blue: #0000FF" << endl;
            break;
        case YELLOW:
            cout << "Yellow: #FFFF00" << endl;
            break;
        default:
            cout << "Unknown color" << endl;
            break;
    }

    return 0;
}

Output:

Green: #00FF00

Tip

When using enum class (scoped enumeration from C++11), you must cast the value or use static_cast for the switch expression. Many compilers will warn you if your switch on an enum doesn’t cover all values.

Nested Switch Statements

You can place a switch inside another switch, although deep nesting is generally discouraged for readability.

#include <iostream>
using namespace std;

int main() {
    int category = 1;
    int item = 2;

    switch (category) {
        case 1:
            cout << "Category: Electronics" << endl;
            switch (item) {
                case 1:
                    cout << "Item: Phone" << endl;
                    break;
                case 2:
                    cout << "Item: Laptop" << endl;
                    break;
                default:
                    cout << "Item: Unknown" << endl;
                    break;
            }
            break;
        case 2:
            cout << "Category: Clothing" << endl;
            break;
        default:
            cout << "Category: Unknown" << endl;
            break;
    }

    return 0;
}

Output:

Category: Electronics
Item: Laptop

Each switch has its own set of case labels. The inner break exits only the inner switch; the outer break exits the outer switch.

Switch vs If-Else Comparison

Both switch and if-else handle multiple conditions, but they have different strengths.

Feature switch if-else
Compares against Constant values only Any expression or range
Supported types int, char, enum Any type (including string, float)
Range checks Not supported Fully supported (>, <, etc.)
Readability Cleaner for many discrete values Better for complex conditions
Performance May use jump tables (faster) Sequential condition checks
Fall-through Yes (requires break) No
Multiple conditions Stack case labels Use || operator

Key Point

Use switch when comparing a single variable against known constant values. Use if-else when conditions involve ranges, multiple variables, or non-integral types like string or double.

Common Mistakes & Pitfalls

1. Forgetting break

The most common mistake. Without break, execution falls through to the next case.

// BUG: Missing break statements
switch (x) {
    case 1:
        cout << "One";    // Falls through!
    case 2:
        cout << "Two";    // This also runs for case 1!
        break;
}

2. Using Non-Constant Case Values

Case labels must be compile-time constants. Variables or function return values are not allowed.

int a = 5;
switch (x) {
    case a:       // ERROR: 'a' is not a constant expression
        break;
    case 5:       // OK: literal constant
        break;
    case 2 + 3:   // OK: constant expression
        break;
}

3. Duplicate Case Values

Each case value must be unique within the same switch. Duplicate values cause a compile error.

switch (x) {
    case 1:
        // ...
        break;
    case 1:      // ERROR: duplicate case value
        break;
}

4. Declaring Variables Inside Cases Without Braces

If you need to declare a variable inside a case, you must enclose it in braces to create a scope.

switch (x) {
    case 1: {
        int y = 10;          // OK: scoped with braces
        cout << y << endl;
        break;
    }
    case 2:
        int z = 20;          // ERROR: jumps over initialization
        break;
}

5. Using float, double, or string

Switch only works with integral and enumeration types. Floating-point and string types are not allowed.

double d = 3.14;
switch (d) {        // ERROR: switch expression must be integral
    case 3.14:
        break;
}

string s = "hi";
switch (s) {        // ERROR: cannot switch on std::string
    case "hi":
        break;
}

Tip

If you need to switch on strings, use an if-else chain or create a mapping from strings to integers (e.g., using std::unordered_map).

C++17: Switch with Initializer

C++17 introduced the ability to include an initializer statement inside the switch, just like if with initializer.

// Syntax: switch (init; expression)
#include <iostream>
using namespace std;

int getValue() { return 2; }

int main() {
    switch (int val = getValue(); val) {
        case 1:
            cout << "Got one" << endl;
            break;
        case 2:
            cout << "Got two" << endl;
            break;
        default:
            cout << "Got something else: " << val << endl;
            break;
    }
    // 'val' is NOT accessible here

    return 0;
}

Output:

Got two

The variable val is declared and initialized in the switch header. Its scope is limited to the switch block, keeping the outer scope clean.

Key Point

Switch with initializer requires the -std=c++17 (or later) compiler flag. The syntax is switch (init; expression) { ... } — note the semicolon separating the initializer from the expression.

constexpr if vs Switch (Brief Note)

C++17 also introduced if constexpr, which is evaluated at compile time. While it may look similar, it serves a very different purpose from switch:

  • switch — runtime branching based on a value.
  • if constexpr — compile-time branching, primarily used in templates to discard code paths that don’t apply.
// if constexpr example (template context)
template <typename T>
void printType(T value) {
    if constexpr (std::is_integral_v<T>) {
        cout << "Integer: " << value << endl;
    } else if constexpr (std::is_floating_point_v<T>) {
        cout << "Float: " << value << endl;
    } else {
        cout << "Other type" << endl;
    }
}

You will explore if constexpr in more detail in the Modern C++ lesson. For now, remember that switch is your go-to for runtime value-based branching.

Practical Examples

Example 1: Simple Calculator

#include <iostream>
using namespace std;

int main() {
    cout << "=== Simple Calculator ===" << endl;
    cout << "1. Add" << endl;
    cout << "2. Subtract" << endl;
    cout << "3. Multiply" << endl;
    cout << "4. Divide" << endl;
    cout << "Choose operation (1-4): ";

    int choice;
    cin >> choice;

    double a, b;
    cout << "Enter two numbers: ";
    cin >> a >> b;

    switch (choice) {
        case 1:
            cout << a << " + " << b << " = " << (a + b) << endl;
            break;
        case 2:
            cout << a << " - " << b << " = " << (a - b) << endl;
            break;
        case 3:
            cout << a << " * " << b << " = " << (a * b) << endl;
            break;
        case 4:
            if (b != 0)
                cout << a << " / " << b << " = " << (a / b) << endl;
            else
                cout << "Error: Division by zero!" << endl;
            break;
        default:
            cout << "Invalid choice!" << endl;
            break;
    }

    return 0;
}

Example 2: Menu System

#include <iostream>
using namespace std;

int main() {
    cout << "===== Main Menu =====" << endl;
    cout << "N - New Game" << endl;
    cout << "L - Load Game" << endl;
    cout << "S - Settings" << endl;
    cout << "Q - Quit" << endl;
    cout << "Enter your choice: ";

    char choice;
    cin >> choice;

    switch (choice) {
        case 'N':
        case 'n':
            cout << "Starting new game..." << endl;
            break;
        case 'L':
        case 'l':
            cout << "Loading saved game..." << endl;
            break;
        case 'S':
        case 's':
            cout << "Opening settings..." << endl;
            break;
        case 'Q':
        case 'q':
            cout << "Goodbye!" << endl;
            break;
        default:
            cout << "Invalid option. Please try again." << endl;
            break;
    }

    return 0;
}

Notice how we use fall-through to handle both uppercase and lowercase letters for each menu option.

Example 3: Month Days Calculator

#include <iostream>
using namespace std;

int main() {
    int month;
    cout << "Enter month number (1-12): ";
    cin >> month;

    switch (month) {
        case 1: case 3: case 5: case 7:
        case 8: case 10: case 12:
            cout << "31 days" << endl;
            break;
        case 4: case 6: case 9: case 11:
            cout << "30 days" << endl;
            break;
        case 2:
            cout << "28 or 29 days" << endl;
            break;
        default:
            cout << "Invalid month!" << endl;
            break;
    }

    return 0;
}

This is a great example of stacking case labels. Months with 31 days share one output, months with 30 days share another, and February is handled separately.

Summary

  • A switch statement compares one expression against multiple constant values.
  • Each case label represents a possible match; default handles everything else.
  • Always include break to prevent unintended fall-through.
  • Intentional fall-through lets multiple cases share the same code. Use [[fallthrough]] in C++17 to document it.
  • Switch works with int, char, enum, and other integral types — not with float, double, or string.
  • Case values must be compile-time constants. Variables are not allowed.
  • Declare variables inside a case only within braces { } to avoid scope issues.
  • C++17 adds switch with initializer: switch (init; expr).
  • Prefer switch for discrete value matching; prefer if-else for ranges and complex conditions.
  • Always include a default case as a safety net.
