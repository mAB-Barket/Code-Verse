14. Break & Continue

📚 Lesson 14 of 55 ⏱ 15 min read 🌱 Beginner

Why Break & Continue?

In the previous lessons you learned for and while loops. Sometimes you need more control over how a loop runs — you may want to exit early when a condition is met, or skip certain iterations without stopping the entire loop.

C++ provides two powerful keywords for this: break and continue. Together, they give you fine-grained control over loop execution.

In this lesson you will learn:

  • The break statement — purpose, syntax, and how it works in loops and switch
  • The continue statement — purpose, syntax, and how it skips iterations
  • Using break and continue in for, while, and do-while loops
  • How break and continue behave in nested loops
  • The difference between break, continue, and return
  • The goto statement and why to avoid it
  • Practical examples: searching arrays, skipping values, input validation, menu with exit, finding primes

Key Point

break exits the loop entirely. continue skips the rest of the current iteration and jumps to the next one. Both only affect the innermost loop they are placed in.

The break Statement

The break statement immediately terminates the loop it is inside. Execution continues with the first statement after the loop.

Syntax

break;

break can be used inside:

  • for loops
  • while loops
  • do-while loops
  • switch statements

break in a for Loop

#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i == 6) {
            break;  // Exit loop when i reaches 6
        }
        cout << i << " ";
    }
    cout << endl;
    return 0;
}

Output:

1 2 3 4 5

When i becomes 6, the break executes and the loop ends immediately. The value 6 is never printed.

break in a while Loop

#include <iostream>
using namespace std;

int main() {
    int i = 1;
    while (true) {         // Infinite loop
        if (i > 5) {
            break;         // Exit when i exceeds 5
        }
        cout << i << " ";
        i++;
    }
    cout << "\nLoop ended." << endl;
    return 0;
}

Output:

1 2 3 4 5
Loop ended.

Tip

Using while(true) with a break inside is a common pattern when you don’t know in advance how many times to loop. Just make sure the break condition is always reachable!

break in a do-while Loop

#include <iostream>
using namespace std;

int main() {
    int n;
    do {
        cout << "Enter a positive number (0 to quit): ";
        cin >> n;
        if (n == 0) break;
        cout << "You entered: " << n << endl;
    } while (true);

    cout << "Goodbye!" << endl;
    return 0;
}

break in switch Statements

You already saw break in the switch lesson. Without break, execution “falls through” to the next case:

#include <iostream>
using namespace std;

int main() {
    int day = 3;
    switch (day) {
        case 1: cout << "Monday"; break;
        case 2: cout << "Tuesday"; break;
        case 3: cout << "Wednesday"; break;  // Matches here
        case 4: cout << "Thursday"; break;
        case 5: cout << "Friday"; break;
        default: cout << "Weekend";
    }
    return 0;
}

Output:

Wednesday

Key Point

In a switch, break prevents fall-through to the next case. In a loop, break exits the loop. If a switch is inside a loop, break inside the switch only exits the switch — not the loop.

The continue Statement

The continue statement skips the rest of the current iteration and jumps to the next iteration of the loop.

Syntax

continue;

How continue Works in Different Loops

  • for loop — jumps to the update expression (e.g., i++), then re-checks the condition.
  • while loop — jumps directly to the condition check.
  • do-while loop — jumps to the condition check at the bottom.

continue in a for Loop

#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0) {
            continue;  // Skip even numbers
        }
        cout << i << " ";
    }
    cout << endl;
    return 0;
}

Output:

1 3 5 7 9

When i is even, continue skips the cout statement and goes directly to i++.

continue in a while Loop

#include <iostream>
using namespace std;

int main() {
    int i = 0;
    while (i < 10) {
        i++;
        if (i % 3 == 0) {
            continue;  // Skip multiples of 3
        }
        cout << i << " ";
    }
    cout << endl;
    return 0;
}

Output:

1 2 4 5 7 8 10

Tip

When using continue in a while loop, make sure the loop variable is updated before the continue. Otherwise, the variable never changes and you get an infinite loop!

// BUG: Infinite loop!
int i = 0;
while (i < 10) {
    if (i == 5) {
        continue;  // i is never incremented past 5!
    }
    cout << i << " ";
    i++;           // This line is skipped when i == 5
}

break vs continue Comparison

Here’s a side-by-side comparison:

// break example: stops at 5
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;
    cout << i << " ";
}
// Output: 1 2 3 4

// continue example: skips 5
for (int i = 1; i <= 10; i++) {
    if (i == 5) continue;
    cout << i << " ";
}
// Output: 1 2 3 4 6 7 8 9 10

Summary:

  • break — Exits the loop completely. No more iterations run.
  • continue — Skips the current iteration. The loop continues with the next iteration.

Key Point

Think of break as “I’m done with this loop” and continue as “Skip this one, keep going.”

break & continue in Nested Loops

Both break and continue only affect the innermost loop they appear in. They do not affect outer loops.

break in Nested Loops

#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 3; i++) {
        for (int j = 1; j <= 5; j++) {
            if (j == 3) break;     // Only breaks the inner loop
            cout << "(" << i << "," << j << ") ";
        }
        cout << endl;
    }
    return 0;
}

Output:

(1,1) (1,2)
(2,1) (2,2)
(3,1) (3,2)

The inner loop breaks when j == 3, but the outer loop continues as normal.

continue in Nested Loops

#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 3; i++) {
        for (int j = 1; j <= 4; j++) {
            if (j == 2) continue;  // Only skips in the inner loop
            cout << "(" << i << "," << j << ") ";
        }
        cout << endl;
    }
    return 0;
}

Output:

(1,1) (1,3) (1,4)
(2,1) (2,3) (2,4)
(3,1) (3,3) (3,4)

Breaking Out of Multiple Loops

Since break only exits the innermost loop, breaking out of multiple nested loops requires a different approach. Common techniques include:

// Method 1: Use a flag variable
#include <iostream>
using namespace std;

int main() {
    bool found = false;
    for (int i = 0; i < 5 && !found; i++) {
        for (int j = 0; j < 5; j++) {
            if (i + j == 6) {
                cout << "Found: i=" << i << ", j=" << j << endl;
                found = true;
                break;   // Breaks inner loop; flag stops outer loop
            }
        }
    }
    return 0;
}
// Method 2: Extract into a function and use return
#include <iostream>
using namespace std;

void findPair() {
    for (int i = 0; i < 5; i++) {
        for (int j = 0; j < 5; j++) {
            if (i + j == 6) {
                cout << "Found: i=" << i << ", j=" << j << endl;
                return;  // Exits the function (and both loops)
            }
        }
    }
}

Tip

Extracting nested loops into a separate function and using return to exit is often the cleanest solution. It keeps the code readable and avoids flag variables.

The goto Statement (And Why to Avoid It)

C++ has a goto statement that can jump to a labeled statement anywhere in the same function. While it can be used to break out of nested loops, it is strongly discouraged in modern C++.

// goto example (NOT recommended!)
#include <iostream>
using namespace std;

int main() {
    for (int i = 0; i < 5; i++) {
        for (int j = 0; j < 5; j++) {
            if (i * j > 6) {
                goto done;  // Jump to the label
            }
            cout << i * j << " ";
        }
        cout << endl;
    }
    done:   // Label
    cout << "\nExited nested loops." << endl;
    return 0;
}

Key Point

Avoid goto in modern C++. It makes code harder to read, debug, and maintain. Use flag variables, return, or refactoring instead. The only semi-acceptable use of goto is breaking out of deeply nested loops in performance-critical code where a function call is too expensive.

return vs break

Both return and break can stop a loop, but they are fundamentally different:

  • break — exits the loop. Code after the loop still runs.
  • return — exits the entire function. Nothing after the return runs.
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i == 5) break;   // Exits the loop only
        cout << i << " ";
    }
    cout << "After loop" << endl;  // This DOES execute
    return 0;
}
// Output: 1 2 3 4 After loop
#include <iostream>
using namespace std;

int main() {
    for (int i = 1; i <= 10; i++) {
        if (i == 5) return 0;  // Exits the entire function
        cout << i << " ";
    }
    cout << "After loop" << endl;  // This does NOT execute
    return 0;
}
// Output: 1 2 3 4

Labeled Loops Concept

Some languages (like Java) support labeled loops, where you can break outerLabel; to exit an outer loop. C++ does not have this feature. In C++, you must use one of the alternative techniques:

  • Flag variable checked in the outer loop condition
  • Extract the loops into a function and use return
  • goto (last resort)
  • Throw an exception (extreme and not recommended for flow control)

Tip

If you find yourself needing labeled breaks, it’s usually a sign that the loop logic should be extracted into its own function. This improves both readability and testability.

Practical Examples

Example 1: Search an Array

Use break to stop searching once the target is found:

#include <iostream>
using namespace std;

int main() {
    int arr[] = {4, 7, 2, 9, 1, 5, 8};
    int size = 7;
    int target = 9;
    int index = -1;

    for (int i = 0; i < size; i++) {
        if (arr[i] == target) {
            index = i;
            break;   // Found it! No need to keep searching
        }
    }

    if (index != -1)
        cout << "Found " << target << " at index " << index << endl;
    else
        cout << target << " not found." << endl;

    return 0;
}

Output:

Found 9 at index 3

Example 2: Skip Specific Values

Use continue to skip unwanted values:

#include <iostream>
using namespace std;

int main() {
    // Print 1-20 but skip multiples of 4
    for (int i = 1; i <= 20; i++) {
        if (i % 4 == 0) continue;
        cout << i << " ";
    }
    cout << endl;
    return 0;
}

Output:

1 2 3 5 6 7 9 10 11 13 14 15 17 18 19

Example 3: Input Validation with Exit

Use break to let the user exit an input loop:

#include <iostream>
using namespace std;

int main() {
    int num;
    int sum = 0;
    int count = 0;

    while (true) {
        cout << "Enter a number (-1 to stop): ";
        cin >> num;

        if (num == -1) break;   // User wants to quit

        if (num < 0) {
            cout << "Negative numbers (except -1) are skipped." << endl;
            continue;           // Skip negative numbers
        }

        sum += num;
        count++;
    }

    if (count > 0)
        cout << "Average: " << (double)sum / count << endl;
    else
        cout << "No numbers entered." << endl;

    return 0;
}

Example 4: Menu with Exit Option

#include <iostream>
using namespace std;

int main() {
    int choice;
    while (true) {
        cout << "\n=== Calculator ===" << endl;
        cout << "1. Add" << endl;
        cout << "2. Subtract" << endl;
        cout << "3. Multiply" << endl;
        cout << "0. Exit" << endl;
        cout << "Choice: ";
        cin >> choice;

        if (choice == 0) {
            cout << "Goodbye!" << endl;
            break;              // Exit the menu loop
        }

        double a, b;
        cout << "Enter two numbers: ";
        cin >> a >> b;

        switch (choice) {
            case 1: cout << "Result: " << a + b << endl; break;
            case 2: cout << "Result: " << a - b << endl; break;
            case 3: cout << "Result: " << a * b << endl; break;
            default:
                cout << "Invalid choice." << endl;
                continue;       // Skip back to menu
        }
    }
    return 0;
}

Example 5: Finding the First Prime

Use break to stop checking divisors once a factor is found:

#include <iostream>
using namespace std;

int main() {
    int start = 20;

    for (int n = start; ; n++) {  // No upper bound
        bool isPrime = true;

        if (n < 2) continue;

        for (int d = 2; d * d <= n; d++) {
            if (n % d == 0) {
                isPrime = false;
                break;           // No need to check more divisors
            }
        }

        if (isPrime) {
            cout << "First prime >= " << start << " is: " << n << endl;
            break;               // Found it, exit outer loop
        }
    }
    return 0;
}

Output:

First prime >= 20 is: 23

Summary

  • break exits the innermost loop or switch immediately.
  • continue skips the rest of the current iteration and moves to the next iteration.
  • In a for loop, continue goes to the update expression; in a while/do-while, it goes to the condition.
  • Both break and continue only affect the innermost loop.
  • To exit multiple nested loops, use a flag variable, return, or (rarely) goto.
  • return exits the entire function, while break only exits the loop.
  • Avoid goto — use structured alternatives instead.
  • Be careful with continue in while loops — update the counter before the continue to avoid infinite loops.
