34. Inheritance

📚 Lesson 34 of 55 ⏱ 18 min read 🔨 Intermediate

What Is Inheritance?

Inheritance is one of the four fundamental pillars of Object-Oriented Programming (OOP). It allows a new class (the derived or child class) to acquire the properties and behaviours of an existing class (the base or parent class).

Inheritance models an IS-A relationship. For example:

  • A Dog is a Animal.
  • A Car is a Vehicle.
  • A Circle is a Shape.

The key benefits of inheritance include:

  • Code Reuse — write common functionality once in the base class and reuse it in all derived classes.
  • Extensibility — add new features to derived classes without modifying the base class.
  • Hierarchical Organisation — model real-world relationships naturally.
  • Polymorphism Foundation — inheritance enables runtime polymorphism through virtual functions.

Base Class and Derived Class Syntax

In C++ you create a derived class using a colon followed by an access specifier and the base class name:

C++
// Base class
class Animal {
public:
    string name;
    void eat() {
        cout << name << " is eating." << endl;
    }
};

// Derived class
class Dog : public Animal {
public:
    void bark() {
        cout << name << " says Woof!" << endl;
    }
};

int main() {
    Dog d;
    d.name = "Buddy";
    d.eat();   // Inherited from Animal
    d.bark();  // Defined in Dog
    return 0;
}

Output:

Buddy is eating.
Buddy says Woof!

The Dog class inherits the name member and the eat() method from Animal, then adds its own bark() method.

Public, Protected, Private Inheritance

The access specifier before the base class name controls how inherited members are accessible in the derived class:

C++
class Derived : public    Base { };  // Most common
class Derived : protected Base { };
class Derived : private   Base { };  // Default for class

Here is the access table showing how base class members are seen inside the derived class:

Base Member public Inheritance protected Inheritance private Inheritance
public public protected private
protected protected protected private
private Not accessible Not accessible Not accessible

Public inheritance is the most widely used form because it preserves the IS-A relationship and keeps the base class’s public interface intact.

Constructor and Destructor Order in Inheritance

When an object of a derived class is created or destroyed, C++ follows a specific order:

  • Construction: Base class constructor runs first, then the derived class constructor.
  • Destruction: Derived class destructor runs first, then the base class destructor.
C++
#include <iostream>
using namespace std;

class Base {
public:
    Base()  { cout << "Base constructor"  << endl; }
    ~Base() { cout << "Base destructor"  << endl; }
};

class Derived : public Base {
public:
    Derived()  { cout << "Derived constructor"  << endl; }
    ~Derived() { cout << "Derived destructor"  << endl; }
};

int main() {
    Derived d;
    return 0;
}

Output:

Base constructor
Derived constructor
Derived destructor
Base destructor

Calling Base Class Constructors

If the base class constructor requires arguments, you must call it explicitly using the member initialiser list:

C++
class Animal {
protected:
    string name;
    int age;
public:
    Animal(string n, int a) : name(n), age(a) {
        cout << "Animal created: " << name << endl;
    }
};

class Dog : public Animal {
    string breed;
public:
    // Call base constructor explicitly
    Dog(string n, int a, string b)
        : Animal(n, a), breed(b) {
        cout << "Dog created: " << breed << endl;
    }

    void info() {
        cout << name << ", Age: " << age
             << ", Breed: " << breed << endl;
    }
};

int main() {
    Dog d("Buddy", 3, "Labrador");
    d.info();
    return 0;
}

Output:

Animal created: Buddy
Dog created: Labrador
Buddy, Age: 3, Breed: Labrador

Overriding Member Functions

A derived class can override (redefine) a member function from the base class by declaring a function with the same name and signature:

C++
class Animal {
public:
    void speak() {
        cout << "Some generic animal sound" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() {  // Overrides Animal::speak
        cout << "Meow!" << endl;
    }
};

int main() {
    Cat c;
    c.speak();           // Calls Cat::speak
    c.Animal::speak();  // Explicitly calls Animal::speak
    return 0;
}

Output:

Meow!
Some generic animal sound

You can still call the base class version using the scope resolution operator (Base::function()).

The protected Access Specifier in Inheritance

The protected access specifier is particularly useful in inheritance scenarios. Members declared protected are:

  • Accessible inside the class itself and in derived classes.
  • Not accessible from outside code (like private to outsiders).
C++
class Vehicle {
protected:
    int speed;
public:
    Vehicle(int s) : speed(s) {}
};

class Car : public Vehicle {
public:
    Car(int s) : Vehicle(s) {}
    void showSpeed() {
        cout << "Speed: " << speed << endl;  // OK — protected is accessible
    }
};

int main() {
    Car c(120);
    c.showSpeed();
    // c.speed = 200;  // ERROR — protected not accessible outside
    return 0;
}

Output:

Speed: 120

Multiple Inheritance

C++ supports multiple inheritance, where a class can inherit from more than one base class:

C++
class Printable {
public:
    void print() { cout << "Printing..." << endl; }
};

class Saveable {
public:
    void save() { cout << "Saving..." << endl; }
};

class Document : public Printable, public Saveable {
public:
    string title;
    Document(string t) : title(t) {}
};

int main() {
    Document doc("Report");
    doc.print();  // From Printable
    doc.save();   // From Saveable
    return 0;
}

Output:

Printing...
Saving...

Diamond Problem and Virtual Inheritance

The diamond problem occurs when a class inherits from two classes that both inherit from the same base class, causing ambiguity and duplicate data:

C++
// Diamond Problem
class Animal {
public:
    string name;
    Animal() { cout << "Animal created" << endl; }
};

class Mammal : public Animal { };
class Bird   : public Animal { };

// Bat inherits TWO copies of Animal — ambiguous!
class Bat : public Mammal, public Bird { };

The solution is virtual inheritance, which ensures only one copy of the base class exists:

C++
class Animal {
public:
    string name;
    Animal() { cout << "Animal created" << endl; }
};

class Mammal : virtual public Animal { };
class Bird   : virtual public Animal { };

// Only ONE copy of Animal — no ambiguity
class Bat : public Mammal, public Bird { };

int main() {
    Bat b;
    b.name = "Batty";  // No ambiguity now
    cout << b.name << endl;
    return 0;
}

Output:

Animal created
Batty

Multilevel Inheritance

In multilevel inheritance, a class is derived from another derived class, forming a chain:

C++
class LivingBeing {
public:
    void breathe() { cout << "Breathing..." << endl; }
};

class Animal : public LivingBeing {
public:
    void eat() { cout << "Eating..." << endl; }
};

class Dog : public Animal {
public:
    void bark() { cout << "Woof!" << endl; }
};

int main() {
    Dog d;
    d.breathe();  // From LivingBeing
    d.eat();      // From Animal
    d.bark();     // From Dog
    return 0;
}

Output:

Breathing...
Eating...
Woof!

Hierarchical Inheritance

In hierarchical inheritance, multiple classes inherit from a single base class:

C++
class Shape {
public:
    void draw() { cout << "Drawing shape" << endl; }
};

class Circle    : public Shape { };
class Rectangle : public Shape { };
class Triangle  : public Shape { };

int main() {
    Circle c;
    Rectangle r;
    Triangle t;
    c.draw();  // All inherit draw()
    r.draw();
    t.draw();
    return 0;
}

When to Use Inheritance vs Composition

Not every relationship should use inheritance. Consider:

Criteria Inheritance (IS-A) Composition (HAS-A)
Relationship “A Dog is a Animal” “A Car has a Engine”
Coupling Tight — derived depends on base Loose — components are independent
Flexibility Less flexible at runtime More flexible, parts can be swapped
Code Reuse Inherits all public/protected members Delegates to contained objects
Best For True type hierarchies Building complex objects from parts
C++
// Composition example (HAS-A)
class Engine {
public:
    void start() { cout << "Engine started" << endl; }
};

class Car {
    Engine engine;  // Car HAS-A Engine
public:
    void start() {
        engine.start();
        cout << "Car is ready" << endl;
    }
};

Rule of thumb: prefer composition over inheritance unless there is a clear IS-A relationship and you need polymorphism.

Complete Example: Shape Hierarchy

Here is a comprehensive example that combines inheritance concepts into a practical shape hierarchy:

C++
#include <iostream>
#include <string>
#include <cmath>
using namespace std;

// Base class
class Shape {
protected:
    string colour;
public:
    Shape(string c = "Red") : colour(c) {}

    void setColour(string c) { colour = c; }
    string getColour() const { return colour; }

    void describe() {
        cout << "I am a " << colour << " shape." << endl;
    }

    virtual double area() { return 0; }
    virtual double perimeter() { return 0; }

    virtual ~Shape() {}
};

// Derived: Circle
class Circle : public Shape {
    double radius;
public:
    Circle(double r, string c = "Blue")
        : Shape(c), radius(r) {}

    double area() override {
        return M_PI * radius * radius;
    }
    double perimeter() override {
        return 2 * M_PI * radius;
    }
};

// Derived: Rectangle
class Rectangle : public Shape {
protected:
    double width, height;
public:
    Rectangle(double w, double h, string c = "Green")
        : Shape(c), width(w), height(h) {}

    double area() override {
        return width * height;
    }
    double perimeter() override {
        return 2 * (width + height);
    }
};

// Multilevel: Square inherits Rectangle
class Square : public Rectangle {
public:
    Square(double side, string c = "Yellow")
        : Rectangle(side, side, c) {}
};

int main() {
    Circle c(5);
    Rectangle r(4, 6);
    Square s(3);

    cout << "Circle    — Area: " << c.area()
         << ", Perimeter: " << c.perimeter() << endl;

    cout << "Rectangle — Area: " << r.area()
         << ", Perimeter: " << r.perimeter() << endl;

    cout << "Square    — Area: " << s.area()
         << ", Perimeter: " << s.perimeter() << endl;

    // Polymorphic usage
    Shape* shapes[] = {&c, &r, &s};
    for (Shape* sh : shapes) {
        sh->describe();
        cout << "  Area: " << sh->area() << endl;
    }

    return 0;
}

Output:

Circle    — Area: 78.5398, Perimeter: 31.4159
Rectangle — Area: 24, Perimeter: 20
Square    — Area: 9, Perimeter: 12
I am a Blue shape.
  Area: 78.5398
I am a Green shape.
  Area: 24
I am a Yellow shape.
  Area: 9

Key Takeaways

  • Inheritance lets a derived class reuse and extend the functionality of a base class.
  • Use public inheritance for IS-A relationships — it is the most common form.
  • protected members are accessible in derived classes but hidden from outside code.
  • Constructors execute base-first; destructors execute derived-first.
  • Pass arguments to the base constructor via the member initialiser list.
  • Override base class functions by declaring a function with the same signature in the derived class.
  • Multiple inheritance is supported but can cause the diamond problem — solve it with virtual inheritance.
  • Multilevel inheritance forms a chain; hierarchical inheritance creates a tree.
  • Prefer composition over inheritance unless a genuine IS-A relationship exists.
  • Inheritance is the foundation for polymorphism, the topic of the next lesson.
