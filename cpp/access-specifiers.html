32. Access Specifiers

📚 Lesson 32 of 55 ⏱ 14 min read 🔨 Intermediate

What Are Access Specifiers?

Access specifiers (also called access modifiers) are keywords in C++ that control who can access the members (variables and functions) of a class. They are a cornerstone of encapsulation — one of the four pillars of object-oriented programming.

C++ provides three access specifiers:

  • public — members are accessible from anywhere the object is visible.
  • private — members are accessible only within the class itself.
  • protected — members are accessible within the class and by derived (child) classes.

Access specifiers appear inside a class definition followed by a colon. Every member declared after a specifier has that access level until the next specifier or the end of the class.

class MyClass {
public:       // accessible from anywhere
    int x;

private:      // accessible only inside this class
    int y;

protected:    // accessible in this class and derived classes
    int z;
};

public Members

Members declared under public can be accessed from any part of the program — inside the class, from derived classes, or from completely unrelated code using an object of the class.

#include <iostream>
#include <string>
using namespace std;

class Animal {
public:
    string name;
    int age;

    void speak() {
        cout << name << " says hello!" << endl;
    }
};

int main() {
    Animal cat;
    cat.name = "Whiskers";   // OK – public member
    cat.age  = 3;            // OK – public member
    cat.speak();             // OK – public member function
    return 0;
}

Public members form the interface of a class — the set of operations that outside code is allowed to use. Keep the public interface small and well-defined.

private Members

Members declared under private are hidden from everything outside the class. Only member functions (and friends) of the same class can access them.

#include <iostream>
using namespace std;

class Secret {
private:
    int code;

public:
    void setCode(int c) {
        if (c > 0)
            code = c;       // OK – accessing private member inside the class
    }

    int getCode() {
        return code;        // OK
    }
};

int main() {
    Secret s;
    // s.code = 42;         // ERROR: 'code' is private
    s.setCode(42);          // OK – using public setter
    cout << s.getCode() << endl;  // 42
    return 0;
}

Private access is the default for members of a class. If you do not specify any access specifier, members are private. This is one way class differs from struct.

protected Members

protected members sit between public and private. They cannot be accessed from outside the class hierarchy, but can be accessed by derived classes.

#include <iostream>
using namespace std;

class Vehicle {
protected:
    int speed;

public:
    Vehicle() : speed(0) {}
};

class Car : public Vehicle {
public:
    void accelerate(int amount) {
        speed += amount;     // OK – protected member accessible in derived class
    }
    void showSpeed() {
        cout << "Speed: " << speed << " km/h" << endl;
    }
};

int main() {
    Car c;
    // c.speed = 100;       // ERROR: 'speed' is protected
    c.accelerate(60);       // OK – public method
    c.showSpeed();           // Speed: 60 km/h
    return 0;
}

Use protected when you want derived classes to have direct access to a member, but you still want to hide it from the rest of the program.

Access Specifier Comparison Table

Access Level Same Class Derived Class Outside Code
public ✔ Yes ✔ Yes ✔ Yes
protected ✔ Yes ✔ Yes ✘ No
private ✔ Yes ✘ No ✘ No

Getters and Setters

Because private data members cannot be accessed directly from outside the class, we use getter (accessor) and setter (mutator) functions to provide controlled access. This pattern lets you add validation or side effects when reading or writing data.

#include <iostream>
#include <string>
using namespace std;

class Student {
private:
    string name;
    int grade;   // 0–100

public:
    // Getter (accessor)
    string getName() const {
        return name;
    }

    // Setter (mutator) with validation
    void setName(const string& n) {
        if (!n.empty())
            name = n;
    }

    // Getter
    int getGrade() const {
        return grade;
    }

    // Setter with validation
    void setGrade(int g) {
        if (g >= 0 && g <= 100)
            grade = g;
        else
            cout << "Invalid grade!" << endl;
    }
};

int main() {
    Student s;
    s.setName("Alice");
    s.setGrade(95);
    cout << s.getName() << ": " << s.getGrade() << endl;  // Alice: 95

    s.setGrade(150);       // Invalid grade!
    cout << s.getGrade() << endl;    // still 95
    return 0;
}

Why use getters/setters instead of public members?

  • You can validate input before storing it.
  • You can make a member read-only (provide a getter but no setter).
  • You can change the internal representation without breaking outside code.
  • You can add logging, triggers, or computed values.

Default Access: class vs struct

In C++, class and struct are almost identical. The only difference is the default access level:

  • class → members are private by default.
  • struct → members are public by default.
class ClassExample {
    int a;       // private by default
};

struct StructExample {
    int a;       // public by default
};

int main() {
    ClassExample c;
    // c.a = 10;    // ERROR: 'a' is private

    StructExample s;
    s.a = 10;       // OK: 'a' is public
    return 0;
}

The same applies to inheritance: inheriting from a class is private by default, while inheriting from a struct is public by default.

class Base { public: int x; };

class  DerivedA : Base {};          // private inheritance (default for class)
struct DerivedB : Base {};          // public inheritance  (default for struct)

The friend Keyword

Sometimes you need an external function or another class to access the private or protected members of a class. The friend keyword grants this access.

Friend Functions

A friend function is declared inside a class with the friend keyword. It is not a member of the class, but it can access all private and protected members.

#include <iostream>
using namespace std;

class Box {
private:
    double width;

public:
    Box(double w) : width(w) {}

    // Declare a non-member function as a friend
    friend void printWidth(const Box& b);
};

// Definition of the friend function (not a member)
void printWidth(const Box& b) {
    cout << "Width: " << b.width << endl;  // can access private member
}

int main() {
    Box box(15.5);
    printWidth(box);   // Width: 15.5
    return 0;
}

Friend Classes

When an entire class is declared as a friend, all of its member functions can access the private and protected members of the granting class.

#include <iostream>
using namespace std;

class Engine {
private:
    int horsepower;

public:
    Engine(int hp) : horsepower(hp) {}

    // Grant Mechanic class full access
    friend class Mechanic;
};

class Mechanic {
public:
    void inspect(const Engine& e) {
        // Can access private member because Mechanic is a friend
        cout << "Engine HP: " << e.horsepower << endl;
    }
};

int main() {
    Engine v8(450);
    Mechanic mike;
    mike.inspect(v8);   // Engine HP: 450
    return 0;
}

Important points about friend:

  • Friendship is not mutual — if A declares B as a friend, B can access A’s private members, but A cannot access B’s.
  • Friendship is not inherited — a derived class of a friend does not automatically become a friend.
  • Friendship is not transitive — a friend of a friend is not automatically a friend.
  • Use friend sparingly — it breaks encapsulation by design, so only use it when there is a clear benefit.

Best Practices for Access Control

  • Make data members private by default. Expose them only through public getter/setter methods when necessary.
  • Keep the public interface minimal. Only make functions public if outside code genuinely needs to call them.
  • Use protected thoughtfully. Prefer private plus public/protected accessors over raw protected data.
  • Prefer class over struct for types that have invariants or non-trivial behaviour. Use struct for simple data aggregates.
  • Use const on getters to indicate they do not modify the object.
  • Validate in setters. This is the whole point of controlled access — enforce your class invariants.
  • Limit friend usage. Every friend declaration is a window through encapsulation. Document why it is needed.
  • Think about the maintenance cost. Once a member is public, changing it can break every consumer of your class.

Complete Example: BankAccount Class

The following example brings together all the concepts — private data, public interface, getters/setters with validation, a friend function, and a protected member for inheritance.

#include <iostream>
#include <string>
using namespace std;

class BankAccount {
private:
    string owner;
    double balance;

protected:
    string accountType;      // accessible to derived classes

public:
    // Constructor
    BankAccount(const string& name, double initial, const string& type)
        : owner(name), balance(initial), accountType(type) {}

    // Getter – owner
    string getOwner() const { return owner; }

    // Getter – balance
    double getBalance() const { return balance; }

    // Getter – account type
    string getType() const { return accountType; }

    // Deposit with validation
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            cout << "Deposited $" << amount << endl;
        } else {
            cout << "Invalid deposit amount!" << endl;
        }
    }

    // Withdraw with validation
    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            cout << "Withdrew $" << amount << endl;
            return true;
        }
        cout << "Insufficient funds or invalid amount!" << endl;
        return false;
    }

    // Friend function – can access private members
    friend void auditAccount(const BankAccount& acc);
};

// Friend function definition
void auditAccount(const BankAccount& acc) {
    cout << "=== Audit ===" << endl;
    cout << "Owner:   " << acc.owner   << endl;
    cout << "Balance: $" << acc.balance << endl;
    cout << "Type:    " << acc.accountType << endl;
}

// Derived class uses protected member
class SavingsAccount : public BankAccount {
private:
    double interestRate;

public:
    SavingsAccount(const string& name, double initial, double rate)
        : BankAccount(name, initial, "Savings"), interestRate(rate) {}

    void applyInterest() {
        double interest = getBalance() * interestRate;
        deposit(interest);
        // accountType is accessible because it is protected
        cout << "[" << accountType << "] Interest applied: $"
             << interest << endl;
    }
};

int main() {
    SavingsAccount sa("Alice", 1000.0, 0.05);

    sa.deposit(500);          // Deposited $500
    sa.withdraw(200);         // Withdrew $200

    cout << "Balance: $" << sa.getBalance() << endl;  // $1300

    sa.applyInterest();       // Deposited $65, Interest applied

    auditAccount(sa);         // Friend function accesses private members

    // sa.balance = 999999;   // ERROR: balance is private
    // sa.owner = "Hacker";   // ERROR: owner is private

    return 0;
}

Key Takeaways

  • C++ has three access specifiers: public, private, and protected.
  • public members are accessible from anywhere; they form the class’s external interface.
  • private members are accessible only inside the class — this is the default for class.
  • protected members are like private, but also accessible in derived classes.
  • Use getters and setters to provide controlled, validated access to private data.
  • The only difference between class and struct is the default access level (private vs public).
  • The friend keyword lets specific functions or classes access private/protected members — use it sparingly.
  • Friendship is not mutual, not inherited, and not transitive.
  • Always default to private data and the smallest public interface possible.
  • Good access control makes code easier to maintain, debug, and extend.
