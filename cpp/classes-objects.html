30. Classes & Objects

📚 Lesson 30 of 55 ⏱ 18 min read 🔨 Intermediate

What Is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm that organises software around objects rather than plain functions and logic. An object bundles together data (attributes) and behaviour (methods) into a single, self-contained unit.

Before OOP, C programmers relied on procedural programming — writing sequences of functions that operate on loose collections of variables. This works for small programs, but as projects grow, keeping track of which functions modify which data quickly becomes unmanageable.

OOP addresses this by grouping related data and the functions that act on that data into cohesive units. The four pillars of OOP are:

  • Encapsulation: Bundling data and methods together and restricting direct access to internal details.
  • Abstraction: Hiding complex implementation details behind a simple interface.
  • Inheritance: Creating new classes from existing ones, reusing and extending behaviour.
  • Polymorphism: Allowing objects of different classes to be treated through a common interface.

C++ is a multi-paradigm language — it supports procedural, object-oriented, and generic programming. Starting with this lesson, we step into OOP territory by learning about classes and objects.

Classes vs Objects

The relationship between a class and an object is like the relationship between a blueprint and a house:

  • A class is the blueprint — it defines what data the object will hold and what operations it can perform, but it does not occupy memory for actual object data on its own.
  • An object is a house built from that blueprint — it is a concrete instance of the class that lives in memory and stores real values.

You can build many houses from a single blueprint. Likewise, you can create many objects from a single class, and each object has its own independent copy of the member data.

// Blueprint (class)
class Car {
    string brand;
    int year;
    void drive();
};

// Houses (objects)
Car myCar;      // object 1
Car yourCar;    // object 2
Car rentalCar;  // object 3

Defining a Class

A class definition starts with the keyword class, followed by the class name, a pair of braces, and a semicolon after the closing brace. Inside the braces you declare member variables (data) and member functions (behaviour).

#include <iostream>
#include <string>
using namespace std;

class Dog {
public:                   // access specifier – more on this later
    string name;          // member variable
    string breed;         // member variable
    int age;              // member variable

    void bark() {         // member function
        cout << name << " says: Woof!" << endl;
    }

    void info() {         // member function
        cout << "Name: "  << name  << endl;
        cout << "Breed: " << breed << endl;
        cout << "Age: "   << age   << " years" << endl;
    }
};

Key points:

  • The public: label makes the members accessible from outside the class. Without it, members default to private in a class.
  • Member variables describe the state of each object.
  • Member functions describe the behaviour of each object.
  • Don’t forget the semicolon after the closing brace — it is required.

Creating Objects

Creating an object from a class is called instantiation. You declare an object the same way you declare any variable — the class name acts as the type:

int main() {
    Dog myDog;              // create an object of type Dog
    myDog.name  = "Buddy";
    myDog.breed = "Golden Retriever";
    myDog.age   = 3;

    myDog.bark();           // Buddy says: Woof!
    myDog.info();           // prints name, breed, age

    return 0;
}

When Dog myDog; executes, the compiler allocates memory for all of Dog’s member variables (name, breed, age). The object myDog now exists in memory and is ready to use.

Accessing Members with the Dot Operator

The dot operator (.) connects an object to its members. Use it to read or write member variables and to call member functions:

myDog.name = "Max";        // write a member variable
cout << myDog.name;        // read a member variable
myDog.bark();              // call a member function

If you have a pointer to an object, you use the arrow operator (->) instead:

Dog* ptr = &myDog;
ptr->bark();               // same as (*ptr).bark()
cout << ptr->name;         // same as (*ptr).name

Member Functions Inside and Outside the Class

You can define member functions in two ways:

1. Inside the Class (Inline)

When you write the function body directly inside the class definition, the compiler treats it as an inline suggestion:

class Circle {
public:
    double radius;

    double area() {                      // defined inside
        return 3.14159 * radius * radius;
    }
};

2. Outside the Class (Scope Resolution Operator ::)

For larger functions, it is cleaner to declare the function inside the class and define it outside using the scope resolution operator :::

class Circle {
public:
    double radius;

    double area();          // declaration only
    double circumference(); // declaration only
};

// Definition outside the class
double Circle::area() {
    return 3.14159 * radius * radius;
}

double Circle::circumference() {
    return 2 * 3.14159 * radius;
}

The syntax Circle::area() tells the compiler that area() belongs to the Circle class. This is the preferred style in production code because it keeps the class definition concise and readable.

Multiple Objects from One Class

Each object has its own copy of the member variables. Changing one object does not affect another:

#include <iostream>
#include <string>
using namespace std;

class Car {
public:
    string brand;
    string model;
    int year;

    void display() {
        cout << year << " " << brand << " " << model << endl;
    }
};

int main() {
    Car car1;
    car1.brand = "Toyota";
    car1.model = "Corolla";
    car1.year  = 2022;

    Car car2;
    car2.brand = "Honda";
    car2.model = "Civic";
    car2.year  = 2023;

    Car car3;
    car3.brand = "Ford";
    car3.model = "Mustang";
    car3.year  = 2024;

    car1.display();  // 2022 Toyota Corolla
    car2.display();  // 2023 Honda Civic
    car3.display();  // 2024 Ford Mustang

    return 0;
}

All three objects share the same structure (brand, model, year, display) but hold completely independent data.

The this Pointer

Every non-static member function has an implicit pointer called this that points to the object on which the function was called. The compiler passes it automatically — you never see it in the parameter list, but you can use it explicitly:

class Rectangle {
public:
    double width;
    double height;

    void setDimensions(double width, double height) {
        // Parameter names shadow member names.
        // Use this-> to refer to the member variables.
        this->width  = width;
        this->height = height;
    }

    double area() {
        return this->width * this->height;   // explicit, but optional here
    }
};

Common uses of this:

  • Disambiguation: When a parameter name shadows a member variable, this->member removes ambiguity.
  • Returning the current object: return *this; enables method chaining.
  • Passing the current object to another function.
class Builder {
public:
    int x = 0, y = 0, z = 0;

    Builder& setX(int x) { this->x = x; return *this; }
    Builder& setY(int y) { this->y = y; return *this; }
    Builder& setZ(int z) { this->z = z; return *this; }
};

int main() {
    Builder b;
    b.setX(1).setY(2).setZ(3);  // method chaining via *this
    return 0;
}

Class vs Struct in C++

In C++, struct and class are almost identical. The only technical difference is the default access level:

Featureclassstruct
Default member accessprivatepublic
Default inheritance accessprivatepublic
Can have methods?YesYes
Can have constructors?YesYes
Can use inheritance?YesYes
 
struct Point {
    double x;   // public by default
    double y;   // public by default
};

class Point2 {
    double x;   // private by default – not accessible outside
    double y;   // private by default
};

Convention: Use struct for simple data holders (PODs) with no invariants. Use class when you have member functions, encapsulation, or complex behaviour.

Separating Declaration and Implementation

In real-world C++ projects, you almost always split a class across two files:

  • A header file (.h or .hpp) — contains the class declaration (member variables and function prototypes).
  • A source file (.cpp) — contains the definitions (function bodies).

student.h — Header File

#ifndef STUDENT_H
#define STUDENT_H

#include <string>
using namespace std;

class Student {
public:
    string name;
    int id;
    double gpa;

    void display();
    bool isHonorRoll();
};

#endif

The #ifndef / #define / #endif pattern is called an include guard. It prevents the header from being included more than once, which would cause duplicate-definition errors.

student.cpp — Source File

#include <iostream>
#include "student.h"   // include the header

void Student::display() {
    cout << "ID: "   << id   << endl;
    cout << "Name: " << name << endl;
    cout << "GPA: "  << gpa  << endl;
}

bool Student::isHonorRoll() {
    return gpa >= 3.5;
}

main.cpp — Using the Class

#include <iostream>
#include "student.h"

int main() {
    Student s;
    s.name = "Alice";
    s.id   = 1001;
    s.gpa  = 3.85;

    s.display();

    if (s.isHonorRoll()) {
        cout << s.name << " is on the Honor Roll!" << endl;
    }

    return 0;
}

This separation has practical benefits: compile times improve because only the .cpp file that changes needs to be recompiled, and the header provides a clean interface for other files to use.

Complete Example: A Student Class

Let’s put everything together in a single, self-contained program that demonstrates class definition, multiple objects, member functions defined outside the class, and the this pointer:

#include <iostream>
#include <string>
using namespace std;

class Student {
public:
    string name;
    int id;
    double grades[5];

    // Declare member functions
    void setInfo(string name, int id);
    double average();
    void display();
    string letterGrade();
};

// ---- Definitions outside the class ----

void Student::setInfo(string name, int id) {
    this->name = name;   // use this-> to disambiguate
    this->id   = id;
}

double Student::average() {
    double sum = 0;
    for (int i = 0; i < 5; i++) {
        sum += grades[i];
    }
    return sum / 5.0;
}

string Student::letterGrade() {
    double avg = average();
    if (avg >= 90) return "A";
    if (avg >= 80) return "B";
    if (avg >= 70) return "C";
    if (avg >= 60) return "D";
    return "F";
}

void Student::display() {
    cout << "=========================" << endl;
    cout << "Student ID : " << id      << endl;
    cout << "Name       : " << name    << endl;
    cout << "Grades     : ";
    for (int i = 0; i < 5; i++) {
        cout << grades[i] << " ";
    }
    cout << endl;
    cout << "Average    : " << average()     << endl;
    cout << "Letter     : " << letterGrade() << endl;
    cout << "=========================" << endl;
}

int main() {
    Student s1;
    s1.setInfo("Alice", 1001);
    s1.grades[0] = 92; s1.grades[1] = 88;
    s1.grades[2] = 95; s1.grades[3] = 79;
    s1.grades[4] = 85;

    Student s2;
    s2.setInfo("Bob", 1002);
    s2.grades[0] = 72; s2.grades[1] = 65;
    s2.grades[2] = 80; s2.grades[3] = 58;
    s2.grades[4] = 70;

    s1.display();
    s2.display();

    // Compare averages
    if (s1.average() > s2.average()) {
        cout << s1.name << " has a higher average." << endl;
    } else {
        cout << s2.name << " has a higher average." << endl;
    }

    return 0;
}

Output:

=========================
Student ID : 1001
Name       : Alice
Grades     : 92 88 95 79 85
Average    : 87.8
Letter     : B
=========================
=========================
Student ID : 1002
Name       : Bob
Grades     : 72 65 80 58 70
Average    : 69
Letter     : D
=========================
Alice has a higher average.

Key Takeaways

  • A class is a user-defined type that bundles data (member variables) and behaviour (member functions) together.
  • An object is a concrete instance of a class — it occupies memory and holds actual values.
  • Use the dot operator (.) to access members of an object, and the arrow operator (->) for pointers to objects.
  • Member functions can be defined inside the class (inline) or outside using the scope resolution operator ::.
  • The this pointer is an implicit pointer to the current object; use it to disambiguate names or enable method chaining.
  • In C++, struct and class differ only in default access: struct defaults to public, class defaults to private.
  • Real-world projects separate declaration (.h) and implementation (.cpp) for cleaner code and faster compilation.
  • You can create many objects from a single class, each with independent data.
  • Classes are the foundation of OOP in C++; upcoming lessons on constructors, access specifiers, inheritance, and polymorphism all build on this concept.
