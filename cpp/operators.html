8. Operators

📚 Lesson 8 of 55 ⏱ 25 min read 🌱 Beginner

What Are Operators?

Operators are special symbols or keywords in C++ that perform operations on one or more operands (values or variables). They are the building blocks of every expression you write—from simple arithmetic to complex logical comparisons. Understanding operators is essential because nearly every line of C++ code uses at least one.

C++ provides a rich set of operators that can be grouped into several categories:

  • Arithmetic operators – perform mathematical calculations
  • Assignment operators – assign values to variables
  • Comparison (relational) operators – compare two values
  • Logical operators – combine boolean expressions
  • Increment/Decrement operators – increase or decrease a value by one
  • Bitwise operators – operate on individual bits
  • Ternary operator – shorthand conditional
  • sizeof operator – determine the size of a type or variable
  • Type casting operators – convert between types

Key Point

An operand is a value that an operator acts upon. A unary operator takes one operand, a binary operator takes two, and a ternary operator takes three.

Arithmetic Operators

Arithmetic operators perform standard mathematical operations on numeric values. C++ supports the following arithmetic operators:

OperatorNameExampleResult
+Addition5 + 38
-Subtraction10 - 46
*Multiplication6 * 742
/Division15 / 43 (integer division)
%Modulus (remainder)15 % 43

Basic Arithmetic Example

#include <iostream>
using namespace std;

int main() {
    int a = 20, b = 7;

    cout << "a + b = " << a + b << endl;   // 27
    cout << "a - b = " << a - b << endl;   // 13
    cout << "a * b = " << a * b << endl;   // 140
    cout << "a / b = " << a / b << endl;   // 2  (integer division)
    cout << "a % b = " << a % b << endl;   // 6

    return 0;
}

Key Point

When you divide two integers in C++, the result is always an integer—the fractional part is discarded (truncated toward zero). To get a decimal result, at least one operand must be a floating-point type (float or double).

Integer vs. Floating-Point Division

#include <iostream>
using namespace std;

int main() {
    int x = 15, y = 4;

    // Integer division - fractional part is truncated
    cout << "15 / 4 = " << x / y << endl;          // Output: 3

    // Floating-point division
    double a = 15.0, b = 4.0;
    cout << "15.0 / 4.0 = " << a / b << endl;      // Output: 3.75

    // Mixed: int and double - int is promoted to double
    cout << "15 / 4.0 = " << x / b << endl;        // Output: 3.75

    return 0;
}

The Modulus Operator (%)

The modulus operator returns the remainder after integer division. It works only with integer types.

#include <iostream>
using namespace std;

int main() {
    cout << "10 % 3 = " << 10 % 3 << endl;   // 1
    cout << "20 % 5 = " << 20 % 5 << endl;   // 0
    cout << "7 % 2 = "  << 7 % 2  << endl;   // 1

    // Common use: check if a number is even or odd
    int num = 42;
    if (num % 2 == 0) {
        cout << num << " is even" << endl;
    } else {
        cout << num << " is odd" << endl;
    }

    return 0;
}

Tip

Use num % 2 == 0 to check if a number is even, and num % 2 != 0 to check if it is odd. This is one of the most common uses of the modulus operator.

Assignment Operators

Assignment operators store a value in a variable. The basic assignment operator is =, but C++ also provides compound assignment operators that combine an arithmetic or bitwise operation with assignment.

OperatorExampleEquivalent To

+=x += 5x = x + 5
-=x -= 3x = x - 3
*=x *= 2x = x * 2
/=x /= 4x = x / 4
%=x %= 3x = x % 3
&=x &= 0xFx = x & 0xF
|=x |= 0x1x = x | 0x1
^=x ^= 0xFFx = x ^ 0xFF
<<=x <<= 2x = x << 2
>>=x >>= 1x = x >> 1

Assignment Operators in Action

#include <iostream>
using namespace std;

int main() {
    int x = 100;
    cout << "Initial: x = " << x << endl;  // 100

    x += 25;   // x = x + 25
    cout << "x += 25 : " << x << endl;      // 125

    x -= 10;   // x = x - 10
    cout << "x -= 10 : " << x << endl;      // 115

    x *= 2;    // x = x * 2
    cout << "x *= 2  : " << x << endl;      // 230

    x /= 5;    // x = x / 5
    cout << "x /= 5  : " << x << endl;      // 46

    x %= 7;    // x = x % 7
    cout << "x %= 7  : " << x << endl;      // 4

    return 0;
}

Tip

Compound assignment operators like += are not just shorter to write—they also make your intent clearer. Instead of repeating the variable name, you communicate “add this value to the existing variable.”

Comparison (Relational) Operators

Comparison operators compare two values and return a boolean result: true (1) or false (0). These are fundamental to decision-making in programs.

OperatorNameExampleResult
==Equal to5 == 5true
!=Not equal to5 != 3true
<Less than3 < 5true
>Greater than5 > 3true
<=Less than or equal to5 <= 5true
>=Greater than or equal to7 >= 3true

Comparison Operators Example

#include <iostream>
using namespace std;

int main() {
    int a = 10, b = 20;

    cout << "a == b : " << (a == b) << endl;   // 0 (false)
    cout << "a != b : " << (a != b) << endl;   // 1 (true)
    cout << "a < b  : " << (a < b)  << endl;   // 1 (true)
    cout << "a > b  : " << (a > b)  << endl;   // 0 (false)
    cout << "a <= b : " << (a <= b) << endl;   // 1 (true)
    cout << "a >= b : " << (a >= b) << endl;   // 0 (false)

    return 0;
}

Key Point

Be careful not to confuse the assignment operator = with the equality operator ==. Writing if (x = 5) assigns 5 to x and always evaluates to true, which is almost certainly a bug. Use if (x == 5) to compare.

Using Comparison Operators with Conditions

#include <iostream>
using namespace std;

int main() {
    int age;
    cout << "Enter your age: ";
    cin >> age;

    if (age >= 18) {
        cout << "You are an adult." << endl;
    } else if (age >= 13) {
        cout << "You are a teenager." << endl;
    } else {
        cout << "You are a child." << endl;
    }

    return 0;
}

Logical Operators

Logical operators combine multiple boolean expressions into a single result. They are essential for building complex conditions in if statements and loops.

OperatorNameDescriptionExample
&&Logical ANDReturns true if both operands are true(5 > 3) && (8 > 6) → true
||Logical ORReturns true if at least one operand is true(5 > 3) || (8 < 6) → true
!Logical NOTReverses the boolean value!(5 > 3) → false

Logical Operators Example

#include <iostream>
using namespace std;

int main() {
    int age = 25;
    bool hasLicense = true;

    // AND: both conditions must be true
    if (age >= 18 && hasLicense) {
        cout << "You can drive." << endl;
    }

    // OR: at least one condition must be true
    int day = 6; // Saturday
    if (day == 6 || day == 0) {
        cout << "It's the weekend!" << endl;
    }

    // NOT: invert the condition
    bool isRaining = false;
    if (!isRaining) {
        cout << "No umbrella needed." << endl;
    }

    return 0;
}

Key Point

C++ uses short-circuit evaluation for logical operators. With &&, if the left operand is false, the right operand is never evaluated. With ||, if the left operand is true, the right operand is skipped. This can improve performance and prevent errors such as null pointer dereferences.

Increment and Decrement Operators

The increment (++) and decrement (--) operators increase or decrease an integer value by 1. They come in two forms: prefix and postfix.

OperatorNameDescription
++xPre-incrementIncrements x, then returns the new value
x++Post-incrementReturns the current value, then increments x
--xPre-decrementDecrements x, then returns the new value
x--Post-decrementReturns the current value, then decrements x

Pre-increment vs. Post-increment

#include <iostream>
using namespace std;

int main() {
    int a = 5;

    // Post-increment: use current value, then increment
    cout << "a++ = " << a++ << endl;   // prints 5
    cout << "a   = " << a   << endl;   // prints 6 (a is now 6)

    int b = 5;

    // Pre-increment: increment first, then use new value
    cout << "++b = " << ++b << endl;   // prints 6
    cout << "b   = " << b   << endl;   // prints 6

    return 0;
}

Pre-decrement vs. Post-decrement

#include <iostream>
using namespace std;

int main() {
    int x = 10;

    cout << "x-- = " << x-- << endl;   // prints 10
    cout << "x   = " << x   << endl;   // prints 9

    int y = 10;
    cout << "--y = " << --y << endl;   // prints 9
    cout << "y   = " << y   << endl;   // prints 9

    return 0;
}

Tip

When the return value isn’t used (e.g., a standalone i++; or ++i;), both forms have the same effect. In loops like for (int i = 0; i < n; ++i), prefer pre-increment for consistency, though modern compilers optimize both equally for primitive types.

Common Use in Loops

#include <iostream>
using namespace std;

int main() {
    // Increment in a for loop
    for (int i = 0; i < 5; ++i) {
        cout << i << " ";
    }
    cout << endl;  // Output: 0 1 2 3 4

    // Decrement in a while loop
    int countdown = 5;
    while (countdown > 0) {
        cout << countdown-- << " ";
    }
    cout << endl;  // Output: 5 4 3 2 1

    return 0;
}

Bitwise Operators

Bitwise operators work at the binary (bit) level of integer values. They are commonly used in low-level programming, embedded systems, graphics, networking, and performance-critical code.

OperatorNameDescription
&Bitwise ANDSets each bit to 1 if both bits are 1
|Bitwise ORSets each bit to 1 if at least one bit is 1
^Bitwise XORSets each bit to 1 if only one of the bits is 1
~Bitwise NOTInverts all the bits (one’s complement)
<<Left shiftShifts bits to the left, filling with zeros
>>Right shiftShifts bits to the right

Bitwise AND, OR, XOR

#include <iostream>
using namespace std;

int main() {
    int a = 12;  // Binary: 1100
    int b = 10;  // Binary: 1010

    cout << "a & b  = " << (a & b)  << endl;  // 8   (1000)
    cout << "a | b  = " << (a | b)  << endl;  // 14  (1110)
    cout << "a ^ b  = " << (a ^ b)  << endl;  // 6   (0110)

    return 0;
}

Bitwise NOT

#include <iostream>
using namespace std;

int main() {
    unsigned char x = 0b00001111;  // 15 in decimal

    unsigned char result = ~x;     // Inverts all bits: 11110000
    cout << "~15 = " << (int)result << endl;  // 240

    return 0;
}

Left Shift and Right Shift

#include <iostream>
using namespace std;

int main() {
    int x = 5;  // Binary: 0101

    // Left shift: shifts bits left, fills with 0s
    cout << "5 << 1 = " << (x << 1) << endl;  // 10  (1010)
    cout << "5 << 2 = " << (x << 2) << endl;  // 20  (10100)
    cout << "5 << 3 = " << (x << 3) << endl;  // 40  (101000)

    // Right shift: shifts bits right
    int y = 40;  // Binary: 101000
    cout << "40 >> 1 = " << (y >> 1) << endl;  // 20
    cout << "40 >> 2 = " << (y >> 2) << endl;  // 10
    cout << "40 >> 3 = " << (y >> 3) << endl;  // 5

    return 0;
}

Key Point

Left-shifting a number by n positions is equivalent to multiplying by 2n. Right-shifting by n positions is equivalent to dividing by 2n (for unsigned integers). For example, 5 << 3 equals 5 × 8 = 40.

Practical Bitwise Example: Checking and Setting Flags

#include <iostream>
using namespace std;

int main() {
    // Using bits as flags
    const int READ    = 1;  // 001
    const int WRITE   = 2;  // 010
    const int EXECUTE = 4;  // 100

    int permissions = 0;

    // Set permissions using OR
    permissions |= READ;
    permissions |= WRITE;
    cout << "Permissions: " << permissions << endl;  // 3 (011)

    // Check permissions using AND
    if (permissions & READ) {
        cout << "Read permission granted." << endl;
    }
    if (permissions & EXECUTE) {
        cout << "Execute permission granted." << endl;
    } else {
        cout << "No execute permission." << endl;
    }

    // Remove a permission using AND with NOT
    permissions &= ~WRITE;
    cout << "After removing WRITE: " << permissions << endl;  // 1 (001)

    return 0;
}

Tip

Bitwise operations are extremely fast because they map directly to CPU instructions. They are commonly used for flags, masks, color manipulation, cryptography, and network protocols.

Ternary Operator (? :)

The ternary operator is the only operator in C++ that takes three operands. It is a compact alternative to a simple if-else statement.

Syntax

condition ? expression_if_true : expression_if_false;

Ternary Operator Examples

#include <iostream>
using namespace std;

int main() {
    int age = 20;

    // Simple ternary
    string status = (age >= 18) ? "Adult" : "Minor";
    cout << status << endl;  // Adult

    // Ternary in output
    int score = 85;
    cout << "Grade: " << ((score >= 90) ? "A" : (score >= 80) ? "B" : "C") << endl;

    // Finding the maximum of two numbers
    int a = 15, b = 22;
    int max = (a > b) ? a : b;
    cout << "Max: " << max << endl;  // 22

    // Finding the absolute value
    int x = -7;
    int absVal = (x < 0) ? -x : x;
    cout << "Absolute value: " << absVal << endl;  // 7

    return 0;
}

Tip

While the ternary operator can make code more concise, avoid nesting multiple ternary operators deeply as it reduces readability. For complex conditions, a standard if-else block is preferred.

Operator Precedence and Associativity

When an expression contains multiple operators, C++ uses precedence (priority) and associativity (direction) rules to determine the order of evaluation. Operators with higher precedence are evaluated first.

PrecedenceOperatorDescriptionAssociativity
1 (highest)::Scope resolutionLeft to right
2++ -- (postfix), (), [], ., ->Postfix operatorsLeft to right
3++ -- (prefix), + - (unary), ! ~, (type), *, &, sizeofUnary / PrefixRight to left
4* / %MultiplicativeLeft to right
5+ -AdditiveLeft to right
6<< >>Bitwise shiftLeft to right
7< <= > >=RelationalLeft to right
8== !=EqualityLeft to right
9&Bitwise ANDLeft to right
10^Bitwise XORLeft to right
11|Bitwise ORLeft to right
12&&Logical ANDLeft to right
13||Logical ORLeft to right
14?:Ternary conditionalRight to left
15= += -= *= /= %= <<= >>= &= ^= |=AssignmentRight to left
16 (lowest),CommaLeft to right

Precedence in Action

#include <iostream>
using namespace std;

int main() {
    // Multiplication has higher precedence than addition
    int result1 = 2 + 3 * 4;
    cout << "2 + 3 * 4 = " << result1 << endl;   // 14, not 20

    // Use parentheses to override precedence
    int result2 = (2 + 3) * 4;
    cout << "(2 + 3) * 4 = " << result2 << endl; // 20

    // Comparison vs. assignment
    int x = 5;
    bool check = x > 3 && x < 10;
    // Evaluated as: (x > 3) && (x < 10) = true && true = true
    cout << "5 > 3 && 5 < 10 : " << check << endl; // 1

    // Shift vs. addition (shift has lower precedence!)
    int val = 1 << 2 + 3;
    // Evaluated as: 1 << (2 + 3) = 1 << 5 = 32
    cout << "1 << 2 + 3 = " << val << endl;  // 32

    return 0;
}

Key Point

When in doubt, use parentheses! They make your intentions explicit and prevent subtle bugs. Code like (a + b) * c is clearer than relying on precedence rules alone.

The sizeof Operator

The sizeof operator returns the size (in bytes) of a type or variable at compile time. It is especially useful when working with arrays, dynamic memory, and platform-dependent data types.

#include <iostream>
using namespace std;

int main() {
    // sizeof with data types
    cout << "Size of char:      " << sizeof(char)      << " byte(s)"  << endl;  // 1
    cout << "Size of short:     " << sizeof(short)     << " byte(s)"  << endl;  // 2
    cout << "Size of int:       " << sizeof(int)       << " byte(s)"  << endl;  // 4
    cout << "Size of long:      " << sizeof(long)      << " byte(s)"  << endl;  // 4 or 8
    cout << "Size of long long: " << sizeof(long long) << " byte(s)"  << endl;  // 8
    cout << "Size of float:     " << sizeof(float)     << " byte(s)"  << endl;  // 4
    cout << "Size of double:    " << sizeof(double)    << " byte(s)"  << endl;  // 8
    cout << "Size of bool:      " << sizeof(bool)      << " byte(s)"  << endl;  // 1

    // sizeof with variables
    int arr[10];
    cout << "Size of arr: " << sizeof(arr) << " bytes" << endl;           // 40
    cout << "Number of elements: " << sizeof(arr) / sizeof(arr[0]) << endl; // 10

    return 0;
}

Tip

The expression sizeof(arr) / sizeof(arr[0]) is a classic C/C++ idiom for calculating the number of elements in a statically allocated array. In modern C++ (C++17), you can also use std::size(arr).

Type Casting Operators

Type casting allows you to convert a value from one data type to another. C++ provides both C-style casts and safer, more specific C++ casting operators.

C-Style Cast (Implicit and Explicit)

#include <iostream>
using namespace std;

int main() {
    // Implicit conversion (automatic)
    int a = 10;
    double b = a;  // int is automatically converted to double
    cout << "Implicit: " << b << endl;  // 10.0

    // C-style explicit cast
    double pi = 3.14159;
    int truncated = (int)pi;
    cout << "C-style cast: " << truncated << endl;  // 3

    // Division fix using cast
    int x = 7, y = 2;
    double result = (double)x / y;
    cout << "7 / 2 = " << result << endl;  // 3.5

    return 0;
}

C++ Casting Operators

C++ introduces four named casting operators that provide greater control and safety:

OperatorPurpose
static_cast<T>(expr)General-purpose cast for well-defined conversions (most common)
dynamic_cast<T>(expr)Safe downcasting in polymorphic class hierarchies
const_cast<T>(expr)Add or remove const or volatile qualifiers
reinterpret_cast<T>(expr)Low-level reinterpretation of bit patterns (use with caution)

static_cast Example

#include <iostream>
using namespace std;

int main() {
    // Convert int to double for accurate division
    int a = 7, b = 2;
    double result = static_cast<double>(a) / b;
    cout << "7 / 2 = " << result << endl;  // 3.5

    // Convert double to int (truncation)
    double pi = 3.14159;
    int whole = static_cast<int>(pi);
    cout << "Truncated pi: " << whole << endl;  // 3

    // Convert char to int (ASCII value)
    char ch = 'A';
    int ascii = static_cast<int>(ch);
    cout << "ASCII of 'A': " << ascii << endl;  // 65

    return 0;
}

Key Point

Prefer static_cast over C-style casts. It is checked at compile time, making your intentions clear and catching invalid conversions. C-style casts can silently perform dangerous conversions that static_cast would reject.

Introduction to Operator Overloading

C++ allows you to redefine how operators work for user-defined types (classes and structs). This is called operator overloading. It lets you use natural syntax like a + b with your own objects.

Basic Operator Overloading Example

#include <iostream>
using namespace std;

class Vector2D {
public:
    double x, y;

    Vector2D(double x = 0, double y = 0) : x(x), y(y) {}

    // Overload the + operator
    Vector2D operator+(const Vector2D& other) const {
        return Vector2D(x + other.x, y + other.y);
    }

    // Overload the == operator
    bool operator==(const Vector2D& other) const {
        return x == other.x && y == other.y;
    }

    // Overload the << operator (as a friend function)
    friend ostream& operator<<(ostream& os, const Vector2D& v) {
        os << "(" << v.x << ", " << v.y << ")";
        return os;
    }
};

int main() {
    Vector2D a(3, 4);
    Vector2D b(1, 2);

    Vector2D c = a + b;  // Uses overloaded +
    cout << "a + b = " << c << endl;  // (4, 6)

    cout << "a == b? " << (a == b) << endl;  // 0 (false)

    Vector2D d(3, 4);
    cout << "a == d? " << (a == d) << endl;  // 1 (true)

    return 0;
}

Key Point

Not all operators can be overloaded. You cannot overload ::, ., .*, ?:, or sizeof. Also, operator overloading should make semantic sense—don’t overload + to mean subtraction, or you’ll confuse everyone (including yourself).

Tip

Operator overloading is a powerful feature that will be covered in much greater depth in later lessons on classes and OOP. For now, just understand that C++ lets you customize how operators work with your own types.

Summary

In this lesson, you learned about the comprehensive set of operators available in C++:

  • Arithmetic operators (+, -, *, /, %) for mathematical calculations
  • Assignment operators (=, +=, -=, etc.) for storing and updating values
  • Comparison operators (==, !=, <, >, <=, >=) for comparing values
  • Logical operators (&&, ||, !) for combining boolean conditions
  • Increment/Decrement (++, --) with prefix and postfix behavior
  • Bitwise operators (&, |, ^, ~, <<, >>) for bit-level manipulation
  • Ternary operator (?:) for concise conditionals
  • Operator precedence determines the order of evaluation
  • sizeof for determining the byte size of types
  • Type casting operators (static_cast, etc.) for safe conversions
  • Operator overloading for customizing operators on user-defined types
