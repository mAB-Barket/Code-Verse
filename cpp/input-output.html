7. Input & Output in C++

📚 Lesson 7 of 55 ⏱ 20 min read 🌱 Beginner

The iostream Library

Input and output (I/O) are fundamental to any program — they allow your program to communicate with the outside world. In C++, I/O is handled through a powerful library called <iostream>, which stands for input/output stream.

A stream is an abstraction that represents a flow of data. Think of it like a river: data flows from a source (like the keyboard) to a destination (like the screen). The <iostream> header provides several stream objects that are automatically available when you include it:

  • cout — Character Output: sends data to the standard output (usually the screen/console)
  • cin — Character Input: reads data from the standard input (usually the keyboard)
  • cerr — Character Error: sends error messages to the standard error stream (unbuffered)
  • clog — Character Log: sends log messages to the standard error stream (buffered)
#include <iostream>  // Required for cout, cin, cerr, clog
using namespace std;

int main() {
    cout << "Hello from cout!" << endl;   // Standard output
    cerr << "Error from cerr!" << endl;   // Standard error (unbuffered)
    clog << "Log from clog!" << endl;     // Standard log (buffered)
    return 0;
}

Key Point

You must include <iostream> at the top of your program to use cout, cin, cerr, and clog. These are objects of stream classes defined in the std namespace. Without using namespace std;, you would need to write std::cout, std::cin, etc.

1. Standard Output with cout

The cout object is used to display output on the screen. It uses the insertion operator << (sometimes called "put to") to send data to the output stream. You can think of << as an arrow pointing in the direction data flows — from your data into the console.

Basic Output

#include <iostream>
using namespace std;

int main() {
    // Output a string
    cout << "Hello, World!";

    // Output a number
    cout << 42;

    // Output a character
    cout << 'A';

    // Output a floating-point number
    cout << 3.14;

    return 0;
}

Chaining the Insertion Operator

One of the most powerful features of cout is the ability to chain multiple << operators in a single statement. This lets you combine text, variables, and expressions in one line:

#include <iostream>
using namespace std;

int main() {
    string name = "Alice";
    int age = 25;
    double gpa = 3.95;

    // Chaining multiple values
    cout << "Name: " << name << ", Age: " << age << ", GPA: " << gpa << endl;

    // You can also split across lines for readability
    cout << "Student Info:" << endl
         << "  Name: " << name << endl
         << "  Age:  " << age << endl
         << "  GPA:  " << gpa << endl;

    return 0;
}

endl vs \n

Both endl and \n create a new line, but they behave differently:

  • \n — Simply inserts a newline character. It is faster because it does not flush the buffer.
  • endl — Inserts a newline character and flushes the output buffer, forcing all pending output to be written to the screen immediately.
#include <iostream>
using namespace std;

int main() {
    // Using \n (faster, no flush)
    cout << "Line 1\n";
    cout << "Line 2\n";
    cout << "Line 3\n";

    // Using endl (flushes buffer each time)
    cout << "Line 4" << endl;
    cout << "Line 5" << endl;
    cout << "Line 6" << endl;

    return 0;
}

Key Point

For most programs, use \n instead of endl for better performance. Flushing the buffer with endl after every line is unnecessary in most situations and can significantly slow down programs that produce a lot of output. Use endl only when you specifically need to ensure the output is displayed immediately (e.g., before waiting for input or before a potential crash).

Flushing the Buffer with flush

You can also explicitly flush the output buffer without inserting a newline by using flush:

#include <iostream>
using namespace std;

int main() {
    cout << "Processing...";
    cout.flush();  // Force output to appear immediately

    // Or using the flush manipulator
    cout << "Still working..." << flush;

    cout << " Done!\n";
    return 0;
}

Escape Sequences in Output

Escape sequences let you include special characters in your output that cannot be typed directly:

#include <iostream>
using namespace std;

int main() {
    cout << "Newline: Hello\nWorld" << endl;
    cout << "Tab: Col1\tCol2\tCol3" << endl;
    cout << "Backslash: C:\\Users\\Documents" << endl;
    cout << "Double quote: She said \"Hello\"" << endl;
    cout << "Single quote: It\'s fine" << endl;
    cout << "Carriage return: ABCDE\rXY" << endl;  // Outputs: XYCDE
    cout << "Null character ends here: Hello\0World" << endl;  // Only prints Hello

    return 0;
}

2. Standard Input with cin

The cin object reads data from the keyboard. It uses the extraction operator >> (sometimes called "get from") to extract data from the input stream and store it in a variable. Think of >> as an arrow pointing from the stream into your variable.

Reading Different Data Types

#include <iostream>
using namespace std;

int main() {
    // Reading an integer
    int age;
    cout << "Enter your age: ";
    cin >> age;
    cout << "You are " << age << " years old.\n";

    // Reading a double
    double height;
    cout << "Enter your height (m): ";
    cin >> height;
    cout << "Height: " << height << " meters\n";

    // Reading a character
    char grade;
    cout << "Enter your grade (A-F): ";
    cin >> grade;
    cout << "Grade: " << grade << "\n";

    // Reading a string (single word)
    string name;
    cout << "Enter your first name: ";
    cin >> name;
    cout << "Hello, " << name << "!\n";

    return 0;
}

Chaining the Extraction Operator

Just like cout, you can chain multiple >> operators to read several values in one statement. Values are separated by whitespace (spaces, tabs, or newlines):

#include <iostream>
using namespace std;

int main() {
    int x, y, z;

    cout << "Enter three integers separated by spaces: ";
    cin >> x >> y >> z;

    cout << "You entered: " << x << ", " << y << ", " << z << endl;
    cout << "Sum: " << (x + y + z) << endl;

    return 0;
}

Key Point

The >> operator skips leading whitespace by default and reads until the next whitespace character. This means cin >> name will only read a single word, not a full line. If the user types "John Doe", only "John" will be stored in name. To read an entire line, use getline().

3. Formatting Output

C++ provides powerful tools for formatting output. The <iomanip> header (input/output manipulators) gives you fine-grained control over how numbers, text, and other data appear on screen.

Setting Field Width with setw

The setw(n) manipulator sets the minimum width of the next output field. If the output is shorter than n characters, it will be padded with spaces (by default, right-aligned):

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    cout << setw(10) << "Name" << setw(10) << "Age" << setw(10) << "Score" << endl;
    cout << setw(10) << "Alice" << setw(10) << 25 << setw(10) << 95.5 << endl;
    cout << setw(10) << "Bob" << setw(10) << 30 << setw(10) << 88.3 << endl;
    cout << setw(10) << "Charlie" << setw(10) << 22 << setw(10) << 92.7 << endl;

    return 0;
}

Setting Precision with setprecision

The setprecision(n) manipulator controls how many digits are displayed for floating-point numbers. Its behavior depends on whether fixed or scientific mode is active:

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double pi = 3.14159265358979;
    double big = 123456789.123456;

    // Default precision (6 significant digits total)
    cout << "Default: " << pi << endl;             // 3.14159

    // Set total significant digits
    cout << setprecision(3) << pi << endl;          // 3.14
    cout << setprecision(8) << pi << endl;          // 3.1415927

    // Fixed notation: setprecision sets decimal places
    cout << fixed << setprecision(2) << pi << endl;  // 3.14
    cout << fixed << setprecision(4) << pi << endl;  // 3.1416

    // Scientific notation
    cout << scientific << setprecision(3) << big << endl;  // 1.235e+08

    return 0;
}

Alignment: left, right, and internal

You can control text alignment within a field using left, right (default), and internal:

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    // Right-aligned (default)
    cout << "Right: [" << setw(15) << right << "Hello" << "]" << endl;

    // Left-aligned
    cout << "Left:  [" << setw(15) << left << "Hello" << "]" << endl;

    // Internal alignment (sign at left, value at right)
    cout << "Internal: [" << setw(10) << internal << -42 << "]" << endl;

    return 0;
}
// Output:
// Right: [          Hello]
// Left:  [Hello          ]
// Internal: [-        42]

Key Point

setw() only affects the next output value — it resets after each use. In contrast, setprecision(), fixed, scientific, left, and right are sticky — they remain in effect until changed. Always remember to reset formatting when you are done with it.

4. The iomanip Header

The <iomanip> header provides a rich set of formatting manipulators. Here are the most commonly used ones:

setfill — Custom Padding Character

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    // Fill with dots
    cout << setfill('.') << setw(20) << left << "Item" << "$9.99" << endl;
    cout << setfill('.') << setw(20) << left << "Tax" << "$0.80" << endl;
    cout << setfill('.') << setw(20) << left << "Total" << "$10.79" << endl;

    // Fill with zeros (common for IDs)
    cout << "\nOrder #" << setfill('0') << setw(6) << right << 42 << endl;    // Order #000042
    cout << "Order #" << setfill('0') << setw(6) << right << 1337 << endl;  // Order #001337

    // Reset fill character
    cout << setfill(' ');

    return 0;
}

showpoint, showpos, uppercase

#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    double x = 100.0;
    double y = 3.14;

    // showpoint: always show decimal point and trailing zeros
    cout << showpoint << x << endl;       // 100.000
    cout << noshowpoint << x << endl;     // 100

    // showpos: show + sign for positive numbers
    cout << showpos << 42 << endl;         // +42
    cout << showpos << -7 << endl;         // -7
    cout << noshowpos;

    // uppercase: use uppercase E in scientific, X in hex
    cout << uppercase << scientific << y << endl;   // 3.140000E+00
    cout << nouppercase;

    // hex, oct, dec for integer bases
    int num = 255;
    cout << "Decimal: " << dec << num << endl;      // 255
    cout << "Hex: " << hex << num << endl;          // ff
    cout << "Octal: " << oct << num << endl;        // 377
    cout << "Hex (showbase): " << showbase << hex << num << endl;  // 0xff
    cout << dec << noshowbase;

    return 0;
}

boolalpha

#include <iostream>
using namespace std;

int main() {
    bool isReady = true;
    bool hasError = false;

    // Default: prints 1 and 0
    cout << "Ready: " << isReady << ", Error: " << hasError << endl;
    // Output: Ready: 1, Error: 0

    // boolalpha: prints true and false
    cout << boolalpha;
    cout << "Ready: " << isReady << ", Error: " << hasError << endl;
    // Output: Ready: true, Error: false

    // Turn it off
    cout << noboolalpha;

    return 0;
}

Key Point

The <iomanip> header must be included separately from <iostream>. Manipulators like setw, setprecision, setfill require <iomanip>. However, manipulators like endl, flush, boolalpha, hex, dec, oct, fixed, scientific, left, right, showpos, and showpoint are available directly from <iostream>.

5. String Input with getline()

As mentioned earlier, cin >> reads only a single word because it stops at whitespace. To read an entire line of text (including spaces), use the getline() function:

#include <iostream>
#include <string>
using namespace std;

int main() {
    string fullName;

    cout << "Enter your full name: ";
    getline(cin, fullName);
    cout << "Hello, " << fullName << "!" << endl;

    // Reading multiple lines
    string address;
    cout << "Enter your address: ";
    getline(cin, address);
    cout << "Address: " << address << endl;

    return 0;
}

The cin.ignore() Problem

A very common issue arises when you mix cin >> with getline(). After cin >> reads a value, the newline character (\n) from pressing Enter remains in the input buffer. When getline() is called next, it reads that leftover newline and appears to "skip" the input:

#include <iostream>
#include <string>
using namespace std;

int main() {
    int age;
    string name;

    cout << "Enter your age: ";
    cin >> age;

    // PROBLEM: getline reads the leftover '\n' and gets an empty string!
    cout << "Enter your name: ";
    getline(cin, name);  // This gets skipped!

    cout << "Age: " << age << ", Name: '" << name << "'" << endl;
    // Output: Age: 25, Name: ''

    return 0;
}

The Fix: cin.ignore()

Use cin.ignore() to discard the leftover newline character from the buffer before calling getline():

#include <iostream>
#include <string>
#include <limits>
using namespace std;

int main() {
    int age;
    string name;

    cout << "Enter your age: ";
    cin >> age;

    // Discard leftover newline (and any extra characters)
    cin.ignore(numeric_limits<streamsize>::max(), '\n');

    cout << "Enter your name: ";
    getline(cin, name);  // Now works correctly!

    cout << "Age: " << age << ", Name: " << name << endl;

    return 0;
}

Key Point

Always call cin.ignore() after using cin >> and before using getline(). The safest version is cin.ignore(numeric_limits<streamsize>::max(), '\n'), which discards all characters up to and including the next newline. A simpler alternative is cin.ignore() (with no arguments), which discards exactly one character.

Custom Delimiters with getline()

The getline() function can also read until a custom delimiter character instead of the default newline:

#include <iostream>
#include <string>
using namespace std;

int main() {
    string data;

    cout << "Enter comma-separated values (end with newline):\n";

    // Read until comma
    getline(cin, data, ',');
    cout << "First value: " << data << endl;

    getline(cin, data, ',');
    cout << "Second value: " << data << endl;

    getline(cin, data);  // Read rest of line
    cout << "Third value: " << data << endl;

    return 0;
}
// Input: Apple,Banana,Cherry
// Output:
// First value: Apple
// Second value: Banana
// Third value: Cherry

6. Multiple Inputs and Validation Basics

Real programs often need to read multiple values and ensure the user provides valid input. Here are common patterns for handling multiple inputs:

Reading Multiple Values

#include <iostream>
using namespace std;

int main() {
    // Method 1: Chained extraction
    int day, month, year;
    cout << "Enter date (day month year): ";
    cin >> day >> month >> year;
    cout << "Date: " << day << "/" << month << "/" << year << endl;

    // Method 2: Separate prompts
    double length, width;
    cout << "Enter length: ";
    cin >> length;
    cout << "Enter width: ";
    cin >> width;
    cout << "Area: " << length * width << endl;

    return 0;
}

Basic Input Validation

When the user enters the wrong type of data (e.g., typing "abc" when a number is expected), cin enters a fail state. You can detect and recover from this:

#include <iostream>
#include <limits>
using namespace std;

int main() {
    int number;

    cout << "Enter a number: ";
    cin >> number;

    if (cin.fail()) {
        cout << "That's not a valid number!" << endl;
        cin.clear();   // Clear the error state
        cin.ignore(numeric_limits<streamsize>::max(), '\n');  // Discard bad input
    } else {
        cout << "You entered: " << number << endl;
    }

    return 0;
}

7. Error Handling with cin

Understanding how cin handles errors is crucial for writing robust programs. The input stream has several state flags that indicate its current condition:

  • cin.good() — Returns true if the stream is in a good state (no errors)
  • cin.fail() — Returns true if an input operation failed (e.g., type mismatch)
  • cin.bad() — Returns true if a serious I/O error occurred
  • cin.eof() — Returns true if end-of-file was reached
#include <iostream>
#include <limits>
using namespace std;

int main() {
    int age;

    // Robust input loop
    while (true) {
        cout << "Enter your age (1-150): ";
        cin >> age;

        if (cin.fail()) {
            // Non-numeric input
            cout << "Invalid input! Please enter a number.\n";
            cin.clear();                                          // Reset error flags
            cin.ignore(numeric_limits<streamsize>::max(), '\n');  // Discard bad input
        } else if (age < 1 || age > 150) {
            // Out of range
            cout << "Age must be between 1 and 150.\n";
        } else {
            // Valid input
            break;
        }
    }

    cout << "Your age is: " << age << endl;
    return 0;
}

Stream State Recovery Pattern

#include <iostream>
#include <limits>
using namespace std;

// Reusable function for safe integer input
int getValidInt(const string& prompt) {
    int value;
    while (true) {
        cout << prompt;
        if (cin >> value) {
            cin.ignore(numeric_limits<streamsize>::max(), '\n');
            return value;
        }
        cout << "Invalid input. Please enter a whole number.\n";
        cin.clear();
        cin.ignore(numeric_limits<streamsize>::max(), '\n');
    }
}

double getValidDouble(const string& prompt) {
    double value;
    while (true) {
        cout << prompt;
        if (cin >> value) {
            cin.ignore(numeric_limits<streamsize>::max(), '\n');
            return value;
        }
        cout << "Invalid input. Please enter a number.\n";
        cin.clear();
        cin.ignore(numeric_limits<streamsize>::max(), '\n');
    }
}

int main() {
    int count = getValidInt("How many items? ");
    double price = getValidDouble("Price per item: $");

    cout << "Total: $" << count * price << endl;
    return 0;
}

Key Point

When cin fails, you must do two things to recover: (1) call cin.clear() to reset the error flags, and (2) call cin.ignore() to discard the invalid input from the buffer. If you forget either step, the program will enter an infinite loop because cin will keep trying to read the same bad data.

8. cerr and clog — Error and Log Streams

Besides cout, C++ provides two additional output streams specifically for error and diagnostic messages:

cerr — Standard Error (Unbuffered)

cerr sends output directly to the error stream without buffering. This means the message appears immediately, even if the program crashes right after. It is ideal for error messages:

#include <iostream>
using namespace std;

int main() {
    int divisor;
    cout << "Enter a divisor: ";
    cin >> divisor;

    if (divisor == 0) {
        cerr << "Error: Division by zero is not allowed!" << endl;
        return 1;  // Exit with error code
    }

    cout << "100 / " << divisor << " = " << 100 / divisor << endl;
    return 0;
}

clog — Standard Log (Buffered)

clog is similar to cerr but is buffered, meaning output is collected and written in chunks for efficiency. It is suitable for non-critical log messages:

#include <iostream>
using namespace std;

int main() {
    clog << "Program started." << endl;

    int sum = 0;
    for (int i = 1; i <= 10; i++) {
        sum += i;
        clog << "Step " << i << ": sum = " << sum << endl;
    }

    cout << "Final sum: " << sum << endl;
    clog << "Program finished." << endl;

    return 0;
}

Key Point

Use cout for normal program output, cerr for error messages (unbuffered, so messages appear immediately), and clog for log/diagnostic messages (buffered for efficiency). In a terminal, you can redirect cout and cerr/clog separately. For example: ./program > output.txt 2> errors.txt redirects normal output to one file and errors to another.

9. printf and scanf — C-Style I/O

C++ inherits the printf() and scanf() functions from C. While cout/cin are preferred in modern C++, knowing C-style I/O is useful because you may encounter it in older codebases and it can be more concise for formatted output:

#include <cstdio>  // Required for printf and scanf

int main() {
    // printf - formatted output
    printf("Hello, World!\n");

    // Format specifiers
    int age = 25;
    double gpa = 3.95;
    char grade = 'A';

    printf("Age: %d\n", age);           // %d for int
    printf("GPA: %.2f\n", gpa);         // %.2f for double with 2 decimal places
    printf("Grade: %c\n", grade);       // %c for char
    printf("Name: %s\n", "Alice");      // %s for string

    // Width and alignment
    printf("[%10d]\n", 42);             // Right-aligned in 10 chars: [        42]
    printf("[%-10d]\n", 42);            // Left-aligned in 10 chars:  [42        ]
    printf("[%010d]\n", 42);            // Zero-padded:               [0000000042]

    // scanf - formatted input
    int x;
    printf("Enter a number: ");
    scanf("%d", &x);                    // Note: & (address-of) is required!
    printf("You entered: %d\n", x);

    return 0;
}

Comparison: cout/cin vs printf/scanf

// C++ style (type-safe, extensible)
#include <iostream>
#include <iomanip>
using namespace std;

// C style (concise formatting, but not type-safe)
#include <cstdio>

int main() {
    double price = 9.99;
    int qty = 3;

    // C++ way
    cout << fixed << setprecision(2);
    cout << "Total: $" << price * qty << endl;

    // C way
    printf("Total: $%.2f\n", price * qty);

    // Both produce: Total: $29.97
    return 0;
}

Key Point

cout/cin are type-safe (the compiler checks types automatically) and extensible (you can overload << for custom types). printf/scanf are not type-safe (wrong format specifiers cause undefined behavior) but offer more concise formatting syntax. Modern C++ code should prefer cout/cin, but printf is sometimes used for complex formatting or performance-critical code.

10. Practical Examples

Example 1: Interactive Calculator

#include <iostream>
#include <iomanip>
#include <limits>
using namespace std;

int main() {
    double num1, num2;
    char op;

    cout << "=== Simple Calculator ===\n\n";

    cout << "Enter first number: ";
    cin >> num1;

    cout << "Enter operator (+, -, *, /): ";
    cin >> op;

    cout << "Enter second number: ";
    cin >> num2;

    cout << fixed << setprecision(2);

    switch (op) {
        case '+':
            cout << num1 << " + " << num2 << " = " << (num1 + num2) << endl;
            break;
        case '-':
            cout << num1 << " - " << num2 << " = " << (num1 - num2) << endl;
            break;
        case '*':
            cout << num1 << " * " << num2 << " = " << (num1 * num2) << endl;
            break;
        case '/':
            if (num2 == 0) {
                cerr << "Error: Cannot divide by zero!" << endl;
            } else {
                cout << num1 << " / " << num2 << " = " << (num1 / num2) << endl;
            }
            break;
        default:
            cerr << "Error: Invalid operator '" << op << "'" << endl;
    }

    return 0;
}

Example 4: Formatted Price Table

#include <iostream>
#include <iomanip>
#include <string>
using namespace std;

int main() {
    cout << fixed << setprecision(2);

    // Print header
    cout << left << setw(20) << "Product"
         << right << setw(8) << "Qty"
         << setw(12) << "Price"
         << setw(12) << "Total"
         << endl;
    cout << string(52, '-') << endl;

    // Print items
    double subtotal = 0;

    double p1 = 12.99, p2 = 5.49, p3 = 24.95;
    int q1 = 2, q2 = 5, q3 = 1;

    cout << left << setw(20) << "Widget Pro"
         << right << setw(8) << q1
         << "$" << setw(11) << p1
         << "$" << setw(11) << p1 * q1
         << endl;
    subtotal += p1 * q1;

    cout << left << setw(20) << "Gadget Mini"
         << right << setw(8) << q2
         << "$" << setw(11) << p2
         << "$" << setw(11) << p2 * q2
         << endl;
    subtotal += p2 * q2;

    cout << left << setw(20) << "Thingamajig XL"
         << right << setw(8) << q3
         << "$" << setw(11) << p3
         << "$" << setw(11) << p3 * q3
         << endl;
    subtotal += p3 * q3;

    cout << string(52, '-') << endl;
    cout << left << setw(40) << "Subtotal:"
         << "$" << right << setw(11) << subtotal << endl;

    double tax = subtotal * 0.08;
    cout << left << setw(40) << "Tax (8%):"
         << "$" << right << setw(11) << tax << endl;

    cout << string(52, '=') << endl;
    cout << left << setw(40) << "TOTAL:"
         << "$" << right << setw(11) << subtotal + tax << endl;

    return 0;
}

Summary

In this lesson, you learned everything you need to know about C++ input and output:

  • iostream — The header that provides cout, cin, cerr, and clog
  • cout — Standard output using the << insertion operator, with chaining support
  • cin — Standard input using the >> extraction operator, reads whitespace-delimited tokens
  • endl vs \n — endl flushes the buffer; \n does not (prefer \n)
  • Formatting — setw, setprecision, fixed, scientific, left, right, setfill
  • iomanip — Header for advanced formatting manipulators
  • getline() — Reads a full line of input including spaces
  • cin.ignore() — Clears the input buffer (essential when mixing cin >> and getline())
  • Error handling — cin.fail(), cin.clear(), cin.ignore() for robust input
  • cerr & clog — Error and log output streams
  • printf/scanf — C-style I/O (available but less preferred in modern C++)

Key Point

Mastering I/O is essential for every C++ programmer. Use cout and cin for most tasks, getline() for full-line input, <iomanip> for formatting, and always validate user input with cin.fail() / cin.clear() / cin.ignore(). Prefer \n over endl for performance.
