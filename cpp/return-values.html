18. Return Values

📚 Lesson 18 of 55 ⏱ 15 min read 🌱 Beginner

How the return Statement Works

The return statement does two things: it sends a value back to the caller and immediately exits the function. When a function is called inside an expression, the returned value takes the place of the function call.

In the previous lesson you learned how to pass data into functions through parameters. Now you’ll master how data flows out of functions through return values.

In this lesson you will learn:

  • How the return statement works
  • Returning different data types (int, double, bool, char, string)
  • void functions vs non-void functions
  • Multiple return statements in a single function
  • Returning early from functions
  • Returning expressions vs variables
  • Common mistakes (forgetting return, returning local references)
  • Using returned values in expressions
  • Practical examples

Key Point

Every non-void function must return a value. The type of the returned value must match (or be implicitly convertible to) the function’s declared return type. Forgetting to return a value leads to undefined behavior.

Basic Return Example

The simplest use of return is sending a computed result back to the caller. The returned value replaces the function call in the expression where it appears.

#include <iostream>
using namespace std;

int add(int a, int b) {
    return a + b;  // Returns the sum to the caller
}

int main() {
    int result = add(3, 4);  // result = 7
    cout << "Sum: " << result << endl;
    cout << "Direct: " << add(10, 20) << endl;
    return 0;
}
Output:
Sum: 7
Direct: 30

Notice that add(3, 4) is replaced by the value 7 wherever it appears. You can store the result in a variable or use it directly inside cout.

Returning Different Data Types

Functions can return any data type: int, double, bool, char, string, and more. The return type in the function signature tells the compiler what kind of value to expect.

Returning int

#include <iostream>
using namespace std;

int square(int n) {
    return n * n;
}

int main() {
    cout << "5 squared = " << square(5) << endl;
    return 0;
}
Output:
5 squared = 25

Returning double

#include <iostream>
using namespace std;

double average(double a, double b) {
    return (a + b) / 2.0;
}

int main() {
    cout << "Average: " << average(7.5, 12.3) << endl;
    return 0;
}
Output:
Average: 9.9

Returning bool

#include <iostream>
using namespace std;

bool isEven(int n) {
    return n % 2 == 0;
}

int main() {
    cout << boolalpha;  // Print true/false instead of 1/0
    cout << "4 is even: " << isEven(4) << endl;
    cout << "7 is even: " << isEven(7) << endl;
    return 0;
}
Output:
4 is even: true
7 is even: false

Returning char

#include <iostream>
using namespace std;

char getGrade(int score) {
    if (score >= 90) return 'A';
    if (score >= 80) return 'B';
    if (score >= 70) return 'C';
    if (score >= 60) return 'D';
    return 'F';
}

int main() {
    cout << "Grade: " << getGrade(85) << endl;
    cout << "Grade: " << getGrade(55) << endl;
    return 0;
}
Output:
Grade: B
Grade: F

Returning string

#include <iostream>
#include <string>
using namespace std;

string greet(const string &name) {
    return "Hello, " + name + "!";
}

int main() {
    cout << greet("Alice") << endl;
    string msg = greet("Bob");
    cout << msg << endl;
    return 0;
}
Output:
Hello, Alice!
Hello, Bob!

void Functions vs Non-void Functions

A void function performs an action but does not return a value. A non-void function must return a value of the declared type. You cannot assign or use the result of a void function in an expression.

#include <iostream>
using namespace std;

// void function - performs an action, returns nothing
void printLine() {
    cout << "====================" << endl;
}

// Non-void function - returns a value
int multiply(int a, int b) {
    return a * b;
}

int main() {
    printLine();                         // OK: just performs an action
    int product = multiply(6, 7);        // OK: stores returned value
    cout << "Product: " << product << endl;
    printLine();

    // int x = printLine();  // ERROR: void function has no value to assign
    return 0;
}
Output:
====================
Product: 42
====================

Tip

A void function can still use return; (with no value) to exit early, but it cannot use return value;. Conversely, a non-void function must not use bare return; — it must always return a value.

Multiple Return Statements

A function can contain more than one return statement. The first return that is reached during execution ends the function immediately. This is commonly used with conditional logic.

#include <iostream>
using namespace std;

string checkNumber(int n) {
    if (n > 0) return "Positive";
    if (n < 0) return "Negative";
    return "Zero";
}

int main() {
    cout << checkNumber(10) << endl;
    cout << checkNumber(-3) << endl;
    cout << checkNumber(0) << endl;
    return 0;
}
Output:
Positive
Negative
Zero

Only one return statement executes per call. Once a return is hit, the function exits immediately and no further code in that function runs.

Returning Early from Functions

Early returns are a powerful technique for reducing nesting and handling edge cases at the top of a function. Instead of wrapping your logic in deep if/else blocks, check for special cases first and return immediately.

#include <iostream>
using namespace std;

double safeDivide(double a, double b) {
    if (b == 0) {
        cout << "Error: Division by zero!" << endl;
        return 0;  // Early return to avoid the divide
    }
    return a / b;
}

int main() {
    cout << safeDivide(10, 3) << endl;
    cout << safeDivide(5, 0) << endl;
    return 0;
}
Output:
3.33333
Error: Division by zero!
0

Early returns can also be used in void functions to skip unnecessary work:

#include <iostream>
using namespace std;

void printPositive(int n) {
    if (n <= 0) return;  // Exit immediately if not positive
    cout << "The number is: " << n << endl;
}

int main() {
    printPositive(5);
    printPositive(-2);
    printPositive(0);
    return 0;
}
Output:
The number is: 5

Returning Expressions vs Variables

You can return an expression directly without storing it in a variable first. Both approaches are valid, but returning expressions directly is often more concise.

#include <iostream>
using namespace std;

// Returning a variable
int maxA(int a, int b) {
    int result;
    if (a > b)
        result = a;
    else
        result = b;
    return result;
}

// Returning an expression directly (cleaner)
int maxB(int a, int b) {
    return (a > b) ? a : b;
}

int main() {
    cout << maxA(10, 20) << endl;
    cout << maxB(10, 20) << endl;
    return 0;
}
Output:
20
20

Tip

Returning expressions directly is often preferred because it reduces the number of local variables and makes the function shorter and easier to read. However, storing in a variable first can aid debugging when the computation is complex.

Common Mistakes

1. Forgetting to Return a Value

If a non-void function reaches the end without hitting a return statement, the behavior is undefined. The compiler may warn you, but the code can still compile and produce unpredictable results.

#include <iostream>
using namespace std;

// BAD: Missing return in some paths
int absolute(int n) {
    if (n >= 0) return n;
    // What if n < 0? No return statement!
    // This is UNDEFINED BEHAVIOR
}

int main() {
    cout << absolute(5) << endl;   // Works (returns 5)
    cout << absolute(-3) << endl;  // Undefined behavior!
    return 0;
}

The fix is simple — ensure every code path returns a value:

// GOOD: All paths return a value
int absolute(int n) {
    if (n >= 0) return n;
    return -n;  // Handles the negative case
}

2. Returning a Reference to a Local Variable

Local variables are destroyed when the function ends. Returning a reference or pointer to them creates a dangling reference — the reference points to memory that no longer exists.

// BAD: Returning reference to local variable
int& badFunction() {
    int local = 42;
    return local;   // DANGER: local is destroyed after this!
}

// GOOD: Return by value instead
int goodFunction() {
    int local = 42;
    return local;   // Returns a COPY of the value
}

Key Point

Never return a reference or pointer to a local variable. The local variable is destroyed when the function exits, making the reference invalid. Always return by value for local data, or return references only to data that outlives the function (like class members or global variables).

3. Ignoring the Return Value

If a function returns a value that you don’t use, the value is simply discarded. While this isn’t an error, it can indicate a logic bug — especially if the return value signals an error condition.

#include <iostream>
using namespace std;

int computeSum(int a, int b) {
    return a + b;
}

int main() {
    computeSum(3, 4);  // Return value is discarded (usually a mistake)
    int sum = computeSum(3, 4);  // Correct: capture the returned value
    cout << sum << endl;
    return 0;
}

Using Return Values in Expressions

Returned values can be used anywhere a value of that type is expected: in assignments, arithmetic, comparisons, function arguments, and more.

#include <iostream>
using namespace std;

int square(int n) { return n * n; }
int cube(int n) { return n * n * n; }
bool isPositive(int n) { return n > 0; }

int main() {
    // Use in arithmetic
    int sum = square(3) + cube(2);
    cout << "9 + 8 = " << sum << endl;

    // Use in conditions
    if (isPositive(sum)) {
        cout << sum << " is positive" << endl;
    }

    // Use as argument to another function
    cout << "square(cube(2)) = " << square(cube(2)) << endl;

    // Use in cout directly
    cout << "4 squared = " << square(4) << endl;

    return 0;
}
Output:
9 + 8 = 17
17 is positive
square(cube(2)) = 64
4 squared = 16

Tip

Nesting function calls like square(cube(2)) is powerful but can reduce readability. If nesting gets deep, consider storing intermediate results in well-named variables.

Practical Examples

Example 1: Temperature Converter

#include <iostream>
using namespace std;

double celsiusToFahrenheit(double celsius) {
    return celsius * 9.0 / 5.0 + 32.0;
}

double fahrenheitToCelsius(double fahrenheit) {
    return (fahrenheit - 32.0) * 5.0 / 9.0;
}

int main() {
    double tempC = 100.0;
    double tempF = celsiusToFahrenheit(tempC);
    cout << tempC << " C = " << tempF << " F" << endl;

    double backToC = fahrenheitToCelsius(tempF);
    cout << tempF << " F = " << backToC << " C" << endl;
    return 0;
}
Output:
100 C = 212 F
212 F = 100 C

Example 2: Factorial Function

#include <iostream>
using namespace std;

long long factorial(int n) {
    if (n <= 0) return 1;  // Base case / edge case
    long long result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

int main() {
    for (int i = 0; i <= 10; i++) {
        cout << i << "! = " << factorial(i) << endl;
    }
    return 0;
}
Output:
0! = 1
1! = 1
2! = 2
3! = 6
4! = 24
5! = 120
6! = 720
7! = 5040
8! = 40320
9! = 362880
10! = 3628800

Example 3: Min of Three Values

#include <iostream>
using namespace std;

int minOfTwo(int a, int b) {
    return (a < b) ? a : b;
}

int minOfThree(int a, int b, int c) {
    return minOfTwo(minOfTwo(a, b), c);
}

int main() {
    cout << "Min of 5, 2, 8: " << minOfThree(5, 2, 8) << endl;
    cout << "Min of 1, 1, 1: " << minOfThree(1, 1, 1) << endl;
    cout << "Min of 9, 3, 6: " << minOfThree(9, 3, 6) << endl;
    return 0;
}
Output:
Min of 5, 2, 8: 2
Min of 1, 1, 1: 1
Min of 9, 3, 6: 3

Example 4: Password Strength Checker

#include <iostream>
#include <string>
using namespace std;

bool hasDigit(const string &s) {
    for (char c : s) {
        if (c >= '0' && c <= '9') return true;
    }
    return false;
}

bool hasUpper(const string &s) {
    for (char c : s) {
        if (c >= 'A' && c <= 'Z') return true;
    }
    return false;
}

string checkStrength(const string &password) {
    if (password.length() < 6) return "Weak";
    if (!hasDigit(password)) return "Medium";
    if (!hasUpper(password)) return "Medium";
    return "Strong";
}

int main() {
    cout << checkStrength("abc") << endl;
    cout << checkStrength("abcdef") << endl;
    cout << checkStrength("abcdef1") << endl;
    cout << checkStrength("Abcdef1") << endl;
    return 0;
}
Output:
Weak
Medium
Medium
Strong

Example 5: Power Function

#include <iostream>
using namespace std;

double power(double base, int exponent) {
    if (exponent == 0) return 1.0;

    double result = 1.0;
    bool negative = exponent < 0;
    if (negative) exponent = -exponent;

    for (int i = 0; i < exponent; i++) {
        result *= base;
    }

    return negative ? 1.0 / result : result;
}

int main() {
    cout << "2^10 = " << power(2, 10) << endl;
    cout << "5^0 = " << power(5, 0) << endl;
    cout << "2^-3 = " << power(2, -3) << endl;
    return 0;
}
Output:
2^10 = 1024
5^0 = 1
2^-3 = 0.125

Summary

  • The return statement sends a value back to the caller and exits the function immediately.
  • Functions can return int, double, bool, char, string, and any other type.
  • void functions perform actions but return no value; non-void functions must return a value.
  • A function can have multiple return statements — only the first one reached executes.
  • Early returns reduce nesting and handle edge cases cleanly.
  • You can return expressions directly without storing them in variables first.
  • Never return a reference or pointer to a local variable — it becomes a dangling reference.
  • Every code path in a non-void function must end with a return statement to avoid undefined behavior.
  • Returned values can be used in assignments, arithmetic, conditions, and as arguments to other functions.
