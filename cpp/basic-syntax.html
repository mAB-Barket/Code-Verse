4. C++ Basic Syntax

📚 Lesson 4 of 55 ⏱ 12 min read 🌱 Beginner

Understanding C++ Program Structure

Every C++ program follows a well-defined structure. Before you can write complex applications, you need to understand the fundamental building blocks that make up a C++ program. Syntax is the set of rules that defines how a C++ program is written and interpreted by the compiler. Just as English has grammar rules, C++ has syntax rules that must be followed precisely for the compiler to understand your code.

A typical C++ program consists of the following components, in order:

  1. Preprocessor directives — instructions for the preprocessor (e.g., #include)
  2. Namespace declarations — such as using namespace std;
  3. Function definitions — including the main() function
  4. Statements — the actual instructions inside functions
// 1. Preprocessor directive
#include <iostream>

// 2. Namespace declaration
using namespace std;

// 3. Main function definition
int main() {
    // 4. Statements
    cout << "Hello, World!" << endl;
    return 0;
}

Key Point

C++ syntax is strict — even a single missing semicolon or misplaced brace can prevent your program from compiling. Understanding the basic syntax rules is essential before moving on to more advanced topics like variables, functions, and classes.

Preprocessor Directives

Preprocessor directives are special instructions that begin with the # symbol. They are processed before the actual compilation of your code begins. The preprocessor is a separate program that runs first and transforms your source code before the compiler sees it.

The most common preprocessor directive is #include, which tells the preprocessor to insert the contents of another file into your source code. This is how you gain access to libraries and their functions.

// Include standard I/O library
#include <iostream>

// Include string library
#include <string>

// Include math library
#include <cmath>

// Include a custom header file
#include "myheader.h"

There are two forms of #include:

  • #include <filename> — searches for the file in the system include directories (used for standard library headers).
  • #include "filename" — searches for the file in the current directory first, then in the system directories (used for your own header files).

Other common preprocessor directives include:

// Define a constant
#define PI 3.14159265

// Conditional compilation
#ifdef DEBUG
    cout << "Debug mode is ON" << endl;
#endif

// Include guard (prevents double inclusion)
#ifndef MYHEADER_H
#define MYHEADER_H
// header contents here
#endif

Key Point

Preprocessor directives do not end with a semicolon. They are not C++ statements — they are instructions for the preprocessor. Adding a semicolon after #include <iostream>; is a common beginner mistake and can cause unexpected errors.

The main() Function

Every C++ program must have exactly one function named main(). This is the entry point of the program — the first function that the operating system calls when your program starts executing. Without main(), the linker will produce an error because it does not know where to begin execution.

// The simplest valid C++ program
int main() {
    return 0;
}

The int before main() specifies the return type. The function returns an integer value to the operating system: 0 means success, and any non-zero value indicates an error. In C++11 and later, if you omit the return 0; statement, the compiler implicitly returns 0.

The main() function can also accept command-line arguments:

// main with command-line arguments
int main(int argc, char* argv[]) {
    // argc = number of arguments
    // argv = array of argument strings
    return 0;
}

Statements and Semicolons

A statement is an instruction that tells the computer to perform a specific action. In C++, most statements must end with a semicolon (;). The semicolon is the statement terminator — it tells the compiler where one statement ends.

#include <iostream>
using namespace std;

int main() {
    int x = 10;              // declaration statement
    cout << x << endl;       // expression statement
    x = x + 5;               // assignment statement
    return 0;                 // return statement
}

There are several types of statements in C++:

  • Declaration statements: Declare variables or functions — int age = 25;
  • Expression statements: Evaluate an expression — cout << "Hello"; or x = x + 1;
  • Compound statements (blocks): Groups of statements enclosed in braces { }
  • Selection statements: if, else, switch
  • Iteration statements: for, while, do-while
  • Jump statements: return, break, continue, goto

Key Point

Forgetting a semicolon is one of the most common mistakes in C++. The compiler error message may sometimes point to the line after the missing semicolon, which can be confusing for beginners. Always check the line above when you get an unexpected error.

Blocks and Curly Braces

A block (also called a compound statement) is a group of zero or more statements enclosed in curly braces { }. Blocks define the scope of variables and group statements together. The body of every function is a block.

#include <iostream>
using namespace std;

int main() {                    // opening brace of main block
    int a = 10;

    {                            // inner block (nested block)
        int b = 20;
        cout << a + b << endl;  // can access both a and b
    }                            // b goes out of scope here

    // cout << b;               // ERROR: b is not accessible here

    return 0;
}                                // closing brace of main block

Important rules about blocks:

  • Variables declared inside a block are local to that block and cannot be accessed outside of it.
  • Blocks can be nested — you can have blocks inside other blocks.
  • An inner block can access variables from outer blocks, but not vice versa.
  • Curly braces do not require a semicolon after the closing brace (except for class and struct definitions).

Whitespace and Indentation

C++ is a free-form language, which means the compiler ignores most whitespace (spaces, tabs, and newlines). You could technically write an entire program on a single line:

// This compiles and runs perfectly (but is terrible to read!)
#include <iostream>
int main(){std::cout<<"Hello";return 0;}

However, just because you can does not mean you should. Proper whitespace and indentation make your code readable, maintainable, and professional. Here is the same program written with proper formatting:

#include <iostream>

int main() {
    std::cout << "Hello";
    return 0;
}

Common indentation conventions:

  • Use consistent indentation — either 4 spaces or 1 tab per level (never mix them).
  • Indent the contents of every block (between { and }).
  • Align closing braces with the statement or keyword that opened the block.
  • Use blank lines to separate logical sections of code.

Key Point

Although C++ ignores extra whitespace, well-formatted code is essential for collaboration and maintenance. Most teams and companies have coding style guides that specify indentation rules. Always write code as if the person who will maintain it is a violent psychopath who knows where you live.

Identifiers and Naming Rules

An identifier is a name given to a variable, function, class, or any other user-defined item. C++ has strict rules for what constitutes a valid identifier:

  • Can contain letters (a-z, A-Z), digits (0-9), and underscores (_).
  • Must begin with a letter or an underscore — cannot start with a digit.
  • Cannot be a C++ keyword (reserved word).
  • Are case-sensitive — myVar and MyVar are different identifiers.
  • Cannot contain spaces or special characters like @, $, %.
  • Should not start with double underscores (__) or underscore followed by a capital letter (_A) — these are reserved for the compiler and standard library.
// VALID identifiers
int age;
int _count;
int totalScore;
int player1Score;
int MAX_SIZE;
int myLongVariableName;

// INVALID identifiers
// int 2ndPlace;       // cannot start with a digit
// int my-var;         // hyphens not allowed
// int my var;         // spaces not allowed
// int class;          // 'class' is a reserved keyword
// int hello@world;    // special characters not allowed

Common naming conventions in C++:

  • camelCase for variables and functions: studentName, calculateTotal()
  • PascalCase for classes and structs: StudentRecord, LinkedList
  • UPPER_SNAKE_CASE for constants and macros: MAX_SIZE, PI
  • snake_case is also common (used by the C++ standard library): my_variable, push_back

You do not need to memorize all of these right now. As you progress through the lessons, you will learn what most of these keywords do. For now, just remember that you cannot use any of these words as variable names, function names, or class names.

Case Sensitivity

C++ is a case-sensitive language. This means that uppercase and lowercase letters are treated as completely different characters. The identifier count is not the same as Count, COUNT, or cOuNt.

#include <iostream>
using namespace std;

int main() {
    int score = 100;
    int Score = 200;
    int SCORE = 300;

    // These are three DIFFERENT variables
    cout << score << endl;   // 100
    cout << Score << endl;   // 200
    cout << SCORE << endl;   // 300

    return 0;
}

This case sensitivity extends to everything in C++:

  • int is a keyword, but Int, INT, and iNt are not.
  • main is the entry point, but Main or MAIN will not be recognized.
  • cout is the output stream, but Cout or COUT will cause errors.
  • return is a keyword, but Return or RETURN are just ordinary identifiers.

Key Point

Because C++ is case-sensitive, always be consistent with your naming. If you declare a variable as playerScore, you must use that exact capitalization every time you refer to it. Writing playerscore or PlayerScore would refer to a different variable (or cause an "undeclared identifier" error).

The std Namespace and Scope Resolution Operator ::

A namespace is a container that groups related identifiers (names) together to prevent naming conflicts. The C++ Standard Library puts all of its components (like cout, cin, string, endl) inside a namespace called std (short for "standard").

To access something inside a namespace, you use the scope resolution operator :::

#include <iostream>

int main() {
    // Using the scope resolution operator
    std::cout << "Hello from std namespace!" << std::endl;
    return 0;
}

Writing std:: before every standard library name can be tedious, so many programs use the using directive:

#include <iostream>
using namespace std;   // bring all std names into scope

int main() {
    cout << "No need for std:: prefix now!" << endl;
    return 0;
}

You can also selectively import specific names from a namespace:

#include <iostream>

// Only import cout and endl
using std::cout;
using std::endl;

int main() {
    cout << "Only cout and endl are imported." << endl;
    // std::cin is still required for cin usage
    return 0;
}

Key Point

Using using namespace std; is convenient for small programs and tutorials, but it is considered bad practice in larger projects and header files because it can cause name collisions. In professional code, prefer using std::cout or selective using std::cout; declarations.

Basic Data Type Overview

C++ provides several built-in (fundamental) data types. Here is a quick overview of the most commonly used ones:

#include <iostream>
#include <string>
using namespace std;

int main() {
    int age = 25;             // whole numbers (integers)
    double price = 19.99;     // decimal numbers (floating-point)
    char grade = 'A';         // single character
    bool isPassed = true;     // boolean (true or false)
    string name = "Alice";    // text string (from <string> library)

    cout << "Name: " << name << endl;
    cout << "Age: " << age << endl;
    cout << "Price: $" << price << endl;
    cout << "Grade: " << grade << endl;
    cout << "Passed: " << isPassed << endl;

    return 0;
}

Here is a quick reference table of the fundamental data types:

  • int — stores whole numbers (e.g., 42, -7, 0)
  • double — stores decimal (floating-point) numbers (e.g., 3.14, -0.001)
  • float — stores decimal numbers with less precision than double (e.g., 3.14f)
  • char — stores a single character (e.g., 'A', '9', '$')
  • bool — stores true or false
  • string — stores a sequence of characters (requires #include <string>)
  • void — represents the absence of a type (used with functions that return nothing)

Variable Declaration and Initialization

In C++, you must declare a variable before you can use it. Declaration tells the compiler the variable's name and type. You can also initialize a variable (give it an initial value) at the time of declaration.

// Declaration without initialization
int count;            // count has an indeterminate value

// Declaration with initialization (copy initialization)
int score = 100;

// Direct initialization (using parentheses)
int total(250);

// Uniform initialization / brace initialization (C++11)
int value{42};

// Multiple variable declarations
int a = 1, b = 2, c = 3;

// Constant variable (value cannot be changed)
const double PI = 3.14159265;

// Auto type deduction (C++11)
auto x = 10;         // compiler deduces x as int
auto y = 3.14;       // compiler deduces y as double
auto z = 'A';        // compiler deduces z as char

There are important differences between the initialization styles:

  • Copy initialization (int x = 5;) — the traditional C-style initialization.
  • Direct initialization (int x(5);) — uses parentheses, common in class construction.
  • Brace initialization (int x{5};) — the modern C++11 style, prevents narrowing conversions and is generally recommended.
#include <iostream>
using namespace std;

int main() {
    // Brace initialization prevents narrowing
    // int x{3.14};  // ERROR: narrowing conversion from double to int
    int x = 3.14;    // OK but x becomes 3 (truncated, not rounded)

    cout << x << endl; // Output: 3

    return 0;
}

Expression, Declaration, and Compound Statements

C++ categorizes statements into several types. Understanding these categories helps you write correct and organized code.

Expression Statements

An expression statement is an expression followed by a semicolon. Expressions can involve arithmetic, assignments, function calls, or any computation:

x = 5;                    // assignment expression statement
x + y;                    // valid but useless (result is discarded)
cout << "Hi";             // function call expression statement
x++;                      // increment expression statement
myFunction(a, b);         // function call expression statement

Declaration Statements

Declaration statements introduce new identifiers (variables, constants, types) into the program:

int x = 10;               // declares and initializes x
double pi = 3.14;         // declares and initializes pi
const int MAX = 100;      // declares a constant
string greeting = "Hi";   // declares a string variable

Compound Statements (Blocks)

A compound statement is a group of statements enclosed in curly braces. It is treated as a single statement by the compiler:

{
    int temp = a;
    a = b;
    b = temp;
}  // this entire block is one compound statement

Header Files and #include

Header files contain declarations of functions, classes, and other resources that your program needs. By including a header file, you tell the compiler about the existence of these resources so you can use them in your code.

// Standard C++ library headers (no .h extension)
#include <iostream>     // cout, cin, endl
#include <string>       // string class
#include <vector>       // vector container
#include <cmath>        // math functions (sqrt, pow, etc.)
#include <fstream>      // file I/O
#include <algorithm>    // sort, find, etc.
#include <iomanip>      // output formatting (setw, setprecision)

// C-style headers (legacy, available in C++)
#include <cstdio>       // printf, scanf (C++ version of stdio.h)
#include <cstdlib>      // rand, srand, exit
#include <cstring>      // C-string functions (strlen, strcpy)

The standard C++ headers use angle brackets and do not have a .h extension. The old C headers (like stdio.h) have been renamed with a c prefix and no .h (e.g., cstdio) for use in C++, and their contents are placed in the std namespace.

Putting It All Together: A Complete Example

Here is a complete example that demonstrates many of the basic syntax elements we have covered in this lesson:

#include <iostream>    // for cout, cin, endl
#include <string>      // for string class
using namespace std;

// A constant using const
const double TAX_RATE = 0.08;

int main() {
    // Variable declarations and initialization
    string productName;
    double price;
    int quantity;

    // User input
    cout << "Enter product name: ";
    getline(cin, productName);

    cout << "Enter price: $";
    cin >> price;

    cout << "Enter quantity: ";
    cin >> quantity;

    // Calculations using a block
    {
        double subtotal = price * quantity;
        double tax = subtotal * TAX_RATE;
        double total = subtotal + tax;

        // Output results
        cout << "\n--- Receipt ---" << endl;
        cout << "Product: " << productName << endl;
        cout << "Price: $" << price << endl;
        cout << "Quantity: " << quantity << endl;
        cout << "Subtotal: $" << subtotal << endl;
        cout << "Tax: $" << tax << endl;
        cout << "Total: $" << total << endl;
    }

    return 0;
}

Multiple Statements on One Line and Multiple Lines for One Statement

C++ allows you to place multiple statements on a single line, and you can also split a single statement across multiple lines:

// Multiple statements on one line (legal but discouraged)
int a = 5; int b = 10; int c = a + b;

// One statement across multiple lines (perfectly fine)
cout << "The sum of "
     << a
     << " and "
     << b
     << " is "
     << c
     << endl;

While both are technically valid, best practices recommend:

  • Put only one statement per line for clarity.
  • Split long statements across multiple lines for readability, aligning continuation lines.

Common Syntax Errors

Here are some of the most frequent syntax mistakes beginners make in C++:

// ERROR 1: Missing semicolon
int x = 10   // ERROR: expected ';' at end of declaration

// ERROR 2: Mismatched braces
int main() {
    cout << "Hello";
// Missing closing brace!

// ERROR 3: Using a keyword as an identifier
int class = 5;   // ERROR: 'class' is a reserved keyword

// ERROR 4: Starting identifier with a digit
int 1stPlace;    // ERROR: identifier cannot start with a digit

// ERROR 5: Wrong case for keywords
Int main() {     // ERROR: 'Int' is not the same as 'int'
    Return 0;    // ERROR: 'Return' is not the same as 'return'
}

// ERROR 6: Semicolon after #include
#include <iostream>;  // Unexpected semicolon

// ERROR 7: Using undeclared variable
cout << myVar;  // ERROR if myVar was never declared

Summary

In this lesson, you learned the fundamental syntax rules of C++:

  • C++ programs consist of preprocessor directives, namespace declarations, function definitions, and statements.
  • Preprocessor directives start with # and do not end with semicolons.
  • Every C++ program must have exactly one main() function.
  • Statements end with semicolons; blocks use curly braces { }.
  • C++ is a free-form language — whitespace is ignored, but proper indentation is essential for readability.
  • Identifiers must start with a letter or underscore, are case-sensitive, and cannot be keywords.
  • The std namespace contains all standard library components; use :: to access them.
  • C++ has several fundamental data types: int, double, char, bool, and string.
  • Variables must be declared before use and can be initialized using copy, direct, or brace initialization.
  • Header files are included with #include to access library functions and classes.

Key Point

Mastering the basic syntax of C++ is the foundation for everything that follows. Every concept you learn — variables, functions, classes, templates — builds on these fundamental rules. Take the time to understand each rule, practice writing small programs, and pay careful attention to the compiler's error messages when things go wrong.

Practice Exercises

Try these exercises to reinforce your understanding of C++ basic syntax:

  1. Fix the Errors: Given a program with 5 syntax errors (missing semicolons, wrong cases, invalid identifiers), find and fix all of them.
  2. Identifier Check: Write down 10 identifiers and determine which ones are valid and which are invalid according to C++ naming rules.
  3. Reformat Code: Take a poorly formatted C++ program written on a single line and reformat it with proper indentation and spacing.
  4. Namespace Practice: Rewrite a program that uses using namespace std; to instead use explicit std:: prefixes everywhere.
  5. Data Types: Write a program that declares at least one variable of each basic data type (int, double, char, bool, string) and prints their values.
  6. Initialization Styles: Write a program demonstrating all three initialization styles (copy, direct, brace) and print the results.
