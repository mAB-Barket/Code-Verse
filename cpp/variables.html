5. Variables & Constants

📚 Lesson 5 of 55 ⏱ 20 min read 🌱 Beginner

What Are Variables?

A variable is a named storage location in the computer's memory that holds a value. Think of a variable like a labeled box: the label is the variable's name, and the content of the box is the value stored in it. Variables allow your program to store, retrieve, and manipulate data during execution.

In C++, every variable has three key properties:

  • Name — A unique identifier you choose (e.g., age, temperature, studentName)
  • Type — The kind of data it stores (e.g., int, double, char, string)
  • Value — The actual data stored in the variable
#include <iostream>
using namespace std;

int main() {
    // Declaring a variable
    int age;          // Declaration: creates a variable of type int

    // Assigning a value
    age = 25;         // Assignment: stores 25 in the variable age

    // Using the variable
    cout << "Age: " << age << endl;  // Output: Age: 25

    return 0;
}

Key Point

A variable is a named memory location that stores data. Every C++ variable must be declared with a type before use. Declaration tells the compiler what kind of data the variable will hold and how much memory to allocate for it.

Declaring Variables

In C++, you must declare a variable before you can use it. Declaration tells the compiler the variable's name and type. There are several ways to declare variables:

Method 1: Declaration Then Assignment

int age;           // Declaration
age = 25;          // Assignment (later)

Method 2: Declaration With Initialization

int age = 25;              // Copy initialization
int height(180);           // Direct initialization
int weight{75};            // Brace initialization (C++11, preferred)
int score = {100};         // Copy-list initialization

Method 3: Multiple Variables

int x = 5, y = 10, z = 15;   // Multiple variables of the same type
double pi = 3.14, e = 2.71;  // Multiple doubles

Method 4: Using auto (C++11)

auto name = "Alice";     // auto deduces type as const char*
auto count = 42;         // auto deduces type as int
auto price = 9.99;       // auto deduces type as double
auto letter = 'A';       // auto deduces type as char

Key Point

C++11 introduced brace initialization int x{5}; which is the safest form because it prevents narrowing conversions (e.g., assigning a double to an int). The auto keyword lets the compiler deduce the type automatically from the initializer.

Variable Naming Rules

C++ has strict rules for naming variables:

  • Must start with a letter (a-z, A-Z) or underscore (_)
  • Can contain letters, digits (0-9), and underscores
  • Cannot start with a digit
  • Cannot use C++ reserved keywords (like int, return, class, if)
  • Case-sensitive: Age, age, and AGE are three different variables
  • No spaces allowed in variable names
// VALID variable names
int age = 25;
int _count = 0;
int student_name = 1;
int myAge = 30;
int MAX_VALUE = 100;
int item2 = 50;

// INVALID variable names
// int 2ndItem = 10;     // Cannot start with a digit
// int my age = 20;      // No spaces allowed
// int class = 5;        // 'class' is a reserved keyword
// int my-var = 3;       // Hyphens not allowed

Naming Conventions

While C++ doesn't enforce a naming style, the community follows common conventions:

  • camelCase for variables and functions: studentAge, totalScore, calculateArea
  • PascalCase for classes and structs: StudentRecord, GameEngine
  • UPPER_SNAKE_CASE for constants and macros: MAX_SIZE, PI, DEFAULT_COLOR
  • snake_case is also common (used by the C++ Standard Library): student_age, total_score

Key Point

Choose descriptive, meaningful names for your variables. A name like studentAge is far better than x or sa. Good naming makes code self-documenting and easier to understand.

Constants

A constant is a variable whose value cannot be changed after initialization. Constants are used to represent values that should remain fixed throughout the program, like mathematical constants (PI), configuration values, or limits.

The const Keyword

The most common way to create a constant in C++ is using the const keyword:

#include <iostream>
using namespace std;

int main() {
    const double PI = 3.14159265358979;
    const int MAX_STUDENTS = 30;
    const string SCHOOL_NAME = "CodeVerse Academy";

    cout << "PI = " << PI << endl;
    cout << "Max students: " << MAX_STUDENTS << endl;
    cout << "School: " << SCHOOL_NAME << endl;

    // PI = 3.0;  // ERROR! Cannot modify a const variable
    // MAX_STUDENTS = 50;  // ERROR!

    return 0;
}

constexpr (C++11)

The constexpr keyword declares a constant whose value is computed at compile time. This is more restrictive than const but can enable optimizations:

#include <iostream>
using namespace std;

constexpr double PI = 3.14159265358979;
constexpr int SQUARE(int x) { return x * x; }

int main() {
    constexpr int SIZE = 100;
    constexpr double AREA = PI * SQUARE(5);  // Computed at compile time

    cout << "Area: " << AREA << endl;  // Output: Area: 78.5398

    // These are evaluated at compile time, producing faster code
    int arr[SIZE];  // Works because SIZE is a compile-time constant

    return 0;
}

#define Macros (Legacy)

In older C++ code, you may see #define used for constants. This is a preprocessor directive that performs text substitution:

#define PI 3.14159
#define MAX_SIZE 100

// Modern C++ discourages #define for constants.
// Prefer const or constexpr instead.
// #define has no type safety and can cause subtle bugs.

Key Point

Use const for runtime constants and constexpr for compile-time constants. Avoid #define for constants in modern C++ because it lacks type safety. Constants should be named in UPPER_SNAKE_CASE by convention.

Variable Scope

Scope determines where a variable is accessible in your code. C++ has several types of scope:

Local Scope (Block Scope)

Variables declared inside a function or block {} are local to that block. They are created when the block is entered and destroyed when the block is exited:

#include <iostream>
using namespace std;

int main() {
    int x = 10;  // x is local to main()

    {
        int y = 20;  // y is local to this inner block
        cout << x << endl;  // OK: x is visible here
        cout << y << endl;  // OK: y is visible here
    }

    cout << x << endl;  // OK: x is still in scope
    // cout << y << endl;  // ERROR: y is out of scope

    return 0;
}

Global Scope

Variables declared outside all functions have global scope. They are accessible from any function in the file:

#include <iostream>
using namespace std;

int globalVar = 100;  // Global variable

void display() {
    cout << "Global in display(): " << globalVar << endl;
}

int main() {
    cout << "Global in main(): " << globalVar << endl;
    display();

    int globalVar = 50;  // Local variable SHADOWS the global one
    cout << "Local in main(): " << globalVar << endl;    // 50
    cout << "Global via :: " << ::globalVar << endl;     // 100

    return 0;
}

Static Variables

A static local variable retains its value between function calls:

#include <iostream>
using namespace std;

void counter() {
    static int count = 0;  // Initialized only once
    count++;
    cout << "Count: " << count << endl;
}

int main() {
    counter();  // Count: 1
    counter();  // Count: 2
    counter();  // Count: 3
    return 0;
}

Key Point

Local variables exist only within their block {}. Global variables are accessible everywhere but should be avoided because they make code harder to maintain and debug. Use the scope resolution operator :: to access a global variable shadowed by a local one. Static local variables persist across function calls.

Type Conversion

Sometimes you need to convert a value from one type to another. C++ supports both implicit (automatic) and explicit (manual) conversion.

Implicit Conversion (Coercion)

The compiler automatically converts compatible types when it is safe:

#include <iostream>
using namespace std;

int main() {
    int myInt = 42;
    double myDouble = myInt;  // int to double (safe, no data loss)
    cout << myDouble << endl; // 42.0

    double pi = 3.14159;
    int truncated = pi;  // double to int (narrowing, loses decimal part)
    cout << truncated << endl; // 3 (truncated, NOT rounded)

    char letter = 'A';
    int ascii = letter;  // char to int (gives ASCII value)
    cout << ascii << endl; // 65

    return 0;
}

Explicit Conversion (Casting)

You can force a type conversion using casts:

#include <iostream>
using namespace std;

int main() {
    double price = 9.99;

    // C-style cast (works but not recommended in C++)
    int rounded1 = (int)price;

    // C++ functional cast
    int rounded2 = int(price);

    // static_cast (preferred in modern C++)
    int rounded3 = static_cast<int>(price);

    cout << rounded1 << endl;  // 9
    cout << rounded2 << endl;  // 9
    cout << rounded3 << endl;  // 9

    // Integer division vs. floating-point division
    int a = 7, b = 2;
    cout << "Integer div: " << a / b << endl;  // 3
    cout << "Float div: " << static_cast<double>(a) / b << endl;  // 3.5

    return 0;
}

Key Point

Use static_cast<type>(value) for explicit conversions in modern C++. Be careful with narrowing conversions (e.g., double to int) as they lose data. Brace initialization int x{3.5}; will produce a compiler error for narrowing conversions, catching bugs early.

Practical Example: Student Record

#include <iostream>
#include <string>
using namespace std;

int main() {
    // Student information using different variable types
    const string SCHOOL = "CodeVerse Academy";
    string studentName = "Bob Smith";
    int studentId = 12345;
    int age = 20;
    double gpa = 3.85;
    char grade = 'A';
    bool isHonorsStudent = true;
    static int totalStudents = 0;

    totalStudents++;

    cout << "===== " << SCHOOL << " =====" << endl;
    cout << "Student: " << studentName << endl;
    cout << "ID: " << studentId << endl;
    cout << "Age: " << age << endl;
    cout << "GPA: " << gpa << endl;
    cout << "Grade: " << grade << endl;
    cout << "Honors: " << boolalpha << isHonorsStudent << endl;
    cout << "Total Students: " << totalStudents << endl;

    // Modify variables
    age++;  // increment age
    gpa = 3.90;
    grade = 'A';

    cout << "\nAfter updates:" << endl;
    cout << "Age: " << age << endl;
    cout << "GPA: " << gpa << endl;

    return 0;
}

Common Mistakes with Variables

  • Using before declaring: cout << x; before int x = 5; will cause a compiler error.
  • Using uninitialized variables: Uninitialized local variables contain garbage values. Always initialize!
  • Shadowing: A local variable with the same name as a global one hides the global version.
  • Modifying constants: Trying to change a const variable causes a compiler error.
  • Type mismatch: Storing a string in an int variable causes errors.
  • Narrowing conversions: Assigning double to int silently truncates the decimal part.

Key Point

Always initialize variables when you declare them. Use const for values that shouldn't change. Prefer constexpr when the value is known at compile time. Choose meaningful, descriptive variable names. Use static_cast for explicit type conversions.
