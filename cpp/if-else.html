10. If/Else Statements

📚 Lesson 10 of 55 ⏱ 20 min read 🌱 Beginner

What Is Conditional Execution?

So far, every program you have written executes its statements from top to bottom, one after another. But real-world programs need to make decisions. Should the app show a welcome message or an error? Should the game move the player left or right? Should the bank approve or deny a withdrawal?

Conditional execution lets your program choose which code to run based on whether a condition is true or false. In C++ the primary tool for this is the if statement and its companions else and else if.

In this lesson you will learn:

  • How the if statement works
  • How to add an alternative path with else
  • How to chain multiple conditions with else if
  • How to nest if statements inside one another
  • How to use comparison and logical operators in conditions
  • Short-circuit evaluation
  • Common mistakes like = vs == and the dangling else
  • The ternary operator (? :) as a shorthand
  • C++17’s if with initializer
  • Practical examples: grade calculator, number classifier, menu system

Key Point

A condition in C++ is any expression that evaluates to a boolean value. The value 0 is treated as false; every other value (including negative numbers) is treated as true.

The if Statement

The simplest form of conditional execution: run a block of code only if a condition is true.

Syntax

if (condition) {
    // Executes only when condition is true
}

The parentheses around the condition are required. The braces { } define the body. If the body contains only a single statement, braces are technically optional—but always using them is strongly recommended to prevent bugs.

Example

#include <iostream>
using namespace std;

int main() {
    int temperature = 35;

    if (temperature > 30) {
        cout << "It's a hot day!" << endl;
    }

    cout << "Program continues..." << endl;
    return 0;
}

Output:

It's a hot day!
Program continues...

Because temperature (35) is greater than 30, the condition is true and the message prints. If you change temperature to 25, the if body is skipped and only “Program continues...” prints.

Tip

Always use braces { } even for single-statement bodies. This prevents subtle bugs when you later add a second statement and forget to add braces.

The if-else Statement

Often you want to do one thing when a condition is true and a different thing when it is false. That is what else provides.

Syntax

if (condition) {
    // Runs when condition is true
} else {
    // Runs when condition is false
}

Example

#include <iostream>
using namespace std;

int main() {
    int age = 16;

    if (age >= 18) {
        cout << "You are eligible to vote." << endl;
    } else {
        cout << "You are not old enough to vote." << endl;
    }

    return 0;
}

Output:

You are not old enough to vote.

Exactly one of the two branches will execute—never both, never neither.

Key Point

The else block has no condition of its own. It is simply the “otherwise” path that runs whenever the if condition evaluates to false.

The if – else if – else Chain

When you have more than two possibilities, chain multiple conditions together with else if.

Syntax

if (condition1) {
    // Runs if condition1 is true
} else if (condition2) {
    // Runs if condition1 is false AND condition2 is true
} else if (condition3) {
    // Runs if condition1 and condition2 are false AND condition3 is true
} else {
    // Runs if ALL conditions above are false
}

The conditions are tested in order. As soon as one is true, its block runs and the rest are skipped. The final else is optional but acts as a catch-all default.

Example: Grade Calculator

#include <iostream>
using namespace std;

int main() {
    int score;
    cout << "Enter your score (0-100): ";
    cin >> score;

    if (score >= 90) {
        cout << "Grade: A" << endl;
    } else if (score >= 80) {
        cout << "Grade: B" << endl;
    } else if (score >= 70) {
        cout << "Grade: C" << endl;
    } else if (score >= 60) {
        cout << "Grade: D" << endl;
    } else {
        cout << "Grade: F" << endl;
    }

    return 0;
}

If the user enters 85, the program checks: is 85 ≥ 90? No. Is 85 ≥ 80? Yes → prints “Grade: B” and skips the remaining conditions.

Tip

Order matters! Because earlier conditions are checked first, you do not need to write score >= 80 && score < 90. If you reach the else if (score >= 80) branch, you already know the score is less than 90.

Nested if Statements

You can place an if statement inside another if (or else) block. This is called nesting and is useful when a second decision depends on the result of the first.

#include <iostream>
using namespace std;

int main() {
    int age = 25;
    bool hasLicense = true;

    if (age >= 18) {
        if (hasLicense) {
            cout << "You can drive." << endl;
        } else {
            cout << "You are old enough but need a license." << endl;
        }
    } else {
        cout << "You are too young to drive." << endl;
    }

    return 0;
}

Output:

You can drive.

Tip

Deep nesting (three or more levels) makes code hard to read. Consider using logical operators (&&, ||) or restructuring into functions to flatten nested conditions.

Comparison (Relational) Operators in Conditions

Conditions in if statements most often use comparison operators to compare two values. Here is the full set:

OperatorMeaningExampleResult (a=5, b=3)
==Equal toa == bfalse
!=Not equal toa != btrue
>Greater thana > btrue
<Less thana < bfalse
>=Greater than or equala >= btrue
<=Less than or equala <= bfalse
 
#include <iostream>
using namespace std;

int main() {
    int x = 10, y = 20;

    if (x == y) {
        cout << "x and y are equal" << endl;
    }
    if (x != y) {
        cout << "x and y are NOT equal" << endl;
    }
    if (x < y) {
        cout << "x is less than y" << endl;
    }

    return 0;
}

Output:

x and y are NOT equal
x is less than y

Logical Operators in Conditions

Logical operators combine multiple conditions into a single expression.

OperatorNameDescription
&&Logical ANDTrue only if both operands are true
||Logical ORTrue if at least one operand is true
!Logical NOTReverses the truth value

AND (&&)

int age = 25;
bool hasTicket = true;

if (age >= 18 && hasTicket) {
    cout << "You may enter the concert." << endl;
}

Both conditions must be true. If age is 15, the entire expression is false regardless of hasTicket.

OR (||)

bool isWeekend = true;
bool isHoliday = false;

if (isWeekend || isHoliday) {
    cout << "You can sleep in!" << endl;
}

At least one must be true. Here isWeekend is true, so the message prints.

NOT (!)

bool isRaining = false;

if (!isRaining) {
    cout << "Let's go for a walk!" << endl;
}

!isRaining flips false to true, so the body executes.

Combining Operators

int temp = 22;
bool isSunny = true;

if (temp > 20 && temp < 35 && isSunny) {
    cout << "Perfect weather for the beach!" << endl;
}

// Using parentheses for clarity
if ((temp > 30 || isSunny) && !isRaining) {
    cout << "Go outside!" << endl;
}

Key Point

Operator precedence: ! binds tightest, then &&, then ||. Use parentheses to make your intent clear: (a || b) && c is very different from a || (b && c).

Short-Circuit Evaluation

C++ evaluates logical expressions from left to right and stops as soon as the result is determined. This is called short-circuit evaluation.

  • && (AND): If the left operand is false, the result is false—the right operand is not evaluated.
  • || (OR): If the left operand is true, the result is true—the right operand is not evaluated.
#include <iostream>
using namespace std;

int main() {
    int x = 0;

    // Short-circuit prevents division by zero!
    if (x != 0 && 10 / x > 2) {
        cout << "This won't cause a crash." << endl;
    } else {
        cout << "x is zero, division skipped." << endl;
    }

    return 0;
}

Because x != 0 is false, C++ never evaluates 10 / x, avoiding a division-by-zero error.

Tip

Short-circuit evaluation is a powerful tool. Place the “cheaper” or “safer” check on the left side of && and || to guard against expensive computations or runtime errors.

Common Mistakes

Mistake 1: = vs ==

One of the most frequent bugs in C++. The single = is the assignment operator; the double == is the comparison operator.

int x = 5;

// BUG: This ASSIGNS 10 to x, then evaluates 10 as true
if (x = 10) {
    cout << "This always runs!" << endl;
}

// CORRECT: This COMPARES x with 10
if (x == 10) {
    cout << "x is 10" << endl;
}

Key Point

Using = instead of == inside an if condition is a legal C++ expression (it assigns and then tests the assigned value). The compiler may warn but will not error. Always double-check your conditions!

Mistake 2: The Dangling Else

Without braces, an else attaches to the nearest preceding if, which can produce surprising behavior:

int x = 5, y = 10;

// Misleading indentation!
if (x > 0)
    if (y > 0)
        cout << "Both positive" << endl;
else
    cout << "This looks like it belongs to the first if..." << endl;
    // But it actually belongs to the SECOND if (y > 0)

The compiler reads it as:

if (x > 0) {
    if (y > 0) {
        cout << "Both positive" << endl;
    } else {
        cout << "This belongs to the second if" << endl;
    }
}

Tip

Always use braces to make the structure explicit. This completely avoids the dangling else ambiguity and makes your code easier to read and maintain.

Mistake 3: Semicolon After the if Condition

int x = 5;

// BUG: The semicolon creates an empty body
if (x > 0);   // <-- empty statement!
{
    cout << "This ALWAYS runs, regardless of x" << endl;
}

// CORRECT:
if (x > 0) {
    cout << "x is positive" << endl;
}

Mistake 4: Comparing Floating-Point Numbers

double a = 0.1 + 0.2;

// BUG: May be false due to floating-point imprecision
if (a == 0.3) {
    cout << "Equal" << endl;
}

// CORRECT: Compare within a tolerance
#include <cmath>
if (fabs(a - 0.3) < 1e-9) {
    cout << "Approximately equal" << endl;
}

The Ternary Operator (? :)

The ternary (conditional) operator is a compact shorthand for a simple if-else that assigns or returns a value.

Syntax

result = (condition) ? value_if_true : value_if_false;

Examples

#include <iostream>
using namespace std;

int main() {
    int age = 20;

    // Instead of if-else:
    string status = (age >= 18) ? "Adult" : "Minor";
    cout << status << endl;  // Adult

    // Inline in output:
    cout << "You are " << ((age >= 18) ? "an adult" : "a minor") << endl;

    // Finding the maximum:
    int a = 7, b = 12;
    int maxVal = (a > b) ? a : b;
    cout << "Max: " << maxVal << endl;  // 12

    // Absolute value:
    int num = -5;
    int absVal = (num >= 0) ? num : -num;
    cout << "Absolute: " << absVal << endl;  // 5

    return 0;
}

Tip

The ternary operator is great for short, simple conditions. If your expression becomes complex or spans multiple lines, use a regular if-else for clarity.

Nested Ternary (Use Sparingly)

int score = 85;

// Nested ternary — hard to read!
string grade = (score >= 90) ? "A"
             : (score >= 80) ? "B"
             : (score >= 70) ? "C"
             : (score >= 60) ? "D"
             : "F";
cout << "Grade: " << grade << endl;  // Grade: B

Key Point

Nested ternary operators are legal but reduce readability. Most style guides discourage nesting more than one level deep. Prefer if-else if chains for multi-way decisions.

if with Initializer (C++17)

C++17 introduced a powerful feature: you can declare and initialize a variable inside the if statement. The variable’s scope is limited to the if-else block, keeping the enclosing scope clean.

Syntax

if (init-statement; condition) {
    // variable from init-statement is in scope here
} else {
    // ...and here too
}
// variable is OUT of scope here

Example

#include <iostream>
#include <string>
#include <map>
using namespace std;

int main() {
    map<string, int> scores = {{"Alice", 92}, {"Bob", 78}};

    // C++17: declare 'it' inside the if
    if (auto it = scores.find("Alice"); it != scores.end()) {
        cout << "Found: " << it->second << endl;
    } else {
        cout << "Not found" << endl;
    }
    // 'it' is no longer accessible here

    return 0;
}

Without C++17, you would have to declare it before the if, polluting the outer scope.

Another Example

// Lock a mutex and check a condition in one statement
if (auto val = getValue(); val > 0) {
    cout << "Positive value: " << val << endl;
} else {
    cout << "Non-positive: " << val << endl;
}

Tip

To use C++17 features, compile with the -std=c++17 flag (GCC/Clang) or set the C++ Language Standard to C++17 in your IDE project settings.

Practical Examples

Example 1: Number Classifier

#include <iostream>
using namespace std;

int main() {
    int num;
    cout << "Enter a number: ";
    cin >> num;

    // Check positive, negative, or zero
    if (num > 0) {
        cout << num << " is positive." << endl;
    } else if (num < 0) {
        cout << num << " is negative." << endl;
    } else {
        cout << "The number is zero." << endl;
    }

    // Check even or odd (only for non-zero)
    if (num != 0) {
        if (num % 2 == 0) {
            cout << num << " is even." << endl;
        } else {
            cout << num << " is odd." << endl;
        }
    }

    // Check divisibility
    if (num % 3 == 0 && num % 5 == 0) {
        cout << num << " is divisible by both 3 and 5." << endl;
    } else if (num % 3 == 0) {
        cout << num << " is divisible by 3." << endl;
    } else if (num % 5 == 0) {
        cout << num << " is divisible by 5." << endl;
    }

    return 0;
}

Example 2: Grade Calculator with Input Validation

#include <iostream>
using namespace std;

int main() {
    int score;
    cout << "Enter your test score (0-100): ";
    cin >> score;

    // Validate input first
    if (score < 0 || score > 100) {
        cout << "Error: Score must be between 0 and 100." << endl;
    } else {
        char grade;
        if (score >= 90) {
            grade = 'A';
        } else if (score >= 80) {
            grade = 'B';
        } else if (score >= 70) {
            grade = 'C';
        } else if (score >= 60) {
            grade = 'D';
        } else {
            grade = 'F';
        }

        cout << "Your grade: " << grade << endl;

        // Additional feedback
        if (grade == 'A' || grade == 'B') {
            cout << "Great job! Keep it up!" << endl;
        } else if (grade == 'F') {
            cout << "Please see the instructor for help." << endl;
        }
    }

    return 0;
}

Example 3: Simple Menu System

#include <iostream>
using namespace std;

int main() {
    cout << "===== Calculator =====" << endl;
    cout << "1. Add" << endl;
    cout << "2. Subtract" << endl;
    cout << "3. Multiply" << endl;
    cout << "4. Divide" << endl;
    cout << "Choose an option (1-4): ";

    int choice;
    cin >> choice;

    double a, b;
    cout << "Enter two numbers: ";
    cin >> a >> b;

    if (choice == 1) {
        cout << a << " + " << b << " = " << (a + b) << endl;
    } else if (choice == 2) {
        cout << a << " - " << b << " = " << (a - b) << endl;
    } else if (choice == 3) {
        cout << a << " * " << b << " = " << (a * b) << endl;
    } else if (choice == 4) {
        if (b != 0) {
            cout << a << " / " << b << " = " << (a / b) << endl;
        } else {
            cout << "Error: Cannot divide by zero!" << endl;
        }
    } else {
        cout << "Invalid choice." << endl;
    }

    return 0;
}

Example 4: Leap Year Checker

#include <iostream>
using namespace std;

int main() {
    int year;
    cout << "Enter a year: ";
    cin >> year;

    // A year is a leap year if:
    // - Divisible by 4 AND not divisible by 100
    // - OR divisible by 400
    if ((year % 4 == 0 && year % 100 != 0) || (year % 400 == 0)) {
        cout << year << " is a leap year." << endl;
    } else {
        cout << year << " is not a leap year." << endl;
    }

    return 0;
}

Summary

  • if runs code only when its condition is true.
  • if-else provides two mutually exclusive paths.
  • if-else if-else handles multiple conditions in sequence; the first true condition wins.
  • Nested if statements allow multi-level decisions but should be kept shallow for readability.
  • Use comparison operators (==, !=, <, >, <=, >=) to form conditions.
  • Use logical operators (&&, ||, !) to combine conditions.
  • Short-circuit evaluation: && stops on false; || stops on true.
  • Watch out for = vs ==, the dangling else, and an accidental semicolon after if.
  • The ternary operator (condition ? a : b) is a compact alternative for simple cases.
  • C++17 if-with-initializer confines variables to the if-else block scope.
