17. Function Parameters

📚 Lesson 17 of 55 ⏱ 18 min read 🌱 Beginner

What Are Function Parameters?

Parameters are variables listed in a function’s declaration that act as placeholders for the values (called arguments) passed when the function is called. They allow you to write flexible functions that work with different data each time.

In the previous lesson you learned the basics of defining and calling functions. Now you’ll master how data flows into functions through parameters, including the critical difference between passing by value and passing by reference.

In this lesson you will learn:

  • Pass by value — the default behavior
  • Pass by reference using &
  • Using const references for efficiency
  • Default parameter values
  • Rules for multiple default parameters
  • Passing arrays to functions
  • Practical examples with references and arrays

Key Point

Understanding how parameters are passed is one of the most important concepts in C++. It determines whether a function works on a copy of the data or on the original data itself.

Pass by Value (Default)

By default, C++ passes arguments by value. This means the function receives a copy of each argument. Any changes made to the parameter inside the function do not affect the original variable.

#include <iostream>
using namespace std;

void doubleValue(int num) {
    num = num * 2;  // Modifies the LOCAL copy only
    cout << "Inside function: " << num << endl;
}

int main() {
    int x = 5;
    doubleValue(x);
    cout << "After function call: " << x << endl;
    return 0;
}
Output:
Inside function: 10
After function call: 5

Notice that x remains 5 after the function call. The function only modified its local copy, num.

Tip

Pass by value is safe because the caller’s data is protected from unintended changes. Use it for small types like int, double, char, and bool.

How Pass by Value Works Internally

#include <iostream>
using namespace std;

void modify(int a, double b) {
    a = 100;
    b = 99.9;
    cout << "Inside: a=" << a << ", b=" << b << endl;
}

int main() {
    int x = 1;
    double y = 2.5;
    modify(x, y);
    cout << "Outside: x=" << x << ", y=" << y << endl;
    return 0;
}
Output:
Inside: a=100, b=99.9
Outside: x=1, y=2.5

Key Point

With pass by value, each parameter is an independent copy. Modifications inside the function are invisible to the caller.

Pass by Reference

To allow a function to modify the original variable, use a reference parameter by adding & after the type. A reference is an alias — it refers directly to the original variable rather than making a copy.

#include <iostream>
using namespace std;

void doubleValue(int &num) {   // & makes it a reference
    num = num * 2;  // Modifies the ORIGINAL variable
    cout << "Inside function: " << num << endl;
}

int main() {
    int x = 5;
    doubleValue(x);
    cout << "After function call: " << x << endl;
    return 0;
}
Output:
Inside function: 10
After function call: 10

This time x is 10 after the call, because num is a reference to x — they share the same memory.

Tip

A common convention: place the & next to the type (int& num) to emphasize the type is “reference to int.” Both int &num and int& num are valid C++.

The Classic Swap Function

Swapping two variables is the textbook example of why pass by reference is essential:

#include <iostream>
using namespace std;

// WRONG: pass by value — doesn't actually swap
void badSwap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}

// CORRECT: pass by reference — swaps the originals
void goodSwap(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}

int main() {
    int x = 10, y = 20;

    badSwap(x, y);
    cout << "After badSwap:  x=" << x << ", y=" << y << endl;

    goodSwap(x, y);
    cout << "After goodSwap: x=" << x << ", y=" << y << endl;

    return 0;
}
Output:
After badSwap:  x=10, y=20
After goodSwap: x=20, y=10

Key Point

badSwap swaps copies and has no effect on the originals. goodSwap uses reference parameters (&) so it modifies x and y directly.

Multiple Reference Parameters

#include <iostream>
using namespace std;

void getMinMax(int a, int b, int c, int &minVal, int &maxVal) {
    minVal = a;
    maxVal = a;
    if (b < minVal) minVal = b;
    if (c < minVal) minVal = c;
    if (b > maxVal) maxVal = b;
    if (c > maxVal) maxVal = c;
}

int main() {
    int lo, hi;
    getMinMax(7, 2, 9, lo, hi);
    cout << "Min: " << lo << ", Max: " << hi << endl;
    return 0;
}
Output:
Min: 2, Max: 9

Reference parameters let a function effectively “return” multiple values by writing directly to the caller’s variables.

const References

For large objects (like strings or vectors), passing by value is expensive because the entire object is copied. Use a const reference (const type&) to avoid the copy while preventing the function from modifying the original.

#include <iostream>
#include <string>
using namespace std;

void printMessage(const string &msg) {
    cout << msg << endl;
    // msg = "changed";  // ERROR: cannot modify a const reference
}

int main() {
    string greeting = "Hello, CodeVerse!";
    printMessage(greeting);
    return 0;
}
Output:
Hello, CodeVerse!

Key Point

Use const& when you want efficiency (no copy) and safety (no modification). This is the recommended way to pass strings, vectors, and other large objects to functions.

When to Use Each Passing Style

Style Syntax Use When
Pass by value void f(int x) Small types, no need to modify original
Pass by reference void f(int& x) You need to modify the original variable
Pass by const reference void f(const int& x) Large objects, read-only access

Tip

A simple guideline: pass built-in types (int, double, char, bool) by value, and pass everything else by const& unless you need to modify it.

Default Parameters

C++ lets you assign default values to parameters. If the caller omits those arguments, the defaults are used automatically.

#include <iostream>
using namespace std;

void greet(string name, string greeting = "Hello") {
    cout << greeting << ", " << name << "!" << endl;
}

int main() {
    greet("Alice");               // Uses default greeting
    greet("Bob", "Good morning"); // Overrides default
    return 0;
}
Output:
Hello, Alice!
Good morning, Bob!

Multiple Default Parameters

You can have multiple default parameters, but they must appear at the end of the parameter list.

#include <iostream>
using namespace std;

double calculatePrice(double base, double tax = 0.08, double discount = 0.0) {
    return base + (base * tax) - (base * discount);
}

int main() {
    cout << "No extras:     $" << calculatePrice(100.0) << endl;
    cout << "Custom tax:    $" << calculatePrice(100.0, 0.10) << endl;
    cout << "Tax+discount:  $" << calculatePrice(100.0, 0.10, 0.05) << endl;
    return 0;
}
Output:
No extras:     $108
Custom tax:    $110
Tax+discount:  $105

Key Point

Default parameter rules:
1. Defaults must start from the rightmost parameter and fill to the left.
2. You cannot skip a default; if you supply argument 3, you must also supply arguments 1 and 2.
3. Defaults are typically specified in the declaration (prototype), not the definition, when the two are separate.

Default Parameters in Prototypes

#include <iostream>
using namespace std;

// Defaults go in the prototype
void printLine(char ch = '-', int length = 40);

int main() {
    printLine();          // Uses both defaults
    printLine('*');       // Uses default length
    printLine('=', 20);   // No defaults used
    return 0;
}

void printLine(char ch, int length) {
    for (int i = 0; i < length; i++) {
        cout << ch;
    }
    cout << endl;
}
Output:
----------------------------------------
****************************************
====================

Tip

Never put default values in both the prototype and the definition — it causes a compiler error. Choose one place, and the prototype is preferred.

Passing Arrays to Functions

Arrays are always passed by reference in C++ (technically, the array name decays to a pointer). This means a function can modify the original array elements. You must also pass the array size since the function has no way to determine it.

#include <iostream>
using namespace std;

void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
}

int main() {
    int numbers[] = {10, 20, 30, 40, 50};
    int size = sizeof(numbers) / sizeof(numbers[0]);
    printArray(numbers, size);
    return 0;
}
Output:
10 20 30 40 50

Modifying Array Elements

#include <iostream>
using namespace std;

void doubleElements(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        arr[i] *= 2;   // Modifies the original array
    }
}

void printArray(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
}

int main() {
    int data[] = {1, 2, 3, 4, 5};
    int size = 5;

    cout << "Before: ";
    printArray(data, size);

    doubleElements(data, size);

    cout << "After:  ";
    printArray(data, size);

    return 0;
}
Output:
Before: 1 2 3 4 5
After:  2 4 6 8 10

Protecting Arrays with const

If a function should only read the array, use const to prevent accidental modification:

#include <iostream>
using namespace std;

int sumArray(const int arr[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += arr[i];
        // arr[i] = 0;  // ERROR: cannot modify a const array
    }
    return total;
}

int main() {
    int nums[] = {10, 20, 30, 40, 50};
    cout << "Sum: " << sumArray(nums, 5) << endl;
    return 0;
}
Output:
Sum: 150

Key Point

Always pass the array size as a separate parameter. Inside a function, sizeof(arr) gives the size of the pointer, not the array — a common bug for beginners.

Practical Examples

Example 1: Statistics Calculator

#include <iostream>
using namespace std;

void calcStats(const int arr[], int size, int &sum, double &avg, int &minVal, int &maxVal) {
    sum = 0;
    minVal = arr[0];
    maxVal = arr[0];

    for (int i = 0; i < size; i++) {
        sum += arr[i];
        if (arr[i] < minVal) minVal = arr[i];
        if (arr[i] > maxVal) maxVal = arr[i];
    }
    avg = static_cast<double>(sum) / size;
}

int main() {
    int data[] = {45, 82, 67, 91, 23, 78, 55};
    int size = 7;
    int sum, minVal, maxVal;
    double avg;

    calcStats(data, size, sum, avg, minVal, maxVal);

    cout << "Sum: " << sum << endl;
    cout << "Avg: " << avg << endl;
    cout << "Min: " << minVal << endl;
    cout << "Max: " << maxVal << endl;
    return 0;
}
Output:
Sum: 441
Avg: 63
Min: 23
Max: 91

Example 2: Array Reversal

#include <iostream>
using namespace std;

void reverseArray(int arr[], int size) {
    for (int i = 0; i < size / 2; i++) {
        int temp = arr[i];
        arr[i] = arr[size - 1 - i];
        arr[size - 1 - i] = temp;
    }
}

void printArray(const int arr[], int size) {
    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    int size = 5;

    cout << "Original: ";
    printArray(nums, size);

    reverseArray(nums, size);

    cout << "Reversed: ";
    printArray(nums, size);

    return 0;
}
Output:
Original: 1 2 3 4 5
Reversed: 5 4 3 2 1

Example 3: Temperature Converter with Default

#include <iostream>
using namespace std;

double convert(double temp, char from = 'C') {
    if (from == 'C' || from == 'c') {
        return temp * 9.0 / 5.0 + 32.0;  // Celsius to Fahrenheit
    } else {
        return (temp - 32.0) * 5.0 / 9.0; // Fahrenheit to Celsius
    }
}

int main() {
    cout << "100C = " << convert(100) << "F" << endl;
    cout << "212F = " << convert(212, 'F') << "C" << endl;
    return 0;
}
Output:
100C = 212F
212F = 100C

Summary

  • Pass by value creates a copy; changes inside the function don’t affect the original.
  • Pass by reference (&) modifies the original variable directly.
  • const references (const&) provide read-only access without copying — ideal for large objects.
  • Default parameters let you omit arguments; defaults must be rightmost.
  • You cannot skip a default argument — supply all preceding arguments first.
  • Arrays are always passed by reference (pointer); always pass the size separately.
  • Use const with array parameters when the function should not modify the array.
  • Reference parameters allow a function to effectively “return” multiple values.
