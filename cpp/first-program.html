3. Your First C++ Program

📚 Lesson 3 of 55 ⏱ 10 min read 🌱 Beginner

Writing Your First C++ Program

Every programmer's journey begins with a simple program that displays a message on the screen. This tradition dates back to the 1978 book "The C Programming Language" by Brian Kernighan and Dennis Ritchie. In this lesson, you will write, understand, compile, and run your very first C++ program — the classic Hello World program.

By the end of this tutorial, you will not only know how to create a working C++ program, but you will also understand every single line of code, how the compilation process works, how to handle errors, and how to write more advanced programs that use input and output.

The Hello World Program

Open your text editor or IDE (such as VS Code, Code::Blocks, or Visual Studio) and type the following code exactly as shown:

#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;
}

Save this file with a .cpp extension, for example hello.cpp. The .cpp extension tells the compiler (and your IDE) that this is a C++ source file.

Key Point

C++ source files use the extension .cpp. Other valid extensions include .cc, .cxx, and .C (uppercase), but .cpp is the most widely used and recommended convention.

Breaking Down Every Part

Let's examine every single line and keyword in the Hello World program so you fully understand what each part does and why it is needed.

1. #include <iostream>

This is a preprocessor directive. Before the compiler translates your code into machine language, a tool called the preprocessor processes all lines starting with #. The #include directive tells the preprocessor to copy the entire contents of the specified file into your program.

<iostream> is a header file from the C++ Standard Library. It provides input and output functionality, specifically:

  • cout — Used to print output to the console (standard output).
  • cin — Used to read input from the keyboard (standard input).
  • cerr — Used to output error messages.
  • endl — Inserts a newline character and flushes the output buffer.

Without this line, the compiler would not know what cout, cin, or endl are, and your program would fail to compile.

// The angle brackets < > tell the preprocessor to look
// in the standard system include directories
#include <iostream>

// Double quotes tell the preprocessor to look in the
// current directory first, then the system directories
#include "myheader.h"

2. using namespace std;

All standard library components in C++ (like cout, cin, string, vector) live inside a namespace called std (short for "standard"). A namespace is a way to group related names together to avoid naming conflicts.

Without using namespace std;, you would need to write std::cout instead of just cout, and std::endl instead of endl. The :: operator is called the scope resolution operator.

// Without "using namespace std;"
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

// With "using namespace std;"
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl;
    return 0;
}

Key Point

While using namespace std; is convenient for beginners and small programs, professional C++ developers often avoid it in header files and large projects because it can cause name collisions. In production code, it is common to use the explicit std:: prefix or to import only specific names like using std::cout;.

3. int main()

This is the main function — the entry point of every C++ program. When you run a C++ program, the operating system calls the main() function first. Every C++ program must have exactly one main() function.

  • int — The return type. It tells the compiler that the main() function will return an integer value to the operating system when it finishes.
  • main — The function name. This specific name is required and cannot be changed.
  • () — The parameter list. Empty parentheses mean the function takes no arguments. You can also write int main(int argc, char* argv[]) to accept command-line arguments.
  • { } — The function body enclosed in curly braces. All the code that belongs to main() goes between these braces.
// The main function with command-line arguments
int main(int argc, char* argv[]) {
    // argc = number of arguments
    // argv = array of argument strings
    return 0;
}

4. cout << "Hello, World!" << endl;

This is the statement that actually prints text to the screen. Let's break it down further:

  • cout — Stands for "character output" (or "console output"). It is an object representing the standard output stream, which is usually your terminal or console window.
  • << — The insertion operator (also called the "put to" operator). It sends the data on its right side to the stream on its left side. Think of it as "sending data into the stream."
  • "Hello, World!" — A string literal. Any text enclosed in double quotes is a string literal in C++.
  • endl — Stands for "end line". It inserts a newline character (\n) and flushes the output buffer, ensuring the text is immediately displayed on the screen.
  • ; — The semicolon marks the end of a statement. Every statement in C++ must end with a semicolon. Forgetting it is one of the most common beginner mistakes.

Key Point

You can chain multiple << operators together in a single statement. For example: cout << "Name: " << "Alice" << ", Age: " << 25 << endl;. This sends multiple pieces of data to the output stream in sequence.

5. return 0;

This statement tells the operating system that the program has finished executing successfully. The value 0 is a convention that means "no errors." Any non-zero return value typically indicates that the program encountered an error.

  • return 0; — Program finished successfully.
  • return 1; — Program finished with an error (convention).
  • In modern C++ (C++11 and later), if you omit return 0; from main(), the compiler implicitly returns 0 for you. However, it is good practice to include it explicitly.
// Return values from main()
int main() {
    // ... program logic ...

    if (/* everything went well */) {
        return 0;  // Success
    } else {
        return 1;  // Error
    }
}

How to Compile Your Program

Unlike interpreted languages (such as Python), C++ is a compiled language. This means you cannot run your source code directly. You must first translate (compile) it into machine code that your computer's processor can understand. This step produces an executable file.

Compiling with g++ (GCC)

g++ is the C++ compiler from the GNU Compiler Collection (GCC). It is the most commonly used compiler on Linux and is also available on macOS (via Xcode Command Line Tools or Homebrew) and Windows (via MinGW or MSYS2).

# Basic compilation
g++ hello.cpp -o hello

# Compile with C++17 standard and warnings enabled
g++ -std=c++17 -Wall -Wextra -o hello hello.cpp

# Compile with debugging information
g++ -g -o hello hello.cpp

# Compile with optimization
g++ -O2 -o hello hello.cpp
  • g++ — Invokes the GNU C++ compiler.
  • hello.cpp — Your source code file.
  • -o hello — Specifies the name of the output (executable) file. Without this flag, the default output name is a.out on Linux/macOS or a.exe on Windows.
  • -std=c++17 — Tells the compiler to use the C++17 standard.
  • -Wall -Wextra — Enables additional warning messages to help you catch potential issues.

Compiling with clang++

clang++ is the C++ compiler from the LLVM project. It is known for its excellent, human-readable error messages and fast compilation times. It is the default compiler on macOS.

# Basic compilation with clang++
clang++ hello.cpp -o hello

# Compile with C++17 and warnings
clang++ -std=c++17 -Wall -Wextra -o hello hello.cpp

Compiling with cl (MSVC on Windows)

cl is the Microsoft Visual C++ compiler, used with Visual Studio on Windows. You typically run it from the "Developer Command Prompt" or "Developer PowerShell" that comes with Visual Studio.

# Basic compilation with MSVC
cl hello.cpp

# Compile with C++17
cl /std:c++17 hello.cpp

# Compile with warnings at level 4
cl /W4 hello.cpp

Key Point

The three most popular C++ compilers are g++ (GCC), clang++ (LLVM/Clang), and cl (MSVC). All three support modern C++ standards. If you are on Linux, use g++ or clang++. On macOS, clang++ is the default. On Windows, you can use MSVC (cl) or install g++ via MinGW.

How to Run the Compiled Executable

After compilation succeeds, you will have an executable file. How you run it depends on your operating system:

On Linux or macOS

# Run the executable
./hello

# Output:
Hello, World!

The ./ prefix tells the terminal to look for the executable in the current directory.

On Windows (Command Prompt or PowerShell)

# Run the executable
hello.exe

# Or simply
hello

# Output:
Hello, World!

Using an IDE

If you are using an IDE like Visual Studio, Code::Blocks, or CLion, you can simply click the Run button (or press F5 / Ctrl+F5) and the IDE will compile and run the program for you automatically.

Understanding Compilation Errors vs Runtime Errors

As a beginner, you will encounter two main types of errors. Understanding the difference is crucial for effective debugging.

Compilation Errors (Compile-Time Errors)

These errors occur during compilation, before your program ever runs. The compiler detects problems in your source code (syntax mistakes, missing semicolons, undeclared variables, type mismatches) and refuses to create an executable. You must fix all compilation errors before you can run your program.

// ERROR: Missing semicolon
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, World!" << endl  // <-- Missing semicolon!
    return 0;
}

The compiler would produce an error message similar to:

hello.cpp:6:43: error: expected ';' after expression
    cout << "Hello, World!" << endl
                                          ^
                                          ;
1 error generated.

Runtime Errors

These errors occur while the program is running. The code compiles successfully, but something goes wrong during execution — for example, dividing by zero, accessing invalid memory, or trying to open a file that doesn't exist.

// This compiles fine but crashes at runtime
#include <iostream>
using namespace std;

int main() {
    int a = 10;
    int b = 0;
    cout << a / b << endl;  // Division by zero - runtime error!
    return 0;
}

Key Point

Compilation errors are caught by the compiler before the program runs — they are usually syntax or type errors. Runtime errors occur during program execution and can cause crashes or unexpected behavior. Always read error messages carefully; they tell you what went wrong and on which line.

Writing Programs with Multiple cout Statements

You are not limited to printing a single line. You can use multiple cout statements to display as much text as you need.

#include <iostream>
using namespace std;

int main() {
    cout << "==========================" << endl;
    cout << "   Welcome to CodeVerse   " << endl;
    cout << "==========================" << endl;
    cout << endl;
    cout << "This is my first C++ program." << endl;
    cout << "I am learning C++ step by step." << endl;
    cout << "C++ is powerful and fast!" << endl;
    return 0;
}

Output:

==========================
   Welcome to CodeVerse   
==========================

This is my first C++ program.
I am learning C++ step by step.
C++ is powerful and fast!

You can also chain everything into a single cout statement using multiple << operators:

#include <iostream>
using namespace std;

int main() {
    cout << "Line 1" << endl
         << "Line 2" << endl
         << "Line 3" << endl;
    return 0;
}

Using Escape Sequences

Escape sequences are special character combinations that begin with a backslash (\). They allow you to insert characters that are difficult or impossible to type directly into a string literal.

Common Escape Sequences

Escape SequenceDescriptionExample Output
\nNewline — moves cursor to the next lineLine break
\tTab — inserts a horizontal tabIndentation
\\Backslash — prints a literal backslash\
\"Double quote — prints a literal double quote"
\'Single quote — prints a literal single quote'
\0Null character — marks the end of a C-string(nothing)
\aAlert — produces a beep sound(beep)
\bBackspace — moves cursor back one position(backspace)
\rCarriage return — moves cursor to beginning of line(reset position)
#include <iostream>
using namespace std;

int main() {
    // Using \n for new lines
    cout << "First line\nSecond line\nThird line\n";

    // Using \t for tabs
    cout << "\nName\tAge\tCity\n";
    cout << "Alice\t25\tNew York\n";
    cout << "Bob\t30\tLondon\n";

    // Using \\ to print a backslash
    cout << "\nFile path: C:\\Users\\Alice\\Documents\n";

    // Using \" to print double quotes
    cout << "She said, \"Hello!\"\n";

    return 0;
}

Output:

First line
Second line
Third line

Name	Age	City
Alice	25	New York
Bob	30	London

File path: C:\Users\Alice\Documents
She said, "Hello!"

Key Point

The \n escape sequence is often preferred over endl for inserting new lines because \n simply inserts a newline character, while endl inserts a newline AND flushes the output buffer. Flushing can be slower, so \n is more efficient when you don't need to guarantee immediate output.

Taking User Input with cin

So far, our programs have only displayed output. Now, let's write a program that reads input from the user using cin (character input / console input).

#include <iostream>
using namespace std;

int main() {
    string name;
    int age;

    cout << "Enter your name: ";
    cin >> name;

    cout << "Enter your age: ";
    cin >> age;

    cout << "\nHello, " << name << "!" << endl;
    cout << "You are " << age << " years old." << endl;

    return 0;
}

Sample Run:

Enter your name: Alice
Enter your age: 25

Hello, Alice!
You are 25 years old.

Let's understand how cin works:

  • cin is the standard input stream object.
  • >> is the extraction operator (or "get from" operator). It reads data from the input stream and stores it in the variable on its right.
  • cin >> name; reads a single word (stops at whitespace) and stores it in the name variable.
  • cin >> age; reads an integer and stores it in age.

Key Point

cin >> reads input separated by whitespace (spaces, tabs, newlines). If you type "Alice Smith", cin >> name will only read "Alice". To read an entire line including spaces, use getline(cin, name) instead.

Reading a Full Line of Input

#include <iostream>
#include <string>
using namespace std;

int main() {
    string fullName;

    cout << "Enter your full name: ";
    getline(cin, fullName);

    cout << "Hello, " << fullName << "!" << endl;

    return 0;
}

Sample Run:

Enter your full name: Alice Smith
Hello, Alice Smith!

A More Complete Input Example

#include <iostream>
#include <string>
using namespace std;

int main() {
    string name;
    int birthYear;
    double height;

    cout << "===== User Profile =====" << endl;

    cout << "Enter your name: ";
    getline(cin, name);

    cout << "Enter your birth year: ";
    cin >> birthYear;

    cout << "Enter your height (in meters): ";
    cin >> height;

    int age = 2026 - birthYear;

    cout << "\n===== Your Profile =====" << endl;
    cout << "Name: " << name << endl;
    cout << "Age: " << age << " years" << endl;
    cout << "Height: " << height << " m" << endl;

    return 0;
}

Common Beginner Mistakes and How to Fix Them

Here are the most frequent mistakes that beginners make when writing their first C++ programs. Understanding these will save you hours of frustration.

Mistake 1: Forgetting the Semicolon

// WRONG - missing semicolon
cout << "Hello" << endl

// CORRECT
cout << "Hello" << endl;

Every statement in C++ must end with a semicolon (;). The compiler error message will usually point to the line after the missing semicolon, which can be confusing at first.

Mistake 2: Using Wrong Quotes

// WRONG - using single quotes for strings
cout << 'Hello, World!';

// CORRECT - use double quotes for strings
cout << "Hello, World!";

// Single quotes are for single characters only
char letter = 'A';  // This is correct

Mistake 3: Misspelling Keywords

// WRONG
#Include <iostream>    // Capital 'I'
Using Namespace Std;   // Capital letters
Int Main() {           // Capital letters
    Cout << "Hello";   // Capital 'C'
}

// CORRECT - C++ is case-sensitive!
#include <iostream>
using namespace std;
int main() {
    cout << "Hello";
}

Mistake 4: Forgetting #include <iostream>

// WRONG - cout is not defined without iostream
int main() {
    cout << "Hello";  // Error: 'cout' was not declared
    return 0;
}

// CORRECT
#include <iostream>
using namespace std;
int main() {
    cout << "Hello";
    return 0;
}

Mistake 5: Using the Wrong Operators

// WRONG - using >> with cout (should be <<)
cout >> "Hello";

// WRONG - using << with cin (should be >>)
cin << name;

// CORRECT
cout << "Hello";  // << is insertion (output)
cin >> name;      // >> is extraction (input)

Mistake 6: Mismatched Braces

// WRONG - missing closing brace
int main() {
    cout << "Hello" << endl;
    return 0;
// <-- Missing closing brace!

// CORRECT
int main() {
    cout << "Hello" << endl;
    return 0;
}

Mistake 7: Writing Code Outside of main()

// WRONG - executable code outside main()
#include <iostream>
using namespace std;

cout << "Hello";  // Error! This must be inside a function.

int main() {
    return 0;
}

// CORRECT
#include <iostream>
using namespace std;

int main() {
    cout << "Hello";  // Inside main()
    return 0;
}

Mistake 8: Using Undeclared Variables

// WRONG - variable not declared before use
int main() {
    cout << x;  // Error: 'x' was not declared
    return 0;
}

// CORRECT
int main() {
    int x = 42;  // Declare first, then use
    cout << x;
    return 0;
}

Key Point

C++ is case-sensitive — Main, MAIN, and main are all different identifiers. Only lowercase main is recognized as the program's entry point. Similarly, Cout is not the same as cout. Always pay attention to capitalization.

Complete Example: A Simple Calculator

Let's put everything together and write a slightly more advanced program that demonstrates input, output, and basic computation:

#include <iostream>
using namespace std;

int main() {
    double num1, num2;

    cout << "============================" << endl;
    cout << "   Simple C++ Calculator    " << endl;
    cout << "============================" << endl;

    cout << "\nEnter first number: ";
    cin >> num1;

    cout << "Enter second number: ";
    cin >> num2;

    cout << "\n--- Results ---" << endl;
    cout << num1 << " + " << num2 << " = " << (num1 + num2) << endl;
    cout << num1 << " - " << num2 << " = " << (num1 - num2) << endl;
    cout << num1 << " * " << num2 << " = " << (num1 * num2) << endl;

    if (num2 != 0) {
        cout << num1 << " / " << num2 << " = " << (num1 / num2) << endl;
    } else {
        cout << "Division by zero is not allowed!" << endl;
    }

    return 0;
}

Sample Run:

============================
   Simple C++ Calculator    
============================

Enter first number: 15.5
Enter second number: 4.2

--- Results ---
15.5 + 4.2 = 19.7
15.5 - 4.2 = 11.3
15.5 * 4.2 = 65.1
15.5 / 4.2 = 3.69048

Practice Exercises

Try these exercises on your own to reinforce what you've learned. Write, compile, and run each one:

  1. Personal Introduction: Write a program that prints your name, age, and favorite programming language on separate lines.
  2. ASCII Art: Use cout statements and escape sequences to print a simple ASCII art shape (like a triangle, house, or smiley face).
  3. Temperature Input: Write a program that asks the user for a temperature in Celsius, then converts it to Fahrenheit using the formula: F = (C * 9/5) + 32. Display the result.
  4. Rectangle Area: Write a program that asks the user for the length and width of a rectangle, then calculates and displays the area and perimeter.
  5. Mad Libs: Create a simple Mad Libs game that asks the user for a noun, verb, adjective, and place name, then prints a funny sentence using those words.
  6. Escape Sequence Art: Write a program that uses \t and \n to display a formatted table with at least 3 columns and 4 rows of data.
  7. Personal Greeting: Write a program that asks for the user's full name (using getline) and birth year, then calculates their approximate age and prints a friendly greeting message.
