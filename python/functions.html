Functions & Lambdas

What Are Functions?

Functions are reusable blocks of code that perform a specific task. They are the building blocks of well-organized, maintainable software. Instead of writing the same code multiple times, you define it once in a function and call it whenever you need it. Functions help you follow the DRY (Don't Repeat Yourself) principle.

In Python, functions are defined using the def keyword followed by the function name, parentheses containing any parameters, and a colon. The function body is indented. Functions can accept inputs (parameters), perform operations, and return results. Python functions are first-class objects — they can be assigned to variables, passed as arguments to other functions, and returned from functions.

Python also comes with many useful built-in functions like print(), len(), range(), type(), input(), sorted(), and enumerate(). Understanding how to create your own functions alongside these built-ins is essential to becoming proficient in Python.

python 
# Defining and calling a basic function
def greet(name):
    """Greet a person by name."""
    print(f"Hello, {name}! Welcome to Python.")

greet("Alice")   # Hello, Alice! Welcome to Python.
greet("Bob")     # Hello, Bob! Welcome to Python.

# Function with return value
def add(a, b):
    """Return the sum of two numbers."""
    return a + b

result = add(3, 7)
print(f"3 + 7 = {result}")  # 3 + 7 = 10

# Function with multiple return values (returns a tuple)
def divide(a, b):
    """Return quotient and remainder."""
    quotient = a // b
    remainder = a % b
    return quotient, remainder

q, r = divide(17, 5)
print(f"17 ÷ 5 = {q} remainder {r}")  # 17 ÷ 5 = 3 remainder 2

Parameters & Arguments

Python functions support several types of parameters, giving you tremendous flexibility in how you design your function interfaces:

python 
# Default parameter values
def power(base, exponent=2):
    """Raise base to exponent (default: squared)."""
    return base ** exponent

print(power(5))       # 25 (uses default exponent=2)
print(power(2, 10))  # 1024

# Keyword arguments — order doesn't matter
def create_profile(name, age, city="Unknown"):
    return {"name": name, "age": age, "city": city}

profile = create_profile(age=25, name="Alice", city="NYC")
print(profile)
# {'name': 'Alice', 'age': 25, 'city': 'NYC'}

# *args — accept any number of positional arguments
def sum_all(*args):
    """Sum any number of arguments."""
    print(f"Received: {args}")  # args is a tuple
    return sum(args)

print(sum_all(1, 2, 3))        # 6
print(sum_all(10, 20, 30, 40))  # 100

# **kwargs — accept any number of keyword arguments
def print_info(**kwargs):
    """Print all keyword arguments."""
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_info(name="Alice", language="Python", level="Expert")
💡

Tip: Write docstrings for your functions! The triple-quoted string right after def becomes the function's documentation. You can access it with help(function_name) or function_name.__doc__. Good docstrings make your code professional and self-documenting.

Scope & the global Keyword

Variables defined inside a function are local to that function — they only exist within the function's scope. Variables defined outside functions are global. Python follows the LEGB rule for name resolution: Local → Enclosing → Global → Built-in:

python 
x = 10  # Global variable

def my_function():
    x = 20  # Local variable — shadows the global
    print(f"Inside function: x = {x}")  # 20

my_function()
print(f"Outside function: x = {x}")  # 10 (unchanged)

# Using 'global' to modify a global variable
counter = 0

def increment():
    global counter
    counter += 1

increment()
increment()
print(counter)  # 2

# Nested functions — closures (enclosing scope)
def outer():
    message = "Hello from outer!"
    
    def inner():
        print(message)  # Accesses enclosing scope
    
    return inner

my_func = outer()
my_func()  # "Hello from outer!" — closure remembers the value
⚠

Best Practice: Avoid using global variables when possible. They make code harder to understand, test, and debug because any function can modify them. Instead, pass values as arguments and return results.

Lambda Functions

Lambda functions are small, anonymous functions defined with the lambda keyword. They can have any number of parameters but only a single expression (no statements, no multi-line bodies). Lambdas are most useful as short, throwaway functions passed as arguments to higher-order functions like sorted(), map(), and filter():

python 
# Lambda syntax: lambda parameters: expression
square = lambda x: x ** 2
print(square(5))  # 25

add = lambda a, b: a + b
print(add(3, 7))  # 10

# Sorting with lambda as the key function
students = [
    {"name": "Alice", "gpa": 3.9},
    {"name": "Bob", "gpa": 3.5},
    {"name": "Charlie", "gpa": 3.7},
]
# Sort by GPA descending
sorted_students = sorted(students, key=lambda s: s["gpa"], reverse=True)
for s in sorted_students:
    print(f"{s['name']}: {s['gpa']}")

# map() — apply a function to every item
nums = [1, 2, 3, 4, 5]
doubled = list(map(lambda x: x * 2, nums))
print(doubled)  # [2, 4, 6, 8, 10]

# filter() — keep items that satisfy a condition
evens = list(filter(lambda x: x % 2 == 0, nums))
print(evens)  # [2, 4]

Decorators

Decorators are a powerful Python feature that allow you to modify or enhance functions without changing their code. A decorator is a function that takes another function as input and returns a new function with added behavior. They're widely used in frameworks like Flask and Django:

python 
import time

# Creating a decorator that measures execution time
def timer(func):
    """Decorator that prints how long a function takes."""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end - start:.4f}s")
        return result
    return wrapper

# Using the decorator with @ syntax
@timer
def slow_function():
    """Simulate a slow operation."""
    time.sleep(1)
    print("Done!")

slow_function()
# Output:
# Done!
# slow_function took 1.0012s

# Another practical decorator: retry on failure
def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"Attempt {attempt} failed: {e}")
            return None
        return wrapper
    return decorator

@retry(max_attempts=3)
def risky_operation():
    # Some operation that might fail
    pass
📝

Note: The @decorator syntax is just shorthand. Writing @timer above a function is equivalent to writing slow_function = timer(slow_function) after the function definition. The @ syntax is cleaner and more readable.
