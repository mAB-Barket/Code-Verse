Conditionals: if, elif, else

Making Decisions in Code

Conditional statements allow your program to make decisions and execute different blocks of code based on whether certain conditions are true or false. They are fundamental to all programming — without them, programs would always do the exact same thing regardless of input. In Python, the primary conditional keywords are if, elif (short for "else if"), and else.

Python uses indentation to define code blocks rather than braces or keywords. The standard is 4 spaces per indentation level. A condition is followed by a colon :, and all indented lines beneath it are part of that block. This enforced structure makes Python code visually clean and consistently readable.

Conditions in Python are expressions that evaluate to either True or False. You can use comparison operators (==, !=, <, >, <=, >=), logical operators (and, or, not), and membership operators (in, not in) to construct conditions.

python 
# Basic if/elif/else
age = 18

if age < 13:
    print("You are a child.")
elif age < 18:
    print("You are a teenager.")
elif age < 65:
    print("You are an adult.")
else:
    print("You are a senior.")

# Output: You are an adult.

# Using logical operators
temperature = 25
is_sunny = True

if temperature > 20 and is_sunny:
    print("Great day for a walk!")
elif temperature > 20 or is_sunny:
    print("Decent weather.")
else:
    print("Stay inside.")

Comparison & Logical Operators

Python provides a comprehensive set of operators for building conditions:

  • Comparison: == (equal), != (not equal), <, >, <=, >=
  • Logical: and, or, not
  • Identity: is, is not (check if objects are the same in memory)
  • Membership: in, not in (check if a value is in a sequence)
💡

Tip: Python supports chained comparisons — you can write 1 < x < 10 instead of x > 1 and x < 10. This is a clean, mathematical way to express range checks that most languages don't support natively.

python 
# Chained comparisons
x = 15
if 10 <= x <= 20:
    print("x is between 10 and 20")

# Membership operator
fruits = ["apple", "banana", "cherry"]
if "banana" in fruits:
    print("We have bananas!")

# Identity operator (is vs ==)
a = [1, 2, 3]
b = [1, 2, 3]
c = a
print(a == b)   # True (same values)
print(a is b)   # False (different objects)
print(a is c)   # True (same object)

# Checking for None — always use 'is'
result = None
if result is None:
    print("No result yet.")
⚠

Common Mistake: Use is to compare with None, not ==. Writing if x is None: is the correct Pythonic way. The == operator checks value equality, while is checks identity — and since None is a singleton, identity checking is more appropriate and reliable.

Truthiness and Falsy Values

In Python, every value has a "truthiness" — it evaluates to either True or False in a boolean context. This allows you to write very clean conditional code without explicit comparisons. The following values are considered falsy (evaluate to False):

  • False, None, 0, 0.0
  • Empty sequences: "", [], (), {}, set()

Everything else is truthy.

python 
# Truthy/Falsy in practice
name = ""
if name:
    print(f"Hello, {name}!")
else:
    print("Name is empty!")  # This runs

# Checking if a list has items
items = []
if not items:
    print("The list is empty.")

# Practical pattern: default values
user_input = ""
username = user_input or "Anonymous"
print(username)  # "Anonymous"

Ternary (Conditional) Expression

Python supports a concise one-line conditional expression, sometimes called the ternary operator. It lets you assign a value based on a condition in a single line:

python 
# Ternary / conditional expression
age = 20
status = "adult" if age >= 18 else "minor"
print(status)  # "adult"

# Useful in expressions and assignments
score = 85
grade = "Pass" if score >= 60 else "Fail"
print(f"Grade: {grade}")  # Grade: Pass

# Can be used in list comprehensions
numbers = [1, -2, 3, -4, 5]
signs = ["positive" if n > 0 else "negative" for n in numbers]
print(signs)  # ['positive', 'negative', 'positive', 'negative', 'positive']

Match Statement (Python 3.10+)

Python 3.10 introduced structural pattern matching with the match statement — similar to switch/case in other languages, but far more powerful. It can match patterns, destructure data, and even add conditions:

python 
# Match statement (Python 3.10+)
command = "quit"

match command:
    case "start":
        print("Starting the program...")
    case "stop" | "quit" | "exit":
        print("Goodbye!")
    case "help":
        print("Available commands: start, stop, help")
    case _:
        print(f"Unknown command: {command}")

# Pattern matching with destructuring
point = (0, 5)

match point:
    case (0, 0):
        print("Origin")
    case (0, y):
        print(f"On the Y-axis at y={y}")
    case (x, 0):
        print(f"On the X-axis at x={x}")
    case (x, y):
        print(f"Point at ({x}, {y})")
📝

Note: The match statement requires Python 3.10 or later. If you're using an older version, use if/elif/else chains instead. The underscore _ in a case acts as a wildcard that matches anything (like default in other languages).
