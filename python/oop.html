Object-Oriented Programming

What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that organizes code around objects ‚Äî entities that bundle related data (attributes) and behavior (methods) together. Instead of writing programs as sequences of instructions, OOP lets you model real-world concepts as objects that interact with each other. This approach makes code more organized, reusable, and easier to maintain.

Python is a multi-paradigm language that fully supports OOP. In fact, everything in Python is an object ‚Äî integers, strings, lists, functions, even classes themselves are objects. The four core principles of OOP are Encapsulation (bundling data and methods), Abstraction (hiding complexity), Inheritance (creating specialized versions of classes), and Polymorphism (treating different objects through a common interface).

Understanding OOP is essential for working with Python frameworks like Django, Flask, and PyQt, and for building large-scale applications. It's also the foundation for understanding design patterns ‚Äî proven solutions to common software design problems.

Classes and Objects

A class is a blueprint that defines the structure and behavior of objects. An object (also called an instance) is a specific entity created from that blueprint. Think of a class as a cookie cutter and objects as the cookies ‚Äî each cookie has the same shape but can have different decorations.

python 
# Defining a class
class Dog:
    """A class representing a dog."""
    
    # Class variable ‚Äî shared by all instances
    species = "Canis familiaris"
    
    # Constructor ‚Äî called when creating a new instance
    def __init__(self, name, age, breed):
        # Instance variables ‚Äî unique to each object
        self.name = name
        self.age = age
        self.breed = breed
    
    # Instance method
    def bark(self):
        return f"{self.name} says: Woof!"
    
    # Instance method with parameter
    def birthday(self):
        self.age += 1
        return f"Happy birthday, {self.name}! Now {self.age} years old."
    
    # String representation
    def __str__(self):
        return f"{self.name} ({self.breed}, {self.age} years)"

# Creating objects (instances)
dog1 = Dog("Buddy", 3, "Golden Retriever")
dog2 = Dog("Max", 5, "German Shepherd")

print(dog1)            # Buddy (Golden Retriever, 3 years)
print(dog1.bark())     # Buddy says: Woof!
print(dog1.birthday()) # Happy birthday, Buddy! Now 4 years old.
print(dog1.species)    # Canis familiaris
💡

Tip: The self parameter is a reference to the current instance of the class. It's always the first parameter of instance methods, but you don't pass it explicitly when calling the method ‚Äî Python handles that automatically. It's similar to this in Java or C++.

Inheritance

Inheritance allows you to create a new class (child/derived) based on an existing class (parent/base). The child class inherits all attributes and methods of the parent, and can add new ones or override existing ones. This promotes code reuse and establishes a logical hierarchy between related classes:

python 
# Base (parent) class
class Animal:
    def __init__(self, name, sound):
        self.name = name
        self.sound = sound
    
    def speak(self):
        return f"{self.name} says {self.sound}!"
    
    def __str__(self):
        return f"Animal: {self.name}"

# Child class ‚Äî inherits from Animal
class Cat(Animal):
    def __init__(self, name, color):
        super().__init__(name, "Meow")  # Call parent constructor
        self.color = color  # New attribute
    
    # New method specific to Cat
    def purr(self):
        return f"{self.name} is purring... üê±"

# Another child class
class Dog(Animal):
    def __init__(self, name, trick):
        super().__init__(name, "Woof")
        self.trick = trick
    
    # Override parent method
    def speak(self):
        return f"{self.name} barks: {self.sound}! {self.sound}!"
    
    def do_trick(self):
        return f"{self.name} performs: {self.trick}"

# Using the classes
cat = Cat("Whiskers", "orange")
dog = Dog("Rex", "roll over")

print(cat.speak())     # Whiskers says Meow!
print(cat.purr())      # Whiskers is purring... üê±
print(dog.speak())     # Rex barks: Woof! Woof!
print(dog.do_trick())  # Rex performs: roll over

# Polymorphism ‚Äî treat different types through same interface
animals = [cat, dog, Animal("Bird", "Tweet")]
for animal in animals:
    print(animal.speak())

Encapsulation

Encapsulation is the practice of restricting direct access to an object's internal data and providing controlled access through methods. In Python, there are no truly private attributes (unlike Java or C++), but the convention is to use a single underscore _ for "protected" and double underscore __ for "name-mangled" (strongly private) attributes:

python 
class BankAccount:
    """A simple bank account with encapsulation."""
    
    def __init__(self, owner, balance=0):
        self.owner = owner            # Public
        self.__balance = balance      # Private (name-mangled)
    
    # Property ‚Äî controlled access to private attribute
    @property
    def balance(self):
        return self.__balance
    
    def deposit(self, amount):
        if amount <= 0:
            raise ValueError("Deposit amount must be positive.")
        self.__balance += amount
        return f"Deposited ${amount}. Balance: ${self.__balance}"
    
    def withdraw(self, amount):
        if amount > self.__balance:
            raise ValueError("Insufficient funds!")
        if amount <= 0:
            raise ValueError("Withdrawal amount must be positive.")
        self.__balance -= amount
        return f"Withdrew ${amount}. Balance: ${self.__balance}"
    
    def __str__(self):
        return f"Account({self.owner}): ${self.__balance}"

# Using the class
account = BankAccount("Alice", 1000)
print(account.deposit(500))   # Deposited $500. Balance: $1500
print(account.withdraw(200))  # Withdrew $200. Balance: $1300
print(account.balance)        # 1300 (via @property)
# account.__balance ‚Üí AttributeError (private!)
📝

Note: Python's @property decorator lets you define methods that are accessed like attributes. This gives you the simplicity of attribute access with the control of getter/setter methods. You can also define @balance.setter to control how values are set.

Special (Dunder) Methods

Python classes can define special methods (also called "dunder" methods for their double underscores) to customize how objects behave with built-in operations like printing, comparison, arithmetic, and indexing:

python 
class Vector:
    """A 2D vector with full operator support."""
    
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    # String representation for users
    def __str__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Official representation for developers
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
    # Addition: v1 + v2
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
    # Length (magnitude) with len()
    def __abs__(self):
        return (self.x**2 + self.y**2) ** 0.5
    
    # Equality: v1 == v2
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
    
    # Scalar multiplication: v * 3
    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)

# Using the Vector class
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1 + v2)        # Vector(4, 6)
print(abs(v1))         # 5.0 (3-4-5 triangle!)
print(v1 * 3)         # Vector(9, 12)
print(v1 == Vector(3, 4))  # True
⚠

Best Practice: Always implement __repr__ for your classes ‚Äî it's used in the debugger and interactive shell. If you only implement one, choose __repr__ over __str__. __str__ falls back to __repr__ if not defined, but not vice versa.

Class Methods & Static Methods

Besides regular instance methods, Python classes can have two other types of methods:

  • Class methods (@classmethod) ‚Äî receive the class itself as the first argument (cls), not an instance. Often used as alternative constructors.
  • Static methods (@staticmethod) ‚Äî don't receive the instance or class. They're regular functions that belong to the class's namespace.
python 
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    
    # Class method ‚Äî alternative constructor
    @classmethod
    def from_string(cls, date_string):
        """Create a Date from 'YYYY-MM-DD' string."""
        year, month, day = map(int, date_string.split("-"))
        return cls(year, month, day)
    
    # Static method ‚Äî utility that doesn't need instance/class
    @staticmethod
    def is_leap_year(year):
        return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)
    
    def __str__(self):
        return f"{self.year}-{self.month:02d}-{self.day:02d}"

# Using regular constructor
d1 = Date(2026, 2, 14)
print(d1)  # 2026-02-14

# Using class method (alternative constructor)
d2 = Date.from_string("2025-12-25")
print(d2)  # 2025-12-25

# Using static method
print(Date.is_leap_year(2024))  # True
print(Date.is_leap_year(2025))  # False
💡

Tip: Use @classmethod for factory methods that create instances in different ways. Use @staticmethod for utility functions that logically belong to the class but don't need access to instance or class data. This keeps your code organized and intentional.
