Strings & String Methods

Working with Strings

Strings are one of the most widely used data types in Python. A string is a sequence of characters enclosed in quotes — either single quotes ('hello'), double quotes ("hello"), or triple quotes ('''hello''' or """hello""") for multi-line strings. Strings are immutable in Python, meaning once created, they cannot be modified in place — any "modification" actually creates a new string.

Python provides an extraordinarily rich set of built-in string methods that make text processing a breeze. Whether you need to search within strings, transform case, split text into parts, or format output, Python has you covered without needing any external libraries.

Strings in Python are also sequences, which means you can access individual characters using indexing, extract substrings using slicing, iterate over characters with loops, and use many of the same operations you'd use with lists.

python 
# Creating strings
single = 'Hello, World!'
double = "Hello, World!"
multi_line = """This is a
multi-line string that spans
across several lines."""

# String indexing (0-based)
greeting = "Python"
print(greeting[0])    # P (first character)
print(greeting[-1])   # n (last character)
print(greeting[2])    # t (third character)

# String slicing [start:end:step]
print(greeting[0:3])  # Pyt (characters 0, 1, 2)
print(greeting[2:])   # thon (from index 2 to end)
print(greeting[::2])  # Pto (every 2nd character)
print(greeting[::-1]) # nohtyP (reversed string)

# String length
print(len(greeting))   # 6
💡

Tip: Use triple-quoted strings ("""...""") for multi-line strings and documentation strings (docstrings). They preserve line breaks and are commonly used to document functions and classes in Python.

String Methods

Python strings come with dozens of powerful built-in methods. These methods don't modify the original string (since strings are immutable) — they return a new string instead. Here are the most commonly used ones:

python 
text = "  Hello, Python World!  "

# Case methods
print(text.upper())       # "  HELLO, PYTHON WORLD!  "
print(text.lower())       # "  hello, python world!  "
print(text.title())       # "  Hello, Python World!  "
print(text.swapcase())    # "  hELLO, pYTHON wORLD!  "

# Whitespace removal
print(text.strip())       # "Hello, Python World!"
print(text.lstrip())      # "Hello, Python World!  "
print(text.rstrip())      # "  Hello, Python World!"

# Search and check
print(text.find("Python"))    # 9 (index of first occurrence)
print(text.count("l"))       # 2
print(text.startswith("  H")) # True
print(text.endswith("!  "))   # True
print("Python" in text)      # True (membership test)

# Replace and split
print(text.replace("Python", "Java"))  # "  Hello, Java World!  "

words = "apple,banana,cherry"
print(words.split(","))    # ['apple', 'banana', 'cherry']

fruits = ["apple", "banana", "cherry"]
print(" & ".join(fruits))  # "apple & banana & cherry"

String Formatting

Python offers multiple ways to format strings. The modern and preferred approach is f-strings (formatted string literals), introduced in Python 3.6. They provide a concise, readable way to embed expressions inside strings:

python 
name = "Alice"
age = 30
gpa = 3.856

# f-strings (Python 3.6+) — the preferred way
print(f"My name is {name} and I'm {age} years old.")
print(f"Next year I'll be {age + 1}.")  # Expressions allowed!
print(f"GPA: {gpa:.2f}")              # GPA: 3.86 (2 decimal places)
print(f"Name: {name:>15}")            # Right-aligned in 15 chars
print(f"Name: {name:<15}")            # Left-aligned in 15 chars
print(f"Name: {name:^15}")            # Center-aligned in 15 chars

# .format() method (older style, still widely used)
print("Hello, {}! You are {} years old.".format(name, age))
print("Hello, {0}! {0} is {1}.".format(name, age))

# %-formatting (oldest style, C-like)
print("Hello, %s! You are %d years old." % (name, age))
📝

Note: F-strings are not just more readable — they're also faster than .format() and % formatting because they're evaluated at runtime by the interpreter rather than calling a method. Always prefer f-strings in modern Python code.

Escape Characters & Raw Strings

Escape characters let you include special characters in strings that would otherwise be difficult or impossible to type directly. Python also supports raw strings (prefixed with r) that treat backslashes as literal characters:

python 
# Common escape characters
print("Line 1\nLine 2")        # \n = newline
print("Column1\tColumn2")      # \t = tab
print("She said \"hi!\"")       # \" = double quote
print("Backslash: \\")          # \\ = literal backslash

# Raw strings — backslashes are treated literally
path = r"C:\Users\name\Desktop"
print(path)  # C:\Users\name\Desktop (no escaping)

# Useful for regex patterns
import re
pattern = r"\d{3}-\d{4}"   # Phone number pattern
print(re.findall(pattern, "Call 555-1234 or 555-5678"))
# ['555-1234', '555-5678']
⚠

Remember: Strings are immutable in Python. You cannot change individual characters of a string. Operations like upper() and replace() always return a new string; the original remains unchanged.
