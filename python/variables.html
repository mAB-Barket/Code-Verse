Variables & Data Types

Understanding Variables

In Python, a variable is a name that refers to a value stored in memory. Unlike languages like C++ or Java, you don't need to declare a variable's type before using it. Python uses dynamic typing, meaning the interpreter automatically determines the data type based on the value you assign. This makes Python extremely flexible and fast to write.

Variables in Python are created the moment you first assign a value to them. There's no special keyword like var, let, or int needed. You simply choose a name, use the assignment operator =, and provide a value. Python will handle the rest behind the scenes.

Variable names in Python must follow certain rules: they must start with a letter or underscore, can contain letters, numbers, and underscores, and are case-sensitive. Python convention (PEP 8) recommends using snake_case for variable names — lowercase words separated by underscores.

python 
# Creating variables — no type declaration needed
name = "Alice"          # str (string)
age = 25                # int (integer)
height = 5.7            # float (decimal)
is_student = True       # bool (boolean)

# Print variables and their types
print(name, type(name))          # Alice <class 'str'>
print(age, type(age))            # 25 <class 'int'>
print(height, type(height))      # 5.7 <class 'float'>
print(is_student, type(is_student))  # True <class 'bool'>

# Variables can be reassigned to different types
x = 10        # x is an int
x = "hello"   # now x is a str — totally valid!
print(x)       # hello

Core Data Types

Python has several built-in data types that form the foundation of all programs. Understanding these types is essential for writing effective Python code. The most commonly used primitive types are:

  • int — Integer numbers (whole numbers) with unlimited precision: 42, -7, 1000000
  • float — Floating-point numbers (decimals): 3.14, -0.001, 2.0
  • str — Strings (text), enclosed in single or double quotes: "hello", 'world'
  • bool — Boolean values: True or False
  • NoneType — The special None value, representing the absence of a value
  • complex — Complex numbers: 3+4j
💡

Tip: Python integers have unlimited precision — they can be as large as your memory allows. Unlike C++ or Java, you'll never encounter integer overflow in Python. Try computing 2 ** 1000 and see!

Type Conversion (Casting)

Sometimes you need to convert values from one type to another. Python provides built-in functions for type conversion, also called casting. This is particularly important when working with user input, since input() always returns a string.

python 
# Type conversion (casting)
x = "42"
y = int(x)         # Convert string to int → 42
z = float(x)       # Convert string to float → 42.0

# Convert numbers to strings
num = 100
text = str(num)     # "100"

# Boolean conversions
print(bool(0))       # False — 0 is "falsy"
print(bool(1))       # True — non-zero is "truthy"
print(bool(""))      # False — empty string is falsy
print(bool("hi"))    # True — non-empty string is truthy

# Practical example: user input is always a string
age_input = input("Enter your age: ")    # Returns a string
age = int(age_input)                      # Convert to integer
print(f"In 10 years you'll be {age + 10}")

Multiple Assignment

Python supports several convenient shorthand techniques for variable assignment that can make your code more concise and readable:

python 
# Multiple assignment on one line
a, b, c = 1, 2, 3
print(a, b, c)  # 1 2 3

# Assign same value to multiple variables
x = y = z = 0
print(x, y, z)  # 0 0 0

# Swap variables — no temp variable needed!
a, b = 10, 20
a, b = b, a       # Swap in one line
print(a, b)      # 20 10

# Augmented assignment operators
count = 0
count += 1   # same as count = count + 1
count *= 5   # same as count = count * 5
count -= 2   # same as count = count - 2
print(count)  # 3
⚠

Caution: Python does NOT have a ++ or -- increment/decrement operator like C++ or Java. Use x += 1 or x -= 1 instead.

Constants

Python doesn't have true constants like some other languages. By convention, variables that should be treated as constants are written in UPPER_SNAKE_CASE. While nothing prevents you from reassigning them, the naming convention signals to other developers that the value should not be changed:

python 
# Constants (by convention — UPPER_SNAKE_CASE)
PI = 3.14159
MAX_SPEED = 300000    # km/s (speed of light)
GRAVITY = 9.81        # m/s²
APP_NAME = "CodeVerse"

# Using constants
radius = 5
area = PI * radius ** 2
print(f"Area of circle: {area}")  # Area of circle: 78.53975
